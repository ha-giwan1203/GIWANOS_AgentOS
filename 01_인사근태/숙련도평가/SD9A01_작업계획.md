## 0. 선행 조건 (Phase 0) — 작업표준서 통합

**⚠️ 중요: 작업표준서가 2개 버전 존재 → 통합본 먼저 작성**

| 파일 | 날짜 | Rev | 경로 |
|-----|-----|-----|------|
| 구버전 | 180706 | Rev.16 | `SD9A01_표준문서/SD9A01 작업표준서(180706) Rev.16.xls` |
| 신버전 | 260101 | Rev.16 | `SD9A01_표준문서/SD9A01 작업표준서(260101) Rev.16.xls` |

**처리 방안 (다음 세션에서 사용자 확인):**
- **A안**: 최신본(260101) 단독 사용, 구버전 아카이브로 이동
- **B안**: 양 파일 시트별 비교 → 차이점 추출 → 통합본 작성 (`SD9A01 작업표준서 통합_Rev16.xlsx`)
- **C안**: 사용자가 직접 통합 후 1개 파일만 지정

**권장**: B안으로 자동 비교 후 사용자 검토 (Python으로 시트명/자주검사 항목 diff)

## 2. 작업 단계

### Phase 1 — 작업표준서 통합 (신규)
1. 두 파일의 시트 목록 비교 → 차이 시트 식별
2. 각 시트의 자주검사 항목 수/내용 비교
3. 통합본 생성: 최신(260101) 기준 + 구버전(180706) 전용 항목 병합
4. 사용자 검토 후 확정
5. 스크립트: `create_sd9a01_workstandard_merged.py` (신규 작성)

### Phase 2 — 공정별 양식 생성 (SP3M3 재사용)
1. 샘플 파일은 SP3M3과 공용 (`공정 평가서표준_SP3M3_샘플.xlsx`)
2. SD9A01 공정 10개 각각에 대해:
   - 공정명 기준으로 작업표준서 통합본 매핑
   - 공정명 기준으로 관리계획서 매핑
   - 작업표준서 5항목 + 관리계획서 4항목 → 전문강화 섹션 9항목
   - 평가기준 Q/U/Y 항목별 의미 맞게 재작성
3. 출력: `SD9A01_공정별 평가서/SD9A01_공정{N} 숙련도 평가서.xlsx` × 10개
4. 스크립트: `create_sd9a01_v1.py` (SP3M3 v4 기반 복사·수정)

### Phase 3 — 개인별 파일 생성 (SP3M3 재사용)
1. 24명 각 파일에서 추출:
   - 이름/사번/직급 (V2, V3, Z3)
   - 평가일 (AB5)
   - 수행공정 목록 (col 14/46/78/110/142/174/...)
   - 공정별 총점/레벨 (col 10/42/74.../col 20/52/84...)
   - 주공정 = 첫 번째 공정
2. 마스터 템플릿 생성 (공정 10개 시트 포함)
3. 작업자별 파일:
   - 수행 공정 시트만 유지, 나머지 삭제
   - 주공정 시트 맨 앞 배치
   - 작업자 정보 기입
   - AC열 점수 랜덤 분배 (기존 총점 보존, 0점 없음)
   - Z3 = "주공정" / "전환공정" 자동 설정
4. 출력: `SD9A01_개인별 평가서/{조}_{이름}/{이름} 숙련도평가서.xlsx` × 24개
5. 스크립트: `create_sd9a01_personal.py` (SP3M3 personal_v2 기반 복사·수정)

### Phase 4 — 검증
- 파일 수: 공정별 10개 + 개인별 24개 = **34개**
- 수식 보존 확인 (data_only=False 로드)
- 총점 = 원본 개인 파일 총점 일치
- 날짜 문자열 'YYYY-MM-DD' 확인
- 주공정/전환공정 라벨 확인
- AC열 0점 없음 확인
