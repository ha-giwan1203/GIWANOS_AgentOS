## 원칙
작업의 규모를 파일 수가 아니라 **영향 반경**으로 판정한다.
파일 1개라도 핵심 수식이면 Full, 파일 3개라도 문구 수정이면 Fast.

---

## Fast Lane 조건 (아래 **전부** 만족 시)

1. **수정 범위가 국소적이다**
   - 한 시트, 한 함수, 한 블록 수준에서 끝난다

2. **구조를 안 건드린다**
   - 시트명, 컬럼 구조, 파일명, 경로, 접두어/코드, 참조 체계, API 입출력 형식 변경 없음

3. **업무 규칙을 안 바꾼다**
   - 생산수량 산식, 정산 단가 계산, 라인배치 판정, 업로드 규칙 같은 비즈니스 로직 변경 없음

4. **검증이 짧고 명확하다**
   - 수정 후 확인 포인트가 3~5개 내에서 끝나고, 전/후 비교로 바로 판정 가능

5. **실패 영향이 낮다**
   - 틀려도 월마감, 생산지시, 업로드, 정산금액, 품번 매칭 전체를 흔들지 않음

### Fast Lane 절차
- plan.md 생략 가능
- research.md 생략 가능
- 구현 → 검증 → 커밋 (짧은 루프)

---

## Full Lane 조건 (아래 중 **하나라도** 해당 시)

1. 시트 간 참조 구조 변경
2. 수식 체계 변경
3. VBA/스크립트 로직 변경
4. 컬럼 추가/삭제/이동
5. 업로드 양식/외부 인터페이스 영향
6. 접두어/코드/품번 정규화 규칙 변경
7. 출력물 서열, 조건부서식, 숨김행, 병합, 인쇄영역 영향
8. 결과가 생산/정산/업로드 실무에 바로 쓰이는 파일이고, 해당 수정이 값/구조/양식/표시 결과에 영향을 주는 경우

### Full Lane 절차
- research.md 작성 → plan.md 승인 → 구현 → 검증 → 커밋
- plan.md 승인 전 구현 금지 (기존 plan-first 규칙 적용)

## 판정 예시

| 작업 | 판정 | 이유 |
|------|------|------|
| 로그 메시지 문구 수정 | Fast | 국소적, 구조 무관, 실패 영향 없음 |
| CLAUDE.md 규칙 추가 | Fast | 문서 변경, 코드/수식 무관 |
| 정산 단가 계산 로직 수정 | Full | 업무 규칙 변경 + 정산금액 직접 영향 |
| 생산지시서 수식 변경 | Full | 핵심 수식 + 실무 파일 |
| 업로드 양식 컬럼 추가 | Full | 외부 인터페이스 + 구조 변경 |
| 스킬 내부 셀렉터 수정 | Fast | 국소적 + 구조 무관 (되돌리기 쉬움) |
