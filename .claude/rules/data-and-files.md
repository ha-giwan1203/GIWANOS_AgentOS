# 데이터 처리 원칙
- 사용자 확인 없이 컬럼 구조를 임의 확정하지 않는다
- 수식 셀, 참조 구조, 코드 값은 검증 없이 수정하지 않는다
- openpyxl 사용 시 data_only=False로 열어 수식을 보존한다
- 결과 보고 시 판정 기준을 함께 남긴다

# 파일 정리 원칙
- 폴더 구조는 유지한다. 폴더 삭제보다 파일 분류를 우선한다
- 즉시 삭제 금지 — 정리 후보는 먼저 `98_아카이브/정리대기_YYYYMMDD/`로 이동
- 애매하면 보류 처리. 대표본 선정 기준: 최종 구조 경로 → 최종/확정/final 포함 → 최신 수정일

## Fast / Full Lane 판정 (영향 반경 기준, 파일 수 무관)

**Fast Lane** (전부 만족 시 → main 직행):
- 수정 범위 국소적 (한 시트/함수/블록), 구조 불변 (시트명/컬럼/경로/참조체계)
- 업무 규칙 불변 (산식/단가/판정 로직), 실패 영향 낮음

**Full Lane** (하나라도 해당 시 → candidate 브랜치):
- 시트 간 참조/수식 체계/스크립트 로직/컬럼 구조 변경
- 업로드 양식/외부 인터페이스 영향, 실무 파일의 값/구조/양식 영향

## Candidate 브랜치 규칙
- Full Lane 작업은 `candidate/{YYYYMMDD}-{주제}` 브랜치에서 작업
- candidate → main 머지는 GPT 리뷰 또는 사용자 확인 후 수행
- 자동 강제 없음 — 사용자가 Full Lane 선언 시 Claude가 candidate 브랜치 생성 → 작업 → PR 생성까지 수행
