## 원칙
독립적인 작업은 항상 병렬 우선. 순차 실행은 의존성이 있을 때만.

---

## 병렬 실행 (기본값)

읽기·탐색·검증 계열은 subagent 2~3개 동시 투입한다.

- 파일 여러 개 읽기/검색
- 독립적인 검증 작업 (smoke test, py_compile, 로그 확인 등)
- 멀티소스 자료 수집
- 코드·문서 교차 리뷰
- 배치형 정적 검사

## 직렬 실행 (의존성 있을 때만)

쓰기·반영·커밋 계열은 순서 보장이 필요하다.

- git commit / push
- 상태 문서(TASKS/HANDOFF/STATUS) 갱신
- ERP·MES 입력
- 최종 판정 (단일 기준 원본 필요)

## GPT 대기 중 병행

GPT 응답 대기(5~60초) 동안 background Agent로 병행 가능한 작업:

- 다음 단계 입력 텍스트 준비
- 로그 점검
- 상태문서 초안 작성
- 독립적인 다른 대기 항목 처리

단, 메인 세션과 동일 파일 동시 수정은 금지.
