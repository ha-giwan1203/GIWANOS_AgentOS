# MES 직접분석 사용설명서 v1.0

작성일: 2026-04-10
분석 방식: MES 직접 로그인 후 화면 탐색 + DOM 스캔
분석 상태: DRAFT

## 1. 목적

이 문서는 AI가 실제 MES 화면을 기준으로
`어떤 메뉴가 있고`, `각 화면에서 무엇을 조회/입력할 수 있으며`,
`어떤 데이터가 보이는지`를 빠르게 파악할 수 있도록 만든 직접 분석형 사용설명서다.

이번 버전은 문서 추정이 아니라
2026-04-10에 실제 `mes-dev`에 로그인해 확인한 화면을 기준으로 정리했다.

## 3. 공통 화면 구조

이번 스캔에서 공통적으로 확인된 UI 패턴은 아래와 같다.

1. 상단 글로벌 메뉴에서 도메인별 화면 진입
2. 메인 내용은 iframe 방식으로 로드
3. 대부분의 화면은 `검색조건 -> 조회 버튼 -> 결과 그리드 -> 상세/집계영역` 구조
4. 공통 버튼은 `초기화`, `검색`, `도움말`, `엑셀`
5. 기본 진입 시 `표시 할 행이 없습니다.` 상태인 화면이 많아, 실제 데이터는 검색 후 확인하는 구조

AI가 화면을 해석할 때는 아래 순서로 보면 된다.

1. 상단 메뉴로 도메인 확인
2. 좌측 또는 상단 검색조건 확인
3. 엑셀/저장/반영 버튼 유무 확인
4. 그리드 제목과 컬럼명으로 데이터 성격 판단

## 4. 상위 메뉴 맵

| 상위 메뉴 | 실제 성격 |
|-----------|-----------|
| 작업자 관리 | 작업자 기본정보, 배치, 숙련도, 근무현황, 인증서 관리 |
| 생산관리 | 생산계획/지시 기반 생산 현황 조회 |
| 공정관리 | 공정이상, 공정재고, 실사반영 등 현장 운영 관리 |

| 모니터링 | 라벨/ALC 이종체크, 생산성, 비가동 분석 |
| 생산현황 | 일일/누계 실적, 계획 대비 실적, 차종·품번별 실적, Excel 업로드 |

## 6. 화면상 공통 데이터 축

직접 확인한 화면들에서 반복적으로 등장한 공통 기준축은 아래와 같다.

| 기준축 | 설명 |
|--------|------|
| 조립업체 | 거의 모든 화면에서 핵심 필터. 기본적으로 `0109 대원테크`를 포함한 다수 업체가 선택식으로 존재 |
| 라인구분 | `OUTER`, `MAIN`, `SUB`가 반복적으로 확인됨 |
| 조립라인 | 라인코드 또는 라인명 기준 필터 |
| 제품군 | `RETRACTOR`, `BUCKLE`, `H/ADJ`, `ANCHOR`, `CONNECT`, `ECU ASSY`, `ATTACHMENT` 등 확인 |
| 기간/생산일자 | 일자 단일 조회와 기간 조회가 화면 유형에 따라 구분됨 |
| 품번/LOT/ALC | 이력 추적 계열 화면의 핵심 검색 키 |

## 7. AI 질문 라우팅 기준

AI가 MES 관련 질문을 받으면 아래처럼 연결하면 된다.

| 사용자 질문 유형 | 먼저 볼 화면 |
|------------------|--------------|
| 작업자 등록, 인증서, 입사/퇴사, 담당라인 | `작업자 등록 [PM]` |
| 오늘 누가 어느 라인에 배치됐는지 | `작업자 배치 [PM]`, `작업자 근무현황[PM]` |
| 작업자 숙련도, 라인별 숙련도 | `라인별 숙련도 MATRIX [PM]` |
| 생산계획과 실적 비교 | `생산 현황`, `생산계획 대비 실적 현황(일일/누계)` |
| 공정이상 원인/조치 | `공정이상 조치`, `공정이상 현황` |
| 공정재고, 현재고, 실사반영 | 공정재고 4개 화면 |
| 제품/부품 LOT 추적 | `부품 LOT 추적`, `제품 LOT 추적`, `제품 생산 이력 추적` |
| 고객 ALC 추적 | `고객 ALC 추적` |
| 미싱라벨, ALC 이종체크 | 모니터링 2개 화면 |
| 생산성, 비가동 분석 | `생산성 분석`, `라인별 비가동 분석` |
| 일일 생산실적 요약 | `생산실적 현황(일일)` |
| 라인별 실적 업로드/수정 | `라인별 생산실적(Excel Upload)` |

## 8. 이번 직접분석으로 확인된 핵심 포인트

1. 현재 MES는 `실적 조회`와 `실적 업로드`가 분리돼 있다.
2. `라인별 생산실적(Excel Upload)`는 조회 화면이 아니라 운영/입력 화면 성격이 강하다.
3. 생산이력 계열은 `LOT`, `ALC`, `BOX`, `공정별 검사결과`까지 추적하는 구조로 되어 있다.
4. 작업자 관리 계열은 단순 명단이 아니라 `배치`, `숙련도`, `근무현황`, `인증서 발급`까지 포함한다.
5. 공정관리 계열은 이상관리와 재고관리 두 축으로 분리돼 있다.

## 10. 실무 메모

- 기본 로그인 후 첫 화면은 `작업자 등록 [PM]`이었다.
