# 업무리스트 커넥터 운영 지침 v1.0

작성일: 2026-03-28
적용 범위: 업무리스트 프로젝트 전체

---

## 1. 운영 전제

Google Drive는 로컬 동기화 폴더로 운영 중이다.
실제 편집 작업은 로컬 업무리스트 폴더에서 수행한다.
커넥터는 검색·참조·상태관리·접수·협업 보조 도구로 역할을 분리한다.

---

## 2. 단일 기준 위치

| 정보 유형 | 기준 위치 |
|----------|----------|
| 실제 작업 | 로컬 업무리스트 폴더 |
| 원본 보관 | Google Drive |
| 규칙/프롬프트/스킬/스크립트 | GitHub |
| 작업 상태 | Notion / STATUS.md / TASKS.md |
| 일정·마감 | Google Calendar |
| 요청 접수 | Gmail |
| 협업/보고 | Slack |
| 완료/구버전/정리대기 | 98_아카이브 |

같은 최신본 파일을 여러 저장소에 중복 유지하지 않는다.

---

## 3. 커넥터별 세부 운영 기준

### 3.1 로컬 업무리스트 폴더 (기준)
- 실제 작업 공간
- 엑셀, 문서, 스크립트 수정은 로컬에서 수행
- 현재 최신 작업본 기준 위치
- 자동화 실행 기준 위치

### 3.2 Google Drive
- 원본 파일 보관 위치
- 수신본, 공유본, 기준본 저장
- 로컬과 동기화되는 원본 저장소
- 실제 편집은 로컬에서, Drive는 보관 기준으로만 본다
- **금지**: 편집 도구 사용, 매일 수정하는 작업 엑셀의 주 작업 도구로 사용

### 3.3 Drive 커넥트 (MCP)
- 파일 검색
- 과거 자료 참조
- PDF/문서 요약
- 기준본 찾기
- **금지**: 편집용 사용, 매일 수정하는 작업 엑셀 주 도구로 사용

### 3.4 GitHub
- 운영 규칙 문서 저장 (CLAUDE.md, README.md, STATUS.md, TASKS.md)
- 프롬프트, 스킬 문서 저장
- 스크립트·자동화 코드 버전관리
- **금지**: 대용량 원본 엑셀 및 첨부파일 저장

### 3.5 Notion
- 작업목록 관리
- 진행상태 관리
- 우선순위 관리
- 보류 이슈 기록
- 회의 메모 요약
- **금지**: 파일 최신본 저장소로 사용

### 3.6 Gmail
- 요청 접수
- 회신 초안 작성
- 첨부파일 수신
- **금지**: 메일 자체를 저장소처럼 사용, 첨부파일 방치 (분류 후 로컬 또는 Drive 기준 위치로 이동 필수)

### 3.7 Google Calendar
- 일정 관리
- 마감일 관리
- 반복업무 리마인드
- **금지**: 상태관리 문서 대체

### 3.8 Slack
- 완료 보고
- 협업 알림
- 이슈 공유
- 빠른 질의응답
- 진행 상황 공지
- 작업 요청 전달
- **금지**: 최종 파일 저장소 사용, Slack에만 있고 폴더에 없는 자료 상태 유지, Slack에서 받은 파일 미이관, 규칙·스킬·스크립트·기준문서를 Slack 고정본으로만 보관

## 4. 파일 처리 절차

1. Gmail, Drive, Slack 등에서 파일 또는 요청 확인
2. 로컬 업무리스트 폴더의 적정 카테고리로 분류
3. 원본 여부 확인
4. 원본은 Drive 기준 위치와 동기화 유지
5. 작업은 로컬에서 수행
6. 규칙/프롬프트/스크립트 변경 시 GitHub 반영
7. 진행 상태 → Notion / STATUS.md 업데이트
8. 일정이 있으면 Calendar 반영
9. 완료 보고 또는 협업 공유가 필요하면 Slack에 공지
10. 완료자료 또는 구버전은 98_아카이브로 이동

---

## 5. 금지사항

- 원본 파일 직접 수정
- 저장 위치 규칙 무시
- 폴더 구조 임의 변경
- 파일명 임의 축약
- 결과물 없이 메일/Slack만 발송
- Notion을 원본 저장소처럼 사용
- Gmail에만 첨부파일을 남겨두고 폴더 분류하지 않는 행위
- Slack에만 파일을 두고 기준 폴더에 저장하지 않는 행위
- Slack 메시지를 기준 문서처럼 취급하는 행위
- Drive 커넥트를 실제 편집 저장소처럼 사용하는 행위
- GitHub에 대용량 원본 파일 적재
- 같은 자료를 여러 저장소에 중복 저장해 최신본 혼선 생성
- 규칙 없이 새 폴더를 무단 생성하는 행위
