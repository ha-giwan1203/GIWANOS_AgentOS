최종 갱신: 2026-03-28 (v1.2 — AI 세션 시작 순서 도메인 확장, 작업 종료 갱신 원칙, 파일명 고정 안내 추가)

## 1. 운영 전제

업무리스트 폴더는 Google Drive와 동기화된 폴더이다.
"로컬 작업"과 "원본 보관"은 동일한 위치(업무리스트 폴더)를 가리키며, Drive는 별도 원본 저장소가 아니라 업무리스트 폴더의 클라우드 동기화 경로이다.
Drive 커넥트(MCP)는 파일 검색·참조 보조 도구로만 사용한다.
GitHub는 GPT와 Claude가 공유하는 AI 업무공유 기준 저장소이다.
Notion은 사람용 진행상황 현황판이며, AI 기준 저장소와 역할을 혼용하지 않는다.

---

## 2. 단일 기준 위치

| 정보 유형 | 기준 위치 |
|----------|----------|
| 실제 작업 / 원본 파일 | 업무리스트 폴더 (Google Drive 동기화 폴더) |
| 규칙/프롬프트/스킬/스크립트/상태문서 | GitHub (AI 업무공유 기준 저장소) |
| 진행상황 현황판 (사람용) | Notion |
| 일정·마감 | Google Calendar |
| 요청 접수 | Gmail |
| 협업/보고/알림 | Slack |
| 완료/구버전/정리대기 | 98_아카이브 |

같은 최신본 파일을 여러 저장소에 중복 유지하지 않는다.

## 3. 커넥터별 세부 운영 기준

### 3.1 업무리스트 폴더 (기준)
- 실제 작업 공간 (= Google Drive 동기화 폴더)
- 엑셀, 문서, 스크립트 수정은 이 폴더에서 수행
- 현재 최신 작업본 기준 위치
- 자동화 실행 기준 위치

### 3.2 Google Drive
- 업무리스트 폴더와 동기화되는 클라우드 경로
- 별도의 원본 저장소가 아니다 — 업무리스트 폴더와 동일 기준
- **금지**: Drive를 별도 편집·보관 저장소처럼 이중 운영, 매일 수정하는 작업 엑셀의 주 작업 도구로 사용

### 3.3 Drive 커넥트 (MCP)
- 파일 검색
- 과거 자료 참조
- PDF/문서 요약
- 기준본 찾기
- **금지**: 편집 도구로 사용, 매일 수정하는 작업 엑셀 주 도구로 사용

### 3.4 GitHub
- **AI 업무공유 기준 저장소**
- GPT와 Claude가 동일한 규칙, 상태, 작업 큐, 인수인계 기록을 공유하는 기준 저장소
- 관리 대상:
  - CLAUDE.md, README.md
  - STATUS.md, TASKS.md, HANDOFF.md (canonical 위치: `90_공통기준/업무관리/`)
  - 커넥터 운영지침
  - 프롬프트, 스킬 문서
  - 자동화 코드
  - 도메인별 규칙 문서
- **금지**: 대용량 원본 엑셀 및 첨부파일 저장

### 3.5 Notion
- **사람용 진행상황 현황판**
- 작업목록 관리
- 진행상태 관리
- 우선순위 관리
- 보류 이슈 기록
- 회의 메모 요약
- AI가 이어받는 기준 상태는 GitHub의 STATUS.md/TASKS.md/HANDOFF.md이다. Notion은 사람에게 보여주는 실시간 진행판이다.
- **금지**: 파일 최신본 저장소로 사용, AI 기준 저장소 혼용

### 3.6 Gmail
- 요청 접수
- 회신 초안 작성
- 첨부파일 수신
- **금지**: 메일 자체를 저장소처럼 사용, 첨부파일 방치 (분류 후 업무리스트 폴더 기준 위치로 이동 필수)

### 3.7 Google Calendar
- 일정 관리
- 마감일 관리
- 반복업무 리마인드
- **금지**: 상태관리 문서 대체

### 3.8 Slack
- 완료 보고
- 협업 알림
- 이슈 공유
- 빠른 질의응답
- 진행 상황 공지
- 작업 요청 전달
- **금지**: 최종 파일 저장소 사용, Slack에만 있고 폴더에 없는 자료 상태 유지, Slack에서 받은 파일 미이관, 규칙·스킬·스크립트·기준문서를 Slack 고정본으로만 보관

## 4. 파일 처리 절차

1. Gmail, Drive, Slack 등에서 파일 또는 요청 확인
2. 업무리스트 폴더의 적정 카테고리로 분류
3. 작업은 업무리스트 폴더에서 수행 (= Google Drive 동기화됨)
4. 규칙/프롬프트/스크립트 변경 시 GitHub 반영
5. 진행 상태 → Notion / STATUS.md 업데이트
6. 일정이 있으면 Calendar 반영
7. 완료 보고 또는 협업 공유가 필요하면 Slack에 공지
8. 완료자료 또는 구버전은 98_아카이브로 이동

## 5. 알림 원칙

- 완료·오류·장애 → Slack 발송
- 최종 기준 이력 → GitHub 문서 (STATUS.md / HANDOFF.md)
- 결과물 없이 메일/Slack만 발송 금지

## 6. AI 인수인계 원칙

새 AI 세션 시작 시 아래 순서로 읽고 작업을 이어받는다:

1. `90_공통기준/업무관리/HANDOFF.md` — 이전 세션 목적·변경·미완료·주의사항
2. `90_공통기준/업무관리/STATUS.md` — 전체 운영 현황
3. `90_공통기준/업무관리/TASKS.md` — 작업 목록 및 우선순위
4. 루트 `CLAUDE.md` — 프로젝트 전체 운영 원칙
5. 관련 도메인 `STATUS.md` — 해당 도메인 진행 위치·재개 지점
6. 관련 도메인 `TASKS.md` / `HANDOFF.md` (있으면) — 도메인 단위 작업 목록·인수인계
7. 관련 도메인 `CLAUDE.md` — 해당 업무 세부 규칙

작업 종료 전 최소 갱신 순서: 도메인 STATUS.md → TASKS.md → HANDOFF.md → Notion

## 6-1. 운영지침 파일명 안내

이 파일의 파일명(`운영지침_커넥터관리_v1.0.md`)은 고정이다.
내부 버전(현재 v1.1)이 최신 기준이며, 파일명의 v1.0은 초판 등록 번호로 변경하지 않는다.
버전 갱신은 문서 상단 `최종 갱신` 항목과 섹션 8 변경 이력에만 반영한다.

## 7. 금지사항

- 원본 파일 직접 수정
- 저장 위치 규칙 무시
- 폴더 구조 임의 변경
- Gmail에만 첨부파일을 남겨두고 폴더 분류하지 않는 행위
- Notion을 원본 저장소처럼 사용
- Slack에만 파일을 두고 기준 폴더에 저장하지 않는 행위
- Slack 메시지를 기준 문서처럼 취급하는 행위
- Drive 커넥트를 실제 편집 저장소처럼 사용하는 행위
- GitHub에 대용량 원본 파일 적재
- 같은 자료를 여러 저장소에 중복 저장해 최신본 혼선 생성
- 규칙 없이 새 폴더를 무단 생성하는 행위

## 7-1. 자동화 연결 권한 경계 (2026-04-01 추가)

> hooks, subagent, scheduled task가 외부 커넥터에 접근하는 권한 경계.

### 읽기 전용 (자동 허용)
- GitHub: git status, git diff, git log
- Google Drive (MCP): 파일 검색, 참조
- Notion: 페이지 읽기

### 쓰기 허용 (조건부)
- GitHub: git add/commit/push — hooks(block_dangerous.sh) 검증 후 허용
- Notion: 페이지 업데이트 — MCP allow 등록된 경우만
- Google Calendar: 이벤트 생성 — MCP allow 등록된 경우만

### 외부 전송 (제한적)
- Slack 알림: notify_slack.sh hook만 허용. 중복 방지 60초 이내 재전송 차단
- Gmail: 초안 작성만. 자동 발송 금지
- GPT 대화방: 토론모드 스킬 실행 중에만 브라우저 자동화로 전송

### 토큰/시크릿 관리
- settings.local.json에 토큰 하드코딩 금지 (2026-04-01 OAuth 토큰 제거 사례)
- 환경변수 또는 .env 파일로 분리
- git에 토큰 포함 파일 push 금지 (GitHub Push Protection 활성)

### 권한 주체별 정리
| 주체 | 외부 읽기 | 외부 쓰기 | 외부 전송 |
|------|----------|----------|----------|
| hooks | Git 읽기 | Git push (검증 후) | Slack 알림 |
| subagent | 파일 읽기만 | 금지 | 금지 |
| scheduled task | TASKS/STATUS 읽기 | 금지 | Slack 알림 (실패 시) |
| 토론모드 | GPT 대화 읽기 | GPT 대화 입력 | 금지 |
