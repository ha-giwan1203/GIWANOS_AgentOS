# 하네스 스킬 평가 기준표

## 목적

스킬 제작 시 Planner → Generator → Evaluator 구조로 품질을 평가하기 위한 기준표.
Generator가 만든 스킬을 별도 컨텍스트의 Evaluator가 이 기준표로 채점한다.

## Skill Rubric (100점 만점)

| 항목 | 가중치 | 합격 기준 | 불합격 예시 |
|------|--------|---------|------------|
| 트리거 정확도 | 30 | 의도한 상황에서만 발동, 오발동 없음 | 너무 넓은 키워드로 불필요한 상황에서 발동 |
| 오류 처리 | 25 | 입력 누락·예외 상황 대응 명시 | 필수 입력 없을 때 처리 없이 진행 |
| 실무 적용성 | 25 | 즉시 사용 가능한 출력 형식, 복붙 가능 | 추가 가공 필요한 중간 결과물만 출력 |
| 문서 완결성 | 20 | SKILL.md 필수 섹션 완비 (개요/트리거/실행절차/오류처리) | 섹션 누락, 예시 없음 |

## 최종 판정

| 판정 | 기준 |
|------|------|
| PASS | 80점 이상 |
| CONDITIONAL | 60~79점 — 조건부 사용, 피드백 반영 후 재평가 권장 |
| FAIL | 60점 미만 — Generator 재작업 필요 |
| BLOCKED | 실제 경로 미확인, 필수 입력 미존재, 검증 불가 상태 |

## 적용 절차

```
[Planner]
- 스킬 목적 / 트리거 조건 / 입출력 명세 / 오류 케이스 설계

[Generator]
- SKILL.md 작성
- 실행 스크립트 작성 (필요 시)
- .skill 패키지화

[Evaluator] ← 별도 컨텍스트
- 위 Rubric으로 채점
- 불합격 시: 항목별 피드백 + 구체적 수정 방향 명시
- PASS 시: 스킬 등록 승인
```

## 재작업 규칙

- 불합격 시 Evaluator 피드백만 Generator에 전달
- 최대 3회 재작업 허용
- 동일 실패 2회 반복 → BLOCKED, 사용자 확인 필요
