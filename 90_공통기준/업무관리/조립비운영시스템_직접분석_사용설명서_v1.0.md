# 조립비운영시스템 직접분석 사용설명서 v1.0

작성일: 2026-04-10
분석 방식: 실제 로그인 후 화면 확인 + 공개된 프론트 소스/실데이터 API 분석
분석 상태: DRAFT

## 1. 목적

이 문서는 AI가 실제 `조립협력사 조립비운영시스템`을 기준으로
`어떤 화면이 있는지`, `어떤 데이터가 조회되는지`, `어떤 운영 기능이 있는지`를
빠르게 파악할 수 있도록 만든 직접 분석형 사용설명서다.

이번 버전은 단순 화면 캡처 수준이 아니라
2026-04-10에 실제 로그인 후 확인한 화면, 실시간 API 응답, 공개된 프론트 소스를 함께 기준으로 정리했다.

## 3. 시스템 구조

### 3.1 확인된 라우트

App 라우터 기준으로 현재 확인된 주요 라우트는 아래 4개다.

| 경로 | 실제 역할 |
|------|-----------|
| `/login` | 조립업체/관리자 로그인 |
| `/assembly-cost-operation` | 조립비 현황 메인 대시보드 |
| `/line-operation-management` | 라인별 운영관리 |
| `/connection-info` | 시스템 연결 정보 |

### 3.2 화면 공통 구조

실제 로그인 후 확인한 공통 구조는 아래와 같다.

1. 상단 헤더에서 `연결정보`, 사용자명, `비밀번호 변경`, `로그아웃` 사용

3. 데이터 조회는 프론트에서 직접 REST API를 호출하는 방식
4. 조회 결과는 차트 + 필터 + 테이블 + 페이지네이션 조합으로 표시

## 5. 라인별 운영관리 화면

|------|------|

## 9. 이번 직접분석으로 확인된 핵심 포인트

1. 이 시스템은 `조립비 현황`, `라인별 운영관리`, `연결정보` 세 축으로 이해하면 가장 빠르다.
2. 메인 화면은 `조립비 금액/수량 분석`, 라인운영 화면은 `인원/시간/시급/인건비 관리` 역할이다.
3. 실제 API 기준 데이터 규모는 조립비 `153,933건`, 라인운영 `671건`이다.
4. 로그인 사용자 `대원테크` 기준 메인 대시보드는 `2026-04 / 1,993건 / 약 8.41천만원 / 616,433ea` 수준이다.
5. 프론트는 React + Wouter + Vite 개발 서버 구조로 보이며, 차트는 `recharts`를 사용한다.

## 10. 보안상 특이사항

이번 분석 중 실무적으로 매우 중요한 특이사항이 확인됐다.

1. 현재 서버는 `/src/*.tsx`, `/@vite/client`가 노출되는 `Vite 개발 서버` 형태로 보인다.
2. `connection-info` 프론트 소스 안에 DB 연결 정보와 추가 접근 비밀번호가 클라이언트 코드로 포함돼 있다.
3. `/api/assembly-cost-operation`, `/api/line-operation`, `/api/line-operation/stats`, `/api/wage-table`는 별도 로그인 세션 없이도 직접 응답이 내려왔다.
4. 즉, 현재 구조는 `소스 노출 + API 무인증 조회 + 연결정보 클라이언트 노출`이라는 보안 리스크가 있다.

주의:

- 이 문서에는 실제 비밀번호 값은 적지 않았다.
- 다만 현재 배포 방식은 즉시 점검이 필요한 수준으로 보인다.

## 11. 한계와 다음 보강 항목

이번 1차 문서는 실제 화면과 API 구조를 빠르게 파악하는 데 초점을 맞췄다.

다음 버전에서 확장하면 좋은 항목은 아래와 같다.

1. `라인별 운영관리` 실화면을 직접 열어 필터/컬럼/팝업을 DOM 기준으로 추가 검증
2. `비밀번호 변경` 화면/팝업과 실제 변경 규칙 확인
3. `Oracle 연동` 버튼 클릭 후 실제 후속 처리 흐름 확인
4. 업체별 권한 차이와 관리자 로그인 차이 비교
5. ERP/MES/ZDM와의 데이터 매핑표 작성

## 12. 실무 메모

- 로그인 상태는 `localStorage.user` 기준으로 유지된다.
- 메인 화면 기본 필터는 현재월 기준으로 자동 세팅된다.
- 조립비 현황은 회사별 월 집계 질문에 바로 쓰기 좋고, 라인운영은 인건비/인원 관련 질문 대응에 적합하다.
- 이 시스템은 기능 분석도 중요하지만, 현재는 보안/배포 형태 점검이 함께 필요하다.
