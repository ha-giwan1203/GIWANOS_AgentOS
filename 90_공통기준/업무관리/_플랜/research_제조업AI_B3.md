## 3. 확대 후보 (현업 업무 원본 기준)

| 후보 | 현재 | 자동화 가능성 | 임팩트 |
|------|------|-------------|--------|
| 생산계획표 자동화 | GPT 프로젝트 존재 | 높음 (엑셀→자동 생성) | 높음 |
| 품질 불량 대책서 정리 | 수동 작성 | 중간 (템플릿+데이터 자동 채움) | 중간 |
| 임률단가 테이블 갱신 | GPT 프로젝트 존재 | 높음 (연도별 자동 계산) | 중간 |
| BI 실적 집계 자동화 | bat 복사만 | 높음 (파싱→집계→대시보드) | 높음 |
| 외주발주 납품 관리 | 수동 | 낮음 (구조 파악 필요) | 낮음 |

| 데이터 기반 생산 최적화 | BI 실적 수집만 | 분석/예측 단계 미도달 |

## 5. 판정

**현재 자동화 범위는 제조업 트렌드 대비 RPA+AI 영역에서 앞서 있음.**
Gap은 "데이터 분석/예측" 단계이나 이는 장기 과제.

**단기 확대 후보 (임팩트 순):**
1. 생산계획표 자동화 — GPT 프로젝트 이미 존재, Claude 스킬화 가능
2. BI 실적 집계 자동화 — 현재 bat 복사만 → 파싱+집계+대시보드 확장
3. 임률단가 테이블 — GPT 프로젝트 존재, 연도별 자동 계산

이 3건은 이미 GPT 프로젝트나 기초 스크립트가 있어 research→plan→구현 가능.

별도 plan 없이 research로 종료 권장 (실행은 현업 우선순위에 따라 별도 착수).
