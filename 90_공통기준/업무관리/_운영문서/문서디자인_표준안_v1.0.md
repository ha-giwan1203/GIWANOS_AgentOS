# 문서 디자인 표준안 v1.0

작성일: 2026-04-09
적용 범위: `업무리스트` 전체 문서, 관리대장, 보고서, 템플릿
상태: DRAFT

## 1. 목적

문서 품질을 높이는 기준은 내용만이 아니다.
같은 저장소 안에서 제목 체계, 표 구조, 파일명, 시트명, 보고 순서가 계속 바뀌면
검색, 인수인계, 자동화, 월간 보고가 모두 느려진다.

이 문서는 `업무리스트` 전반에 공통 적용할 문서 디자인 기준을 정한다.

## 2. 공통 원칙

1. 입력 문서와 보고 문서를 분리한다.
2. 원장은 하나로 유지하고, 설비/품질/자재는 보기 시트를 분리한다.
3. 제목, 표 머리글, 파일명은 사람이 찾기 쉬운 동일 규칙을 따른다.
4. 디자인은 화려함보다 `가독성`, `비교 가능성`, `재사용성`을 우선한다.
5. 같은 성격의 문서는 같은 순서와 같은 라벨을 유지한다.

## 3. 문서 분류

문서는 아래 4종으로 관리한다.

| 분류 | 목적 | 대표 예시 |
|------|------|-----------|
| 운영문서 | 규칙, 상태, 인수인계 | `README.md`, `STATUS.md`, `HANDOFF.md` |
| 보고서 | 기간별 분석 결과 정리 | 월간 이슈 보고서, ERP-MES-Flow 분석 보고서 |
| 관리대장 | 원장, 사건별 관리, 검토 이력 | 통합 원장 CSV, 설비/품질 시트 |
| 템플릿 | 반복 입력/반복 보고의 재사용 틀 | 일일 보전이력, 이슈분석 템플릿 |

## 4. Markdown 문서 기준

### 4.1 제목 구조

- 문서 제목은 `H1` 1개만 사용한다.
- 주요 섹션은 `H2`로 통일한다.
- 필요 시 상세 항목만 `H3`를 사용한다.
- `H4` 이하 단계는 원칙적으로 금지한다.

### 4.2 권장 섹션 순서

운영문서:

1. 목적
2. 현재 상태
3. 주요 파일/경로
4. 작업 기준
5. 주의사항

보고서:

1. 개요
2. 요약
3. 집계 결과
4. 주요 이슈
5. 판정/해석
6. 후속조치

템플릿 안내문서:

1. 사용 목적
2. 입력 대상
3. 시트 구성
4. 입력 규칙
5. 집계 규칙
6. 수정 이력

### 4.3 표 기준

- 표 제목은 본문 한 줄로 먼저 설명하고 표를 배치한다.
- 표 헤더는 가능한 한 짧고 고정된 단어를 사용한다.
- 같은 의미는 항상 같은 헤더를 쓴다.

권장 헤더:

- 날짜
- 라인명
- 공정명
- 대분류
- 중분류
- 소분류
- 내용요약
- 조치내용
- 정지시간(분)
- 금액
- 검토상태
- 비고

## 5. 파일명 기준

### 5.1 Markdown

- 보고서: `YYYY-MM-DD_주제_report.md`
- 운영문서: `주제_가이드.md`, `주제_표준안_v1.0.md`
- 인수인계: `HANDOFF.md`, `STATUS.md`, `TASKS.md` 유지

### 5.2 CSV

- 원장: `YYYY-MM-DD_주제_register.csv`
- 분류본: `주제_facility.csv`, `주제_quality.csv`, `주제_material.csv`
- 검토본: `주제_review.csv`

### 5.3 XLSX

- 템플릿: `주제_template.xlsx`
- 관리통합본: `주제_management_workbook.xlsx`
- 배포본: `주제_report.xlsx`

## 6. 엑셀 디자인 기준

### 6.1 시트명

공통 시트명은 아래 규칙을 우선한다.

- `원장`
- `설비이슈`
- `품질이슈`
- `자재이슈`
- `월간집계`
- `코드관리`
- `가이드`

영문 시트명은 외부 연동이나 코드 의존이 있을 때만 사용한다.

### 6.2 헤더

- 1행은 헤더 전용
- `A2` 고정틀 적용
- 헤더는 굵게 표시
- 필수 입력 컬럼은 좌측에 배치
- 계산 컬럼은 우측에 배치

권장 컬럼 순서:

1. 날짜
2. 라인
3. 공정
4. 대분류
5. 중분류
6. 소분류
7. 내용요약
8. 조치내용
9. 담당자
10. 정지시간
11. 금액
12. 검토상태
13. 비고

### 6.3 시각 규칙

- 입력 시트: 단순 표 중심
- 집계 시트: 상단 요약 카드 + 하단 표
- 코드 시트: 표준값 목록만 유지
- 가이드 시트: 예시 문장과 입력 규칙만 배치

## 7. 현재 저장소 기준 개선 우선순위

1. 보고서형 Markdown 구조 통일
2. 엑셀 시트명 한글 기준 통일
3. 원장/보고/템플릿 파일명 규칙 정리
4. `설비/품질/자재` 대분류 라벨 전 저장소 동일화
5. 초안 보고서를 표준 템플릿 기반으로 재작성

## 8. 적용 메모

- 기존 문서는 즉시 전면 수정하지 않는다.
- 신규 문서와 새로 갱신하는 문서부터 이 기준을 우선 적용한다.
- 코드/자동화가 이미 영문 시트명에 의존하는 파일은 영향 확인 후 전환한다.
