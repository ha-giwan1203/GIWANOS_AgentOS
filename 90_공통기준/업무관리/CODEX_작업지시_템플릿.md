## 4. 실행 지시

  - destructive git 명령 금지

## 5. 검증
- 실행할 검증:
  - `<예: python ... --json>`
- 작업 유형별 검증:
  - 브라우저 수집이면 조회 조건·건수·합계·예외를 확인한다.
  - PDF/이미지/HTML이면 열림·깨짐·잘림·겹침을 확인한다.
  - 코드 변경이면 문법검사 또는 관련 테스트를 실행한다.
- 확인할 결과:
  - `<PASS 조건>`
- 실패 시:
  - 원인과 재개 위치를 기록하고 멈춘다.
  - 마지막 줄에 `[NEEDS_CLARIFICATION] <필요한 결정>`을 남긴다.

## 6. 완료 처리
- TASKS.md:
  - `[완료] owner=Codex / <작업명> / 잠금 해제: <경로1>, <경로2>`
- HANDOFF.md:
  - 완료 내용, 검증 결과, 남은 예외를 1단락으로 기록
- STATUS.md:
  - 전체 운영 상태에 영향이 있으면 최종 업데이트만 갱신
- 리뷰:
  - 중요 변경은 `CODEX_리뷰루틴.md` 기준으로 Claude 검증 또는 `/codex:review` 대상 여부를 남긴다.
