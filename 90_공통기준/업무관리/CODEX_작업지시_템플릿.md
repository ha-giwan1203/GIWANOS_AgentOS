## 2. 작업 라우팅
> 비유: 작업 라우팅은 현장 작업반 편성표이다. 가장 잘하는 담당에게 맡기고, 만든 사람과 검사하는 사람을 분리한다.

| 작업 유형 | 기본 담당 | 검증 담당 | 외부 워커 |
|-----------|-----------|-----------|-----------|
| 사용자 의도 해석·도메인 판단·작업 설계 | Claude | Claude | 불필요 |
| 파일 수정·엑셀/CSV·문서·스크립트 실행 | Codex | Claude | 불필요 |
| 정산·라인배치·MES·D0 실행 패치 | Codex | Claude 도메인 검증 | 불필요 |
| 브라우저 수집·화면 확인 | Codex browser | Claude | 불필요 |
| PDF·이미지·수식·긴 문서 인식 | Codex 문서/비전 도구 우선 | Claude | 필요 시 승인 후 Gemini/비전 워커 |
| hook·skill·자동화 코드 변경 | Codex | Claude + Codex Critic | 불필요 |
| git commit·push | Codex | Claude diff 승인 후 | 불필요 |

- 외부 워커 필요 시 먼저 적을 것:
  - `누구 / 왜 / 입력 / 산출물 / 검증`

### git 권한 경계
> 비유: 로컬 커밋은 공장 안 작업완료 도장이고, push는 외부 거래처로 서류를 보내는 출하다. 둘은 같은 버튼처럼 보여도 위험도가 다르다.

| 단계 | 처리 기준 |
|------|-----------|
| 로컬 작업 | Codex가 실행한다. |
| 로컬 검증 | `daily_doc_check`, 테스트, 산출물 확인, git diff 중 작업에 맞는 증거를 남긴다. |
| 로컬 커밋 | Claude diff 검증 PASS 후 Codex가 실행한다. `.git/index.lock` 문제가 있으면 lock 잔존 확인/정리, 일반 경로 재시도, 실패 시 escalated 1회 순서로 제한한다. |
| 원격 push | 외부 반출 단계로 분리한다. Claude 검증 PASS와 사용자 push 발화가 모두 있어야 하며, Codex 단독 자동 push는 금지한다. |

push 전 사용자에게 확인할 4항목:

- 원격 주소(`origin`)
- 브랜치
- 커밋 해시
- 외부 반출 고지: 작업 문서가 GitHub 원격 저장소로 올라간다는 점

push가 차단되거나 실패하면 로컬 커밋 해시와 미푸시 상태를 보고하고, 우회 명령으로 재시도하지 않는다.

## 3. 작업 전용 하네스
> 명령이 들어오면 Codex는 바로 실행하지 말고, 먼저 이 5줄을 자체 설계한다.
> 비유: 작업 전용 하네스는 현장 작업 전 "무엇을 보면 통과인지" 정해 둔 검사표이다.
> 하네스 5필드가 빠지면 `doc_worklog.py start`와 `daily_doc_check.py`가 실패한다.

- 입력:
  - `<요청 원문, 기준 파일, 대상 데이터>`
- 작업범위:
  - `<바꿀 파일/폴더와 건드리지 않을 경계>`
- 성공기준:
  - `<무엇이면 완료인지>`
- 검증명령:
  - `<실행할 명령 또는 확인 방법>`
- 중단기준:
  - `<어떤 오류/불일치가 나오면 멈출지>`

### 큰 작업 추가 흐름
> 파일 2개 이상 수정, 코드/데이터/브라우저/시각산출물, 비가역 가능성이 있는 작업만 적용한다.
> 새 close-lite/full 같은 별도 구조를 만들지 말고 이 작업지시 안에 짧게 적는다.

- Plan:
  - `<목표, 제약, 잠금 파일, 먼저 읽을 문서>`
- Design:
  - `<바꿀 구조, 검증 기준, Claude 판단이 필요한 도메인 결정 여부>`
- Do:
  - `<최소 수정 순서>`
- Check:
  - `<자동검사, 실제 산출물 확인, git diff 확인>`

### 브리프 압축 원칙
> 비유: 작업자에게 모든 서류 박스를 넘기지 말고, 필요한 철과 검사표만 주는 방식이다.

- 긴 문서·코드·자막·로그는 전문을 붙이지 말고 파일 경로와 필요한 범위만 전달한다.
- 목표·성공기준·핵심 제약은 앞쪽에 둔다.
- 금지사항·중단기준·되돌리기 어려운 위험은 뒤쪽에 다시 적는다.
- 외부 AI/워커에게 넘기는 지시문은 해당 워커에게 필요한 입력·산출물·검증기준만 담는다.
- 내용이 길어지면 요약은 남기고 원문은 `99_임시수집/` 또는 기존 산출 경로에 둔다.

- 실행할 검증:
  - `<예: python ... --json>`
- 검증 원칙:
  - AI 완료 보고만으로 PASS 금지. 작업에 맞는 명령 결과·실제 산출물·브라우저 화면·엑셀/PDF 열림·git diff 중 최소 1개 이상을 확인한다.
