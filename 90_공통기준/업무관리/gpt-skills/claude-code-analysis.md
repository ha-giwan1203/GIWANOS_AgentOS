# 클로드코드 정밀평가 프로토콜 (GPT 전용)

> 이 파일은 GPT가 따르는 프로토콜이다. Claude Code skill과 달리 실행 엔진이 없다. 사용자가 트리거 단어를 입력하면 GPT가 이 파일을 GitHub App으로 조회하고 아래 절차·템플릿을 그대로 실행한다.
>
> **판정 우선순위**: 이 프로토콜 > `gpt-instructions.md` (상충 시 이 프로토콜이 우선, 단 "Git 기준 원본" 원칙은 공통 유지)

---

## 0. 트리거 & 모드

사용자 입력에 아래 문자열이 포함되면 이 프로토콜을 실행한다.

```
"클로드코드 정밀평가" | "/클코평가" | "harness audit" | "정밀평가"
```

옵션 플래그(사용자가 덧붙일 수 있음):

| 플래그 | 의미 | 기본값 |
|--------|------|--------|
| `--repo` | 이 저장소 하네스 전체 평가 | (기본) |
| `--plan` | Claude가 방금 낸 플랜/제안 검토 (플랜 경로·본문 동봉 필요) | — |
| `--cli` | Claude Code CLI 일반 설계 원론 질의 | — |
| `--depth={대\|중\|소\|전체}` | 분석 깊이 | `전체` |

플래그 없이 트리거만 입력되면 `--repo --depth=전체`로 간주.

## 1. 입력 수집 (시작 전 필수)

### 1.1 조회 우선순위

저장소 루트(`업무리스트/`)는 Google Drive 동기화 폴더다. Git App과 Drive는 같은 파일의 서로 다른 시점을 본다.

1. **GitHub App = 마지막 푸시 시점 = 기준 원본**. 최종 판정(정합/부분반영/미반영)은 Git 커밋만 근거.
2. **Drive = 로컬 실시간(미커밋 포함) = 보조**. Drive 단독 근거는 `임시검토`·`중간확인`까지.
3. **Git/Drive 차이 = 미커밋 = "커밋 누락 리스크"**. 리포트 §0에 필수 기록.

### 1.2 수집 체크리스트

아래 순서로 조회. 누락된 파일이 있는 축은 "기준 미확인"으로 표시하고 진행.

1. 루트 `CLAUDE.md` + `.claude/rules/*.md`
2. `.claude/settings.json`, `.claude/settings.local.json`
3. `.claude/hooks/README.md` + 가능하면 `list_active_hooks.sh` 최근 결과
4. `.claude/self/DESIGN_PRINCIPLES.md`, `selfcheck.sh`, `90_공통기준/protected_assets.yaml`
5. `90_공통기준/업무관리/TASKS.md` / `STATUS.md` / `HANDOFF.md`
6. `90_공통기준/스킬/*/SKILL.md` 목록 (파일명만)
7. `.claude/agents/*.md`, `.claude/commands/*.md` 목록
8. `.claude/incident_ledger.jsonl` 최근 50건
9. 도메인 CLAUDE.md: `05_생산실적/`, `10_라인배치/`, `02_급여단가/`, `04_생산계획/`, `06_생산관리/`, `90_공통기준/토론모드/` 최소
10. `.claude/plans/` 최근 5개 (`--plan` 모드일 때만)

## 2. 분석 계층 — 大 · 中 · 小 (순서 고정)

**大 → 中 → 小 순서로만 출력한다.** 역순·혼재 금지. 大를 생략하면 영향반경을 계산할 수 없다.

### 2.1 [大] Level-1 — 시스템 지형도

저장소를 아래 **7개 서브시스템**으로 분해한다.

| # | 서브시스템 | 포함 | 상태 원본 | 외부 의존 |
|---|-----------|------|----------|----------|
| S1 | 하네스 코어 | `.claude/hooks/`, `.claude/settings*.json`, `.claude/rules/` | settings.json + hooks/README.md | Claude Code CLI |
| S2 | 자기유지 층 | `.claude/self/`, `protected_assets.yaml`, selfcheck | DESIGN_PRINCIPLES.md | hooks(S1) |
| S3 | 스킬/커맨드/에이전트 | `90_공통기준/스킬/*`, `.claude/commands/`, `.claude/agents/` | 각 SKILL.md | hooks(S1), Chrome MCP/ERP/MES |
| S4 | 기준 문서 체인 | 루트 CLAUDE.md + 도메인 CLAUDE.md | 루트 CLAUDE.md | — |
| S5 | 상태 원본 계층 | TASKS.md, STATUS.md, HANDOFF.md, 도메인 STATUS | TASKS.md | incident_ledger |
| S6 | 도메인 실무 | 05_생산실적/, 10_라인배치/, 02_급여단가/, 04_생산계획/, 06_생산관리/ 등 | 각 도메인 STATUS.md | ERP/MES/xlsx 원본 |
| S7 | 외부 연동·토론 | 토론모드(ChatGPT/Gemini), Git/GitHub App, Drive, Notion | `90_공통기준/토론모드/CLAUDE.md` | Chrome MCP, GPT/Gemini 웹, Drive 동기화 |

**大 층 필수 출력**:
1. 서브시스템 7개 각각 건강도(정상/경계/위험) + 1줄 근거
2. 의존성 그래프 — 서브시스템 간 edge를 텍스트로 나열 (예: `S3(debate-mode) → S1(navigate_gate) → S7(ChatGPT 탭)`). 최소 5개 edge
3. **SPOF 후보** — 어느 서브시스템/파일이 깨지면 다수가 연쇄 실패하는가
4. **이번 분석 포커스** — 최근 커밋·세션·HANDOFF 맥락에서 추정되는 관심 영역 2~3개

### 2.2 [中] Level-2 — 5축 판정

위 S1~S7을 5개 축으로 재분류한다.

- **A1. hooks/permissions** (주로 S1·S2)
- **A2. skill/agent/command** (주로 S3)
- **A3. 기준 문서 정합** (주로 S4, 일부 S6)
- **A4. 상태·incident 정합** (주로 S5)
- **A5. 외부 연동·토론 프로토콜 준수** (주로 S3·S7)

각 축마다 판정 1개(정합/부분반영/미반영/보류/기준미확인) + 근거 파일:줄 + 심각도(상/중/하).

### 2.3 [小] Level-3 — 축 내 세부 질문 (전부 답)

GPT가 한 질문이라도 건너뛰면 출력 무효.

#### A1. hooks/permissions
- Q1-1. `.claude/hooks/README.md` 등급 분류(advisory/gate/measurement)가 실제 훅 스크립트 동작과 일치?
- Q1-2. exit 2 전파 훅이 전부 gate 등급으로 분류?
- Q1-3. `settings.local.json` permissions에 1회용·중복 항목? (`permissions_sanity.sh` 결과와 대조)
- Q1-4. 5단계 의사결정 트리(CLAUDE.md) 위반 항목?
- Q1-5. 활성 hook 수가 `list_active_hooks.sh --count` ↔ `AGENTS_GUIDE.md` 수치 일치?
- Q1-6. `hook_common.sh` wrapper(hook_advisory/hook_gate/hook_measure) 누락 훅?
- Q1-7. 동일 event(PreToolUse/PostToolUse/Stop 등)에 등록된 훅들의 실행 순서 의존 또는 중복 차단?

#### A2. skill/agent/command
- Q2-1. 동일 trigger 단어가 여러 skill·command에 중복 등록?
- Q2-2. 기능 중복 skill 쌍 (예: production-report vs daily-routine 교집합)?
- Q2-3. 최근 30일 호출 기록 없는 dead skill 추정?
- Q2-4. agent 설명문과 실제 tool 권한 모순?
- Q2-5. 같은 이름으로 command와 skill 이중 등록?
- Q2-6. SKILL.md Step 흐름이 해당 도메인 CLAUDE.md/REFERENCE.md와 일치?

#### A3. 기준 문서 정합
- Q3-1. 루트 CLAUDE.md ↔ 도메인 CLAUDE.md 규칙 모순?
- Q3-2. 같은 규칙이 2곳 이상 중복 기재?
- Q3-3. 참조된 경로·파일명이 실재?
- Q3-4. 400줄 초과 CLAUDE.md(토큰 위협)?
- Q3-5. 읽기 순서 선언 ↔ 실제 `@import` 체인 일치?
- Q3-6. 사용자 MEMORY.md와 저장소 CLAUDE.md 규칙 충돌?

#### A4. 상태·incident 정합
- Q4-1. TASKS.md "최종 업데이트" 날짜 ↔ HANDOFF 헤더 날짜 일치?
- Q4-2. HANDOFF 헤더 판정 레이블 ↔ 본문 결론 일치? (예: "GPT PASS" vs 본문 "조건부")
- Q4-3. 미해결 incident 중 7일 이상 잔존 태그?
- Q4-4. 동일 태그 재발 10회 이상 (구조적 버그 의심)?
- Q4-5. STATUS.md 재개 지점 ↔ TASKS.md 진행 상태 모순?
- Q4-6. Drive/Git 불일치 파일이 상태 원본 3종(TASKS/STATUS/HANDOFF)에 포함?

#### A5. 외부 연동·토론 프로토콜
- Q5-1. 2자·3자 토론 라운드 로그(`logs/debate_*/`) 완결성?
- Q5-2. 상호 감시 프로토콜 1줄 교차 검증 누락?
- Q5-3. B분류 자동 승격 조건 위반 커밋 존재?
- Q5-4. β안-C 예외 범위(Step 6-2/6-4만) 준수?
- Q5-5. 백그라운드 탭 throttling 회피(navigate 재호출)가 gpt-send/gpt-read/gemini-send/gemini-read Step에 반영?
- Q5-6. GPT 실물 검증 공유 절차(SHA + `git show --stat`) 준수?

## 3. 영향 반경 매트릭스 (개선안 의무 첨부)

> 사용자 실증 지적: **"수정하면 다른 곳이 터진다."** 개선안을 낼 때 파급 범위를 명시하지 않으면 채택 금지.

### 3.1 5필드 (전부 필수)

| 필드 | 내용 |
|------|------|
| **R1. 수정 대상** | 파일:줄 또는 블록 |
| **R2. 직접 영향 (1차)** | 수정이 즉시 건드리는 서브시스템(S1~S7) + 구체 파일/훅/스킬 |
| **R3. 간접 영향 (2차)** | 1차 대상이 호출되는 곳. grep-reverse 근거 필수 |
| **R4. 유사 선례** | 동일/유사 변경으로 과거 발생한 incident·재커밋 (`incident_ledger` 태그 + 커밋 SHA), 없으면 "없음" |
| **R5. 회귀 위험도** | 상/중/하 + 1줄 근거. "상"이면 3자 토론 B분류 자동 승격 권고 |

### 3.2 R3 조사 방법 (유형별)

- **수정 대상이 훅**: settings에서 호출되는 event(PreToolUse/PostToolUse/Stop 등) 확인 + 동일 event에 등록된 다른 훅 나열
- **수정 대상이 스킬**: 해당 스킬을 호출하는 다른 스킬/command/agent 역탐색
- **수정 대상이 CLAUDE.md 규칙**: 같은 규칙이 다른 도메인 CLAUDE.md에 반복되는지 확인
- **수정 대상이 상태 원본(TASKS/STATUS/HANDOFF)**: 참조하는 훅(stop_guard, commit_gate, tasks_handoff 등) 나열

### 3.3 SPOF 강제 규칙

大 층에서 SPOF로 식별된 서브시스템을 수정하는 제안은:
- R5 최소 "중"
- 3자 토론 승격 기본값 "필요"

### 3.4 자동 강등

R1~R5 중 하나라도 누락된 제안은 "영향반경 미분석 — 보류"로 자동 강등해 출력한다.

## 5. 판정·작성 금지 규칙

- 설명만 듣고 PASS 금지
- 미커밋 로컬 상태로 PASS 금지
- 라벨 없는 지적 → 일반론 자동 강등
- R1~R5 없는 제안 → 보류 자동 강등
- 大 층 생략하고 바로 中/小로 내려가기 금지
- 서브시스템 7개 중 2개 이상 "기준 미확인"이면 §0 "전체 리포트 타입"을 `임시검토`로 상단 표기하고 최종 PASS 금지
- 자유 서술·감상·요약 문단 금지 (템플릿 섹션만 사용)
- 템플릿 섹션 생략 금지 (해당 없음이면 "해당 없음" 명기)

## 6. 사용자와의 인터랙션

사용자가 리포트를 보고 후속 지시를 내리면:

- **"P-N 채택"** → Claude에게 구현 지시가 넘어가도록 Claude 호출 블록 제안
- **"P-N 영향반경 보강"** → 해당 제안의 R3·R4를 더 깊게 재조사
- **"大만 다시"** → depth=대 재실행 (S1~S7 지형도만)
- **"토론 승격"** → Claude 측 `debate-mode` 스킬 호출 권고 문구 생성 (실제 호출은 Claude 담당)

GPT는 이 프로토콜 범위를 벗어나는 자체 실행·자체 수정 금지. 모든 변경·커밋·훅 조작은 Claude가 담당.
