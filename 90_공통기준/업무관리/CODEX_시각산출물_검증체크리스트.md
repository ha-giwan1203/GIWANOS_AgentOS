# CODEX 시각산출물 검증 체크리스트

> 목적: PDF, 이미지, HTML 대시보드, 인쇄물처럼 눈으로 확인해야 하는 산출물의 품질 누락을 줄인다.
> 비유: 엑셀 숫자는 계산기로 확인하고, 포스터나 보고서는 실제 인쇄 미리보기로 확인해야 한다.

## 1. 적용 대상
- PDF 보고서
- 이미지 시안
- HTML 대시보드
- 인쇄용 문서
- 엑셀/문서 파일을 PDF로 내보낸 결과

## 2. 저장 기준
- 임시 시안: `99_임시수집/`
- 업무 공식 산출물: 해당 도메인 폴더
- 파일명에는 날짜와 목적을 넣는다.
  - 예: `2026-05-22_라인정지비_요약보고서.pdf`
  - 예: `2026-05-22_프로모션이미지_시안01.png`
- 여러 시안은 `시안01`, `시안02`처럼 번호를 붙인다.
- 최종본은 `최종` 또는 `확정`을 파일명에 붙인다.

## 3. PDF 검증
- PDF가 열리는지 확인한다.
- 첫 페이지와 마지막 페이지를 확인한다.
- 표, 금액, 품번, 날짜가 잘리지 않는지 확인한다.
- 한글 깨짐, 빈 페이지, 겹침, 잘림이 없는지 확인한다.
- 엑셀/Word 원본에서 만든 PDF는 가능하면 렌더링 이미지 또는 Export 결과로 확인한다.

## 4. 이미지 검증
- 파일이 열리는지 확인한다.
- 이미지 크기와 비율을 확인한다.
- 글자가 잘리지 않는지 확인한다.
- 로고, 품번, 금액, 날짜 등 중요한 문구가 틀리지 않았는지 확인한다.
- 여러 장이면 선택 기준과 추천본을 HANDOFF에 적는다.

## 5. HTML 대시보드 검증
- 브라우저에서 열리는지 확인한다.
- 주요 카드, 표, 필터, 버튼이 보이는지 확인한다.
- 모바일/작은 창에서 텍스트가 겹치지 않는지 확인한다.
- 인쇄용이면 `Ctrl+P` 미리보기 기준으로 배경색과 테두리 표시를 확인한다.
