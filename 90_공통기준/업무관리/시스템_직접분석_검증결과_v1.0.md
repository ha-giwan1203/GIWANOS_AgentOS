검증 방식: PowerShell 기반 재조회 + 공개 API 응답 확인 + HTTP 헤더 확인
검증 상태: PARTIALLY VERIFIED

## 1. 목적

이 문서는 앞서 작성한 직접분석 설명서들이
`실제로 지금도 재현되는지`, `어디까지 검증됐는지`, `어디부터는 아직 화면 근거만 있는지`를
구분해서 남기기 위한 검증 기록이다.

이번 검증은 설명서를 다시 쓰는 작업이 아니라
2026-04-10 현재 시점에 시스템별 핵심 경로와 조회 결과를 다시 확인해
신뢰 가능한 범위와 미검증 범위를 명확히 표시하는 데 목적이 있다.

## 4. 시스템별 상세 결과

### 4.1 MES

이번 검증에서 셸 기준으로 아래 경로를 다시 호출했다.

| 확인 경로 | 결과 |
|-----------|------|
| `http://mes-dev.samsong.com:19200/layout/layout.do` | `302` -> `/oauth2/sso` |
| `http://mes-dev.samsong.com:19200/prdtstatus/selectPrdtRsltByLine.do?prdtDa=20260409` | `302` -> `/oauth2/sso` |
| `http://mes-dev.samsong.com:19200/prdtstatus/selectPrdtRsltLine.do?prdtDa=20260409` | `302` -> `/oauth2/sso` |

추가로 `C:\\Users\\User\\.flow-chrome-debug` 프로필의 Chrome 디버그 세션으로
MES 탭을 새로 열어 실제 브라우저 상태도 확인했다.

| 브라우저 검증 항목 | 결과 |
|-------------------|------|
| 열린 URL | `http://auth-dev.samsong.com:18100/login` |
| 페이지 타이틀 | `SAMSONG | OAuth` |
| 입력 필드 | `userId`, `password`, `idSaveYn` 확인 |
| 자동 복원 여부 | `미복원` |
| 입력값 길이 | `userId=0`, `password=0` |

판정:

1. 셸에서 세션 없이 접근하면 현재는 SSO 인증이 먼저 걸린다.
2. 로그인용 Chrome 프로필에서도 MES 세션이 자동 복원되지 않았고 입력값도 비어 있었다.
3. 따라서 이번 검증 패스에서는 `실제 데이터 재조회`까지는 확인하지 못했다.
4. 기존 `MES_직접분석_사용설명서_v1.0.md`는 브라우저 로그인 기반 화면 탐색 근거 문서로 유지한다.

검증 완료 항목:

- 접속 URL 유효성
- SSO 진입 경로
- 비인증 상태에서 직접 API 접근이 막혀 있다는 점

미검증 항목:

- 생산실적 실제 조회값 재현
- 라인별 생산실적 Excel Upload 저장 동작
- 작업자/공정/생산이력 화면의 검색 결과 값

ERP 탭을 새로 열어 실제 브라우저 상태도 확인했다.

| 브라우저 검증 항목 | 결과 |

1. ERP도 현재 셸 기준으로는 SSO 인증을 통과하지 않으면 내부 화면을 직접 재현할 수 없다.

3. 이번 패스에서는 `접속 경로가 살아 있는지`와 `대표 운영 화면이 SSO 뒤에 있는지`까지만 재확인했다.
4. 기존 `ERP_직접분석_사용설명서_v1.0.md`는 브라우저 로그인 기반 화면 스캔 근거 문서로 유지한다.

검증 완료 항목:

- 접속 URL 유효성
- 대표 운영 화면 URL 유효성
- SSO 리다이렉트 동작

미검증 항목:

- 라인배치 조회 결과
- 생산실적조회 화면 실제 수치

1. 브라우저 로그인 세션 기준으로 `MES 생산실적` 실제 값 재검증
2. 브라우저 로그인 세션 기준으로 `ERP 라인배치 / 생산실적조회 / 조립비마감(New)` 값 재검증
3. 위 1~2를 진행하려면 먼저 MES/ERP OAuth 로그인 세션을 수동으로 다시 열어야 함
4. 저장/수정/삭제 같은 쓰기 기능은 별도 테스트 계정 또는 테스트 데이터 기준으로 분리 검증

## 7. 결론

지금 상태를 한 줄로 정리하면 아래와 같다.

`ZDM`과 `조립비운영시스템`은 읽기 조회 기준으로 바로 믿고 써도 되는 수준까지 올라왔고,
`MES`와 `ERP`는 직접분석 설명서는 만들어졌지만 이번 셸 검증에서는 SSO 뒤 데이터까지 다시 찍지는 못했다.
