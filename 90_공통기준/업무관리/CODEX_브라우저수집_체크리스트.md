# CODEX 브라우저 수집 체크리스트

> 목적: Codex가 브라우저로 ERP/MES/QIS/웹 자료를 확인하거나 수집할 때, 조건과 증거를 빠뜨리지 않기 위한 기준.
> 비유: 화면 수집은 현장 사진 찍기와 같아서, 언제 어디서 무엇을 봤는지 같이 적어야 나중에 다시 믿고 쓸 수 있다.

## 1. 시작 전
- 작업지시서에 대상 시스템과 화면명을 적는다.
  - 예: G-ERP 라인보상상세현황, SmartMES 잡셋업, QIS 기타생산비용
- 조회 조건을 적는다.
  - 기간:
  - 라인:
  - 품번:
  - 상태값:
- 인증이 필요한 경우 기존 자동화 자산을 먼저 확인한다.
  - D0/ERP 계열: `90_공통기준/스킬/d0-production-plan/`
  - 라인정지비: `90_공통기준/스킬/line-stoppage/`

## 2. 수집 중
- 화면 진입 경로 또는 API/스크립트 호출 명령을 기록한다.
- 수집 기준 시각을 KST로 남긴다.
- 화면 데이터와 파일 데이터가 다르면 화면 기준인지 파일 기준인지 표시한다.
- 대량 출력은 원본 파일로 저장하고 보고에는 건수·합계·예외만 쓴다.

## 3. 저장 위치
- 임시 수집물: `99_임시수집/`
- 도메인 공식 산출물: 해당 도메인 폴더
- 자동화 로그: 해당 스킬의 `logs/` 또는 `state/`
- 원본 화면 덤프나 원본 JSON은 삭제하지 말고 `_raw` 또는 `_meta`로 구분한다.

## 4. 검증
- 최소 검증:
  - 조회 조건이 맞는지 확인
  - 총 건수 확인
  - 합계 금액 또는 총 수량 확인
  - 누락·중복·미승인·오류 상태 확인
- 화면 캡처가 필요한 작업이면 이미지 파일 경로를 HANDOFF에 적는다.
- API/스크립트 수집이면 실행 명령과 종료 결과를 HANDOFF에 적는다.

## 5. 금지
- 사용자에게 화면 캡처를 요구하지 않는다.
- 사용자에게 export/download를 요구하지 않는다.
- 인증 토큰, 쿠키, 비밀번호를 문서에 직접 남기지 않는다.
- 조회 조건을 추측해서 바꾸지 않는다.
