# ERP 직접분석 사용설명서 v1.0

작성일: 2026-04-10
분석 방식: ERP 직접 로그인 후 메뉴 구조 수집 + 생산관리 화면 스캔
분석 상태: DRAFT

## 1. 목적

이 문서는 AI가 실제 ERP 화면을 기준으로
`메뉴 구조`, `현재 업무와 직접 연결되는 생산관리 화면`, `검색조건`, `주요 기능`을 빠르게 찾을 수 있도록 만든 직접 분석형 사용설명서다.

이번 버전은 전체 ERP 메뉴맵을 실제로 수집했고,
그중 현재 저장소 업무와 가장 밀접한 `생산관리` 영역은 화면 단위로 직접 스캔해 정리했다.

## 4. 공통 화면 구조

직접 확인한 ERP의 공통 구조는 아래와 같다.

1. 상단 글로벌 메뉴에서 대도메인 진입
2. 실제 화면은 iframe 방식으로 로드
3. 대부분 화면은 `검색조건 -> 결과 그리드 -> 엑셀/저장/확정/적용 버튼` 구조
4. 공통 버튼은 `초기화`, `검색`, `도움말`, `엑셀`
5. 일정/월 기준 화면은 캘린더 헤더로 `일 월 화 수 목 금 토`가 반복 표시된다

MES보다 ERP 쪽이 더 강한 특징은 아래와 같다.

| 특징 | 설명 |
|------|------|
| 다단계 메뉴 | 2단계가 아니라 3~4단계 메뉴가 많다 |
| 운영 버튼 다양성 | `저장`, `확정`, `취소`, `적용`, `이관`, `생성`, `엑셀업로드`가 혼재 |
| 입력과 조회 혼합 | 단순 조회가 아니라 기준정보 수정, 계획 생성, 마감 처리 기능이 함께 존재 |

## 5. 현재 업무와 직접 연결되는 ERP 생산관리 구조

이번 스캔에서 확인한 `생산관리` 영역은 총 29개 화면이며,
현재 업무 기준으로는 아래 5개 묶음으로 이해하면 된다.

| 묶음 | 핵심 역할 |
|------|-----------|
| 기준정보 | 조립업체, 라인배치, 라인 적용 구조 관리 |
| 부하율관리 | 년간/월간 라인 운영 목표 관리 |
| 생산계획관리 | 월간/주간 계획, 생산서열, D0/D+1 지시 관리 |
| 재작업관리 | 재작업 계획/실적 관리 |
| 생산실적관리 | 생산실적, 라인가동, 출하지시 대비 실적 조회 |
| 생산비용관리 | 조립비 마감, 이월, 이의처리, 결재, 비율 관리 |

| 조립비마감(New) | `/assycostClose/viewListAssycostClose.do` | 마감월, 조립업체, 조립라인, 마감상태 | 초기화, 검색, 도움말, 마감생성, 마감확정, 마감취소, 엑셀 | 신규 조립비 마감 화면 |
| 조립비마감상세 | `/assycostClose/viewListAssycostCloseDtlAll.do` | 마감월, 조립업체, 조립라인, 품번 | 초기화, 검색, 도움말, 엑셀 | 조립비 마감 상세 조회 화면 |

## 7. 현재 업무 관점에서 가장 중요한 ERP 화면

업무리스트 저장소 기준으로 특히 중요한 화면은 아래다.

| 우선순위 | 화면 | 이유 |
|----------|------|------|
| 1 | 라인배치 관리 | 현재 자동화와 직접 연결 |
| 2 | 주간생산계획_OUTER_통합 | 생산계획 자동화와 연결 |
| 3 | 주간생산계획 MAIN&SUB 통합 | MAIN/SUB 계획 기준 화면 |
| 4 | 생산서열확정_OUTER / MAIN&SUB | 서열 확정과 D+1 지시 연결 |
| 5 | 생산실적조회(라인별) | 생산실적 분석과 직접 연결 |
| 6 | 조립비마감(New) / 조립비마감상세 | 정산 자동화와 직접 연결 |

## 8. AI 질문 라우팅 기준

AI가 ERP 관련 질문을 받으면 아래처럼 연결하면 된다.

| 사용자 질문 유형 | 먼저 볼 ERP 화면 |
|------------------|------------------|
| 라인배치 확인/수정 | `라인배치 조회`, `라인배치 관리` |
| 주간 생산계획 확인 | `주간생산계획_OUTER_통합`, `주간생산계획 MAIN&SUB 통합` |
| 생산서열 확인/확정 | `생산서열확정_OUTER`, `생산서열확정_MAIN&SUB` |
| 오늘 추가 생산지시 | `D0추가생산지시` |
| 내일 생산지시 | `D+1생산지시(임시)` |
| 재작업 계획/실적 | `재작업주간계획`, `재작업실적관리` |
| 업체/라인/제품 실적 조회 | `생산실적조회(업체별/라인별/제품별)` |
| 라인가동률/가동현황 | `생산라인가동조회(업체별/라인별/제품별)` |
| 출하지시 대비 생산 확인 | `출하지시대비 생산실적조회` |
| 조립비 마감/결재/이의처리 | 생산비용관리 7개 화면 |

## 9. 이번 직접분석으로 확인된 핵심 포인트

1. ERP는 MES보다 훨씬 크고, 현재 확인된 leaf 메뉴만 `392개`다.
2. 현재 업무 자동화와 직접 연결된 축은 `생산관리 > 기준정보 / 생산계획관리 / 생산실적관리 / 생산비용관리`다.
3. `라인배치 관리`는 ERP 안에서도 실무 입력형 운영 화면으로 기능이 강하다.
4. 생산계획 쪽은 `계획생성`, `계획확정`, `확정취소`, `엑셀업로드`처럼 실제 운영 액션 버튼이 많다.
5. 조립비 관련 화면은 기존 방식(`assycostCut`)과 신규 방식(`assycostClose`)이 공존한다.

## 11. 실무 메모

- ERP는 상단 메뉴가 매우 크므로, 전체를 다 외우기보다 `메뉴맵 JSON`과 `생산관리 직접분석 문서`를 함께 보는 편이 효율적이다.
- 현재 저장소 기준으로는 `생산관리` 영역을 먼저 정확히 아는 것이 ROI가 가장 높다.
- MES와 ERP를 함께 볼 때는 `ERP는 계획/배치/정산`, `MES는 실적/이력/모니터링` 축으로 이해하면 빠르다.
