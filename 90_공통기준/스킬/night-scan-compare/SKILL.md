# 야간 스캔실적 비교 스킬

## 목적
BI 생산실적과 MES 스캔실적을 일자별로 비교하여 차이를 자동 분석한다.
대상 라인/야간구분을 지정하면 BI에서 해당 날짜를 추출하고, MES API로 스캔실적을 조회한 뒤 비교 엑셀에 대입한다.

## 대상 파일

| 역할 | 경로 |
|------|------|
| BI 원본 | `05_생산실적/BI실적/대원테크_라인별 생산실적_BI.xlsx` |
| 비교 엑셀 | `05_생산실적/조립비정산/{MM}월/야간_생산_{라인명}.xlsx` |

## 비교 엑셀 시트 구조

| 시트 | 역할 |
|------|------|
| 생산기준DATA | BI 생산실적 (자동/붙여넣기) |
| 스캔실적DATA | MES 스캔실적 입력 대상 |
| 일자별비교 | 생산량 vs 스캔실적 수식 비교표 |
| 월간요약 | KPI + 차트 데이터 |

## 총 차이 산식 정의
- **총 차이** = SUM(생산량 - 스캔실적) where 스캔실적 ≠ 빈값
- 스캔미입력 날짜는 합계에서 자동 제외 (수식: `=IF(D="","",C-D)`)
- 예: 25일 중 스캔미입력 5일이면 총 차이는 실적 있는 20일 합산

## 실행 단계

### Phase 0: BI 최신본 갱신
1. Z드라이브 원본(`Z:\★ 라인별 생산실적\대원테크_라인별 생산실적_BI.xlsx`) 확인
2. 로컬 BI 파일과 수정일자 비교
3. Z드라이브가 최신이면 로컬로 복사 (production-result-upload 스킬 0단계 재사용)
4. Z드라이브 접근 불가 시 로컬 파일 사용 + 경고 출력

### Phase 1: 날짜 추출
1. BI 파일에서 대상 라인/야간/월 조건으로 실적 있는 날짜 목록 추출
2. openpyxl data_only=True로 생산량 함께 읽기

### Phase 2: MES 스캔실적 조회
1. CDP(`cdp_tabs.py`)로 MES 탭 확인 — 로그인 상태 검증
2. CDP(`cdp_exec.py`)로 iframe 내부 PqGrid API 엔드포인트 확인
3. MES API 직접 호출하여 전체 날짜 일괄 조회
   - **엔드포인트:** `/prdtstatus/selectPrdtRsltLine.do?prdtDa={YYYYMMDD}`
   - **응답 구조:** `json.data.list[]` → `lineCd`, `shiftsCd`(02=야간), `prdtQty`
4. 대상 라인 + 야간(shiftsCd=02) 필터링
5. 데이터 없는 날짜 식별 → 빈값 처리

### Phase 3: 엑셀 대입
1. 비교 엑셀을 openpyxl로 열기 (data_only=False, 수식 보존)
2. **생산기준DATA 시트:**
   - 기존 데이터 전체 클리어 (이전 달 잔존 방지)
   - BI에서 추출한 날짜별 데이터 재대입 (A~K열)
   - 재실행 시 고아데이터 방지를 위해 반드시 클리어 후 입력
3. **스캔실적DATA 시트:**
   - 기존 데이터 클리어
   - 날짜(A), 라인(B), 스캔실적(C), 비고(D) 입력
   - MES 데이터 없는 날짜: C열 빈값, D열 "MES데이터없음"
3. **일자별비교 시트:**
   - 날짜(A), 라인(B) 입력
   - C~H열 수식 행별 생성 (SUMIFS 참조 범위를 실제 데이터 행 수에 맞춤)
   - 상단 합계 수식(Row 3) 범위 갱신
4. **월간요약 시트:**
   - 날짜(F), 수식(G~I) 갱신
   - KPI 수식은 일자별비교 합계 참조이므로 자동 유효

### Phase 4: 양식 통일
1. **테이블 객체 범위 확장:** 데이터 행 수에 맞춰 `ref` 갱신
   - tbl_scan, tbl_compare 모두 처리
2. **개별 셀 fill 제거:** `PatternFill(fill_type=None)` 적용 → 테이블 스타일 자동 색상에 위임
3. **조건부 서식 범위 확장:** 기존 규칙 백업 → 삭제 → 새 범위로 재적용
   - 불일치: `FFFCE4D6` (주황)
   - 일치: `FFE2EFDA` (초록)
   - 스캔미입력: `FFF2F2F2` (회색)
4. **셀 서식 통일:** 기준 행의 font/alignment/border/number_format 복사

### Phase 5: 검증
1. 테이블 범위 일치 확인
2. 개별 fill 없음(None) 확인
3. 조건부 서식 범위 = 데이터 행 전체 확인
4. MES없음 날짜 빈값 확인
5. 월간요약 행 수/수식 확인
6. 고아데이터 없음 확인

## 핵심 규칙

- **CDP 기본 사용:** 브라우저 작업은 CDP 래퍼(`.claude/scripts/cdp/`) 우선. Chrome MCP는 토론모드에서만.
- **MES 로그인 필수:** CDP로 탭 목록 확인하여 MES 세션 유효 여부 먼저 검증. 로그인 안 되어있으면 사용자에게 요청.
- **API 직접 호출:** UI 조작 대신 `fetch()` + iframe contentWindow 통해 MES API 직접 호출.
- **수식 보존:** openpyxl data_only=False로 열어 기존 수식 파괴 방지.
- **양식 3종 세트:** 테이블 범위 + fill 제거 + 조건부 서식 범위를 반드시 함께 처리.
- **빈값 vs 0:** MES 데이터 없는 날짜는 0이 아닌 빈값(None)으로 처리하여 "스캔미입력" 상태 표시.
