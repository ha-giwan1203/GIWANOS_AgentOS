# YouTube 영상 분석 (youtube-analysis)

## 모드 구분

| 입력 | 모드 | 동작 |
|------|------|------|
| "영상분석" 단독 / "영상분석 {주제}" | **자동 모드** | 상태 진단 → 검색 → 분석 → 교차검증 → GPT 토론 → 구현 |
| "유튜브 분석", "영상 요약" + URL | **수동 모드** | 해당 영상 자막 추출 → 분석 → 적용안 출력 |

> "영상 분석해줘" 같은 일반 표현은 URL 없으면 자동 모드, URL 있으면 수동 모드.
> "유튜브 분석", "영상 요약"은 URL 필수 — 없으면 URL 요청.

---

## 자동 모드 (URL 없이 "영상분석"만 입력)

### 전체 흐름

```
Step 0    현재 상태 로드 + 중복 검색 방지
Step 0.5  실행 가능성 점검
Step 1    개선 가설 수립
Step 2    YouTube 검색 + 후보 수집
Step 3    자막 추출 + 영상별 분석
Step 4    후보 추출 · 교차검증 · A/B/C 판정
Step 5    GPT 공동작업 토론 (A/B 등급만)
Step 6    구현 또는 plan.md 생성
Step 7    상태 문서 갱신 + 결과 보고
```

### Step 0 — 현재 상태 로드

아래 파일을 읽어 현재 시스템 상태를 파악한다.

| 대상 | 경로 | 확인 내용 |
|------|------|----------|
| 작업 상태 | `90_공통기준/업무관리/TASKS.md` | 진행중/대기중 + **완료 이력 (중복 검색 방지)** |
| 운영 현황 | `90_공통기준/업무관리/STATUS.md` | 파이프라인 상태, 주의사항 |
| 인수인계 | `90_공통기준/업무관리/HANDOFF.md` | 미해결 항목, 발견사항 |
| hooks 설정 | `.claude/settings.local.json` | 현재 훅 구성 |
| rules | `.claude/rules/*.md` | 현재 규칙 체계 |
| 스킬 목록 | `90_공통기준/스킬/` 하위 | 보유 스킬 현황 |
| CLAUDE.md | 루트 + 도메인 | 운영 지침 구조 |
| 이전 분석 로그 | `logs/debate_*.md` | 기존 분석 이력 (있으면) |

### Step 0.5 — 실행 가능성 점검

자동 모드 전체 루프 실행 가능 여부를 확인한다.

| 점검 항목 | 불가 시 대응 |
|----------|------------|
| youtube_transcript.py + youtube-transcript-api | 설치 안내 후 분석 전용 모드 강등 |
| WebSearch 사용 가능 여부 | 불가 시 수동 URL 입력 요청 |
| 토론모드 (ChatGPT 탭 + Chrome MCP) | 불가 시 분석 전용 모드 강등 (GPT 토론 스킵) |

**분석 전용 모드**: Step 0~4까지만 실행, Step 5~6 스킵, Step 7에서 분석 결과만 보고

### Step 1 — 개선 가설 수립

상태 진단 결과에서 아래 관점으로 갭을 식별한다.

| 관점 | 예시 |
|------|------|
| Claude Code 운영 안정성 | hooks 누락, 규칙 미비, 에러 패턴 |
| 자동화 수준 | 수동 반복 작업, 파이프라인 미연결 |
| 스킬 커버리지 | 미보유 스킬, 기존 스킬 개선 여지 |
| MCP/도구 활용 | 미사용 커넥터, 연동 갭 |
| 프롬프트/워크플로우 | 비효율 패턴, 반복 지시 |

출력: **문제 / 찾을 패턴 / 우리 환경 적용 지점** 3줄 요약 (최대 5건)

### Step 2 — YouTube 검색 + 후보 수집

Step 1 가설을 키워드로 변환하여 WebSearch로 영상 탐색.

검색 전략:
- 영어 검색 우선 (영어 콘텐츠가 양과 질에서 유리)
- `Claude Code` + 개선 키워드 조합
- 검색 기간: **주제별 가변** (Claude Code/hooks/MCP → 3~6개월, 범용 도구 → 12개월)
- 구현 사례/데모 포함 영상 우선, 이론만 있는 영상 후순위

선정 기준: **최소 2개, 최대 5개** 영상

### Step 3 — 자막 추출 + 영상별 분석

선정된 영상마다 자막 추출:

```bash
PYTHONUTF8=1 python "90_공통기준/스킬/youtube-analysis/youtube_transcript.py" "<URL>"
```

자막 추출 실패 시: WebSearch로 영상 요약 블로그/리뷰 검색하여 대체

### Step 4 — 후보 추출 · 교차검증 · A/B/C 판정

> **GPT 합의 (2026-04-01): 이 단계 내부 순서는 반드시 아래를 따른다.**
> A/B/C 판정은 교차검증 뒤에만 가능하다.

```
4-1. 영상 핵심 주장 추출
4-2. 우리 환경 적용 후보 생성
4-3. 공식 문서 교차검증 (Claude Code docs / API docs / 도구별 공식 문서)
4-4. 현재 저장소 · 상태 문서 · 실행 가능성 대조
4-5. A/B/C 판정
4-6. 근거 패키지 저장
```

#### A/B/C 판정 기준

| 등급 | 조건 | 후속 |
|------|------|------|
| **A** (즉시 구현) | 작고 명확한 변경 + 공식 문서 확인됨 + 되돌리기 쉬움 | Step 6에서 바로 구현 |
| **B** (계획 후 구현) | 복수 파일/도메인 영향 + 공식 문서 확인됨 + plan 필요 | Step 6에서 plan.md 작성 |
| **C** (참고만) | 환경 미스매치 / 이미 보유 / 증거 불충분 / 되돌리기 어려움 | 로그 저장만, 구현 안 함 |

#### 분석 관점 (9개 — 매번 지시 불요)

| # | 관점 | 설명 |
|---|------|------|
| 1 | Claude Code 운영 | hooks, rules, settings, agents, commands, CLAUDE.md 구조 |
| 2 | 업무 자동화 | ERP 라인배치, 조립비 정산, 생산관리, MES 업로드 |
| 3 | MCP/커넥터 | Notion, Slack, Gmail, Google Calendar, Chrome 연동 |
| 4 | 스킬/워크플로우 | 스킬 체계, 프롬프트, 자동화 파이프라인 |
| 5 | GPT 공동작업 | 토론모드, 검증 체계, 하네스 방식 |
| 6 | **재현 조건** | 우리 환경(Windows + Claude Code + 삼송 ERP)에서 바로 재현 가능한지 |
| 7 | **운영 리스크** | 토큰 낭비, 권한 남용, 유지보수 부담, DOM 취약성 |
| 8 | **증거 품질** | 구현 화면 있는지, 말만인지, 공식 문서와 일치하는지 |
| 9 | **되돌리기 난이도** | 실패 시 쉽게 끌 수 있는지, 부작용 범위 |

### Step 5 — GPT 공동작업 토론 (A/B 등급만)

> 토론모드 CLAUDE.md 규칙을 따른다. C등급은 토론 대상이 아니다.

Step 4 결과 중 A/B 등급만 GPT 대화방에 공유하고 토론.

토론 목표:
- A/B 판정에 대한 GPT 관점 검증
- 우선순위 합의
- 구현 범위 확정

### Step 6 — 구현 또는 plan.md 생성

| 등급 | 동작 |
|------|------|
| A등급 합의 완료 | 즉시 구현 → GPT에 커밋 SHA + 실물 증거 공유 → 검증 PASS 루프 |
| B등급 합의 완료 | plan.md 작성 → 사용자 승인 대기 |

### Step 7 — 상태 문서 갱신 + 결과 보고

**상태 갱신** (선택이 아니라 종료 조건):
- TASKS.md: 완료 이력 추가
- HANDOFF.md: 세션 변경사항 기록
- STATUS.md: 갱신

**사용자 보고** (1회):


## 수동 모드 (URL 직접 제공)

### 트리거

- 메시지에 `youtu.be/`, `youtube.com/watch` URL 포함
- "유튜브 분석", "영상 요약", "이 영상 분석" + URL
- `/video <URL>` 슬래시 커맨드

### 실행 절차

#### Step 1 — 영상 다운로드 + 프레임 추출 + 자막 추출

```bash
PYTHONUTF8=1 python "90_공통기준/스킬/youtube-analysis/youtube_analyze.py" "<URL>" --max-frames 15
```

파이프라인이 자동 수행하는 작업:
1. 자막 추출 (youtube_transcript.py, 한국어 우선)
2. 영상 다운로드 (yt-dlp, 480p)
3. 프레임 추출 (ffmpeg, 챕터 기준 or 30초 간격)
4. manifest.json 생성

출력 위치: `90_공통기준/스킬/youtube-analysis/cache/<video_id>/`

자막만 필요한 경우: `--no-download` 플래그 추가

#### Step 2 — 프레임 + 자막 통합 분석

1. `manifest.json` Read → 영상 메타 정보 확인
2. `transcript.txt` Read → 전체 내용 파악
3. `frames/` 폴더의 이미지를 **Read 도구로 직접 열어** 시각 정보 확인
   - 코드 화면, UI 데모, 설정 화면, 터미널 출력 등
   - 프레임별로 자막 타임스탬프와 교차 매핑
4. 9개 관점으로 분석 + 공식 문서 교차검증 + A/B/C 판정

> **프레임 분석이 핵심 차별점**: 자막만으로는 코드 내용, UI 구조, 설정 화면 등을 파악할 수 없음

#### Step 3 — 출력

```
## 영상 정보
- 제목: [manifest.json 기반]
- 채널: [채널명]
- 길이: [N분 N초]
- 분석 프레임: [N장]
- 자막 언어: [언어 정보]

## 요약
[3~5줄 핵심]

## 주요 화면 분석
| # | 시점 | 화면 내용 | 자막 맥락 | 시사점 |
|---|------|----------|----------|--------|

## 적용 판정
| # | 주제 | 내용 | 등급 | 공식 문서 확인 | 되돌리기 | 근거 |
|---|------|------|------|--------------|---------|------|

## 다음 액션
[A등급: 구현 제안 / B등급: plan 필요 / C등급: 참고만]
```

## 스크립트

| 스크립트 | 용도 | 의존성 |
|---------|------|--------|
| `youtube_transcript.py` | 자막만 추출 | `youtube-transcript-api` |
| `youtube_analyze.py` | **전체 파이프라인** (다운로드+프레임+자막) | `yt-dlp`, `ffmpeg`, `youtube-transcript-api` |

설치: `pip install youtube-transcript-api yt-dlp` + `winget install Gyan.FFmpeg`

## 오류 처리

| 오류 | 대응 |
|------|------|
| 자막 비활성화/없음 | WebSearch로 영상 요약 블로그/리뷰 검색하여 대체 |
| yt-dlp 다운로드 실패 | `--no-download` 모드로 자막만 분석 |
| ffmpeg 미설치 | PATH 자동 탐지 실패 시 설치 안내 |
| URL 오류 | 검색 결과에서 재추출 |
| 검색 결과 부족 | 키워드 변형 재검색 (최대 2회) |
| GPT 대화방 진입 실패 | 토론모드 CLAUDE.md 셀렉터 규칙 따름 |
| 토론모드 불가 | 분석 전용 모드 강등 (Step 5~6 스킵) |

## 주의사항

- 자동 모드에서 중간 승인 불요 (비가역 작업 제외)
- 영어 영상은 영어 자막 추출 후 한국어로 번역하여 분석
- 토론모드 연동 시 토론모드 CLAUDE.md 규칙 준수
- **영상은 아이디어 소스. 최종 기준은 공식 문서 + 현재 저장소 상태**
- A/B/C 판정은 반드시 교차검증 후에만 가능
