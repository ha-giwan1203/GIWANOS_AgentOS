# 월정산 손익 통합팩 (rollup)

> 도메인 규칙: `05_생산실적/조립비정산/CLAUDE.md`
> 본체 파일: `05_생산실적/조립비정산/{MM+1}월/정산_수식버전_MM월.xlsx` 본체 (`정산결과_MM월.xlsx` 는 산출용 임시본·교차대조)

## 절차 (요약)
1. 본체 정산_수식버전 존재 확인 → 백업 (`.bak`)
2. 6시트 빌드 (90~95) 후 본체에 append (기존 시트 보존)
3. 검증: B 산식 수치 = 시트 91 합계 = 시트 90 KPI

## 호출
```bash
python 90_공통기준/스킬/monthly-pnl-rollup/run.py --month 04
```

## 2시트 구조 (통합)

| 시트 | 섹션 | 데이터 소스 |
|------|------|----------|
| **90_월정산_요약** | ① 최종손익 KPI (A·B·C·D) ② 라인별 손익 ③ 검증·이슈 ④ 보고 문장 템플릿 | 본체 정산집계 + 91 cross |
| **91_정산상세** | A. 야간 BI 정합 / B. 기타비용 / C. 라인지원 (raw 근거) | BI 원본 + 라인정지 raw + 지원자료 |

> 이전 6시트(90~95) → 2시트로 통합 (2026-05-14, 사용자 시트 통합 지시). 각 섹션은 ■ 또는 A./B./C. 헤더로 구분.

## 최종 산식

```
A. 기본조립비    = GERP 정산금액 (라인 합산)
B. 지원순액      = 준지원금 - 받은지원금  (94 시트, 폴더 부재 시 0)
C. 비용보전      = 인정된 (정지 + 재작업 + 선별 + 기타)
D. 차감비용      = 미인정 청구 + 회사 부담분 (1차 안: 미인정만)
────────────────────────────────────
최종 정산금액    = A + B + C − D
```

## 부호 자동 판정 (94_라인지원)

| 토큰 (파일명·시트명) | 부호 | 의미 |
|---------------------|------|------|
| "지원받은", "이관", "← " | **−** | 우리가 지급할 금액 (받은지원금) |
| "지원해줌", "→ ", "대원테크 →" | **+** | 우리가 받을 금액 (준지원금) |
| 명확 토큰 없음 | log warning + 0 처리 | 사용자 수동 분류 필요 |

## verify

- 91 합계 = 90 KPI "최종 정산금액"
- 92 GERP야간 = 본체 라인시트 야간수량 합
- 93 합계 = `라인정지_MM월_raw.xlsx` 합계와 일치
- 94 폴더 부재 시 cells B5~ = 0 + "당월 지원 없음" 표시

## 실패 시

- 본체 정산_수식버전 미존재 → 중단 + `/settlement MM` 선행 안내
- 라인정지_MM월_raw.xlsx 미존재 → 93 빈 시트 + warning (line-stoppage 스킬 선행 안내)
- BI/MES 접근 불가 → 92 빈 시트 + warning (운영 계속)
