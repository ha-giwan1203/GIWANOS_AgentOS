# 품번-라인배정 정합성 검증

## 개요

기준 품번 파일(마스터)과 라인배정 파일(검증 대상)을 비교하여,
품번 단위로 정합성을 판정하고 실무자가 바로 검토할 수 있는 보고서를 만든다.

이 스킬의 핵심 가치:
- 수동 대조 시 2~3시간 걸리는 품번 검증을 10분 이내로 줄인다
- "누락인지 오기입인지 컬러확장인지"를 사람 대신 1차 판정해준다
- 판정 근거를 제시해서 실무자가 최종 확인만 하면 된다

| 라인배정 파일 | 검증 대상 (xlsx/csv) | ENDPART라인배정.xlsx (1,476건) |

| 시트명 | 대상 시트 | 자동 탐색: 데이터가 가장 많은 시트 선택 |
| 품번 컬럼 | 품번이 들어있는 컬럼 | 헤더에서 "품번", "제품번호", "PART" 등 탐색 |
| 라인코드 컬럼 | 라인코드 컬럼 | "조립라인", "라인", "LINE" 등 탐색 |

| 컬러코드 규칙 | 품번에서 컬러코드 분리법 | 뒤 2~3자리, 사용자 확인 후 적용 |

### 파일 구조 주의사항

이 업무에서 다루는 엑셀 파일들은 대부분 아래 특성이 있다:
- **2행 헤더**: 1행이 상위 카테고리, 2행이 실제 컬럼명인 경우가 많다
- **병합셀**: 헤더에 병합셀이 있어서 openpyxl로 읽을 때 None이 나올 수 있다
- **시트명이 공백이거나 특수문자**: `ws = wb.worksheets[0]`처럼 인덱스로 접근이 안전하다
- **빈 행 섞임**: 중간에 빈 행이 있을 수 있으므로 None 체크 필수

## 품번 비교 원칙

GPT가 설계한 핵심 원칙을 그대로 채택하되, 실행 방법을 구체화한다.

### 1. 기준 파일이 마스터다
기준 파일의 품번을 정답 목록으로 본다. 라인배정 파일은 확인 대상이다.
라인배정에만 있고 기준에 없는 품번은 별도 "기준 미등록" 목록으로 분리한다.

### 2. 접두어는 건드리지 않는다
접두어(예: SP3M3-)는 품번의 일부다. 접두어가 다르면 다른 품번이다.
"접두어 빼면 맞을 것 같은데" 싶어도 임의로 제거하지 않는다.

### 3. 컬러코드는 2단계로 비교한다
자동차 부품 품번은 보통 [모품번]+[컬러코드] 구조다.
기준 파일은 모품번만, 라인배정 파일은 컬러코드까지 붙어있는 경우가 많다.

```
1차: 전체 품번 직접 비교 → 완전 일치 먼저 잡기
2차: 모품번 기준 확장 비교 → 기준 모품번이 라인배정 품번의 앞부분과 일치하는지
```

컬러코드 분리 규칙이 불명확하면 **임의로 잘라내지 말고** 사용자에게 확인:
"뒤 2자리를 컬러코드로 보고 분리했습니다. 맞습니까?"

### 4. 품번이 있어도 라인이 없으면 문제다
품번 자체는 존재하지만 해당 라인코드가 배정되지 않은 경우 →
"완전누락"이 아니라 "라인누락"으로 분리. 실무적으로 조치가 다르다.

### 5. 애매하면 확정하지 않는다
아래 경우는 바로 누락 확정하지 않는다:
- 유사 품번이 존재 (숫자 1~2자리 차이)
- 모품번은 같은데 컬러코드만 다름
- 품번 표기 규칙(하이픈, 공백, 대소문자)만 다름

→ "판정유보" 또는 "기준오기입의심"으로 분리하고 사유를 적는다.
실무자가 최종 판단할 수 있도록 근거를 제공하는 게 이 스킬의 역할이다.

## 판정 코드

| 코드 | 판정 | 의미 | 실무 조치 |
|------|------|------|-----------|
| A | 일치 | 품번·라인코드 모두 일치 | 없음 |
| B | 컬러확장일치 | 모품번 일치 + 컬러코드 확장 존재 + 라인 일치 | 없음 (정상) |
| C | 라인누락 | 품번/모품번은 존재하나 대상 라인코드 없음 | 라인배정 추가 필요 |
| D | 기준오기입의심 | 유사품번 존재, 오타/자릿수 차이 추정 | 기준파일 확인 필요 |
| E | 완전누락 | 직접·모품번·유사 모두 불일치 | 라인배정 신규 등록 필요 |
| F | 판정유보 | 자료 부족/컬럼 불명확/규칙 모호 | 추가 확인 필요 |

## 실행 절차

### Step 1. 파일 구조 파악

openpyxl로 두 파일을 열어 구조를 확인한다.

```python
import openpyxl

def scan_file(path):
    wb = openpyxl.load_workbook(path, data_only=True)
    for i, name in enumerate(wb.sheetnames):
        ws = wb.worksheets[i]
        print(f"시트 [{i}] '{name}': {ws.max_row}행 x {ws.max_column}열")
        # 첫 3행 출력 (헤더 파악)
        for row in ws.iter_rows(min_row=1, max_row=3, values_only=True):
            print([str(v)[:20] if v else '(빈칸)' for v in row])
```

**사용자에게 확인받을 것:**
- "기준 파일의 품번 컬럼은 B열('품번'), 라인배정 파일의 품번은 D열('제품번호'), 라인코드는 G열('조립라인')로 보입니다. 맞습니까?"
- 확인 없이 진행하지 않는다.

### Step 2. 품번 정규화

```python
def normalize(part_no):
    """품번 정규화: 공백제거, 대문자, 접두어 유지"""
    if part_no is None:
        return ""
    s = str(part_no).strip().upper()
    s = " ".join(s.split())  # 연속 공백 → 단일 공백
    return s

def extract_base(part_no, color_len=0):
    """모품번 추출. color_len=0이면 전체 반환 (컬러코드 규칙 미확정)"""
    if color_len > 0 and len(part_no) > color_len:
        return part_no[:-color_len]
    return part_no
```

**주의**: `extract_base`의 `color_len`은 사용자가 확인한 후에만 0이 아닌 값을 사용한다.

### Step 3. 1차 직접 비교

기준 품번 전체를 라인배정 품번과 정규화 후 직접 비교한다.

```python
# 기준 품번 set
ref_parts = {normalize(row[품번열]) for row in 기준시트 if row[품번열]}

# 라인배정 품번 → {품번: [라인코드 목록]}
assign_map = {}
for row in 배정시트:
    pn = normalize(row[품번열])
    line = normalize(row[라인열])
    if pn:
        assign_map.setdefault(pn, set()).add(line)

# 1차 판정
for ref_pn in ref_parts:
    if ref_pn in assign_map:
        if 대상라인 in assign_map[ref_pn]:
            판정 = "A"  # 일치
        else:
            판정 = "C"  # 라인누락
    else:
        # 2차 비교로 넘긴다
        pass
```

### Step 4. 2차 모품번 비교

1차에서 불일치한 품번에 대해 모품번 기준으로 재비교.

```python
# 라인배정 품번의 모품번 → {모품번: [전체품번 목록]}
base_map = {}
for pn in assign_map:
    base = extract_base(pn, color_len)
    base_map.setdefault(base, []).append(pn)

for ref_pn in 불일치목록:
    ref_base = extract_base(ref_pn, color_len)
    if ref_base in base_map:
        # 컬러확장 존재
        expanded = base_map[ref_base]
        lines = set()
        for epn in expanded:
            lines.update(assign_map.get(epn, set()))
        if 대상라인 in lines:
            판정 = "B"  # 컬러확장일치
        else:
            판정 = "C"  # 라인누락
    else:
        # 3차(유사품번)로 넘긴다
        pass
```

### Step 5. 유사품번 탐색

2차까지 불일치한 품번에 대해 유사도를 계산한다.

```python
from difflib import SequenceMatcher

def find_similar(target, candidates, threshold=0.85):
    """유사품번 찾기. threshold 이상이면 후보로 반환"""
    results = []
    for c in candidates:
        ratio = SequenceMatcher(None, target, c).ratio()
        if ratio >= threshold:
            results.append((c, ratio))
    return sorted(results, key=lambda x: -x[1])

for ref_pn in 여전히_불일치:
    similar = find_similar(ref_pn, assign_map.keys())
    if similar:
        판정 = "D"  # 기준오기입의심
        사유 = f"유사품번 발견: {similar[0][0]} (유사도 {similar[0][1]:.0%})"
    else:
        판정 = "E"  # 완전누락
```

모든 기준 품번에 A~F 판정을 확정하고, 출력 형식에 따라 결과를 정리한다.

## 검증 기준

아래를 모두 만족해야 완료로 본다:
- 기준 파일 품번 총 건수가 명시되어 있다
- 판정별 건수 합계 = 기준 품번 총 건수 (빠짐 없이 전수 처리)
- 모든 C·D·E·F 항목에 판정사유와 조치의견이 있다
- 접두어를 임의로 제거하지 않았다
- 컬러코드를 사용자 확인 없이 제거하지 않았다
- 사용자가 확인하지 않은 가정은 "⚠️ 가정" 표시가 되어 있다
