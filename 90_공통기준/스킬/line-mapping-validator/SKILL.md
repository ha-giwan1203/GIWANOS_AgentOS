# 품번-라인배정 정합성 검증

> 6 Step 절차 / 비교 원칙 / 판정 코드 / 출력 형식은 [MANUAL.md](MANUAL.md). 용어는 ../GLOSSARY.json.
> 라인배치 작업(line-batch-management) 후 결과 검증 시도 사용.

## 입력
- 기준 파일 (마스터 품번 xlsx/csv)
- 라인배정 파일 (검증 대상 xlsx/csv)
- 시트/컬럼/대상라인 (자동 탐색 + 사용자 확인)

## 절차 (요약)
1. 파일 구조 파악 (openpyxl) + 사용자 컬럼 확인
2. 품번 정규화 (공백/대문자/접두어 유지)
3. 1차 직접 비교 → A 일치 / C 라인누락
4. 2차 모품번 비교 → B 컬러확장일치 / C 라인누락
5. 유사품번 탐색 (SequenceMatcher ≥ 0.85) → D 오기입의심 / E 완전누락
6. 판정 확정 + 결과 정리 (요약/상세표/개선안/3줄)

## 판정 코드
| 코드 | 판정 | 조치 |
|------|------|------|
| A | 일치 | 없음 |
| B | 컬러확장일치 | 없음 |
| C | 라인누락 | 라인배정 추가 |
| D | 기준오기입의심 | 기준파일 확인 |
| E | 완전누락 | 신규 등록 |
| F | 판정유보 | 추가 확인 |

## verify
- 기준 품번 총 건수 명시
- 판정별 합계 = 기준 총 건수 (전수 처리)
- C·D·E·F 모두에 판정사유 + 조치의견
- 접두어 미제거 / 컬러코드 보존
- 사용자 미확인 가정에 "⚠️ 가정"

## 금지
- 접두어 임의 제거 / 컬러코드 사용자 확인 없이 제거
- 라인배정을 마스터로 삼기
- 유사품번 즉시 "완전누락" 확정
- 원본 파일 직접 수정
- 컬럼 사용자 확인 없이 임의 결정

## 실패 시
- 헤더 인식 실패 → 첫 5행 출력 후 컬럼 지정 요청
- A+B = 0건 → 정규화/컬럼 오류 가능
- 상세 → MANUAL.md "오류 대응" + "되돌리기"
