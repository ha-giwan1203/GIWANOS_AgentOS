# supanova-deploy eval cases

## 목적
이 문서는 `supanova-deploy` 스킬이 실제 사용 중 자주 발생하는 실패 상황을 안정적으로 감지하고,
누락 없이 보완 동작을 수행하는지 점검하기 위한 평가 케이스 모음이다.

평가 기준:
- 실패 상황을 정확히 감지하는가
- 누락된 전제조건을 보완하거나 명확히 경고하는가
- 산출물 기준을 흐리지 않는가
- 완료 판정을 너무 쉽게 내리지 않는가

---

## CASE 01. WebP 변환 문구 누락

### 입력 예시
영상을 히어로 섹션에 넣고 스크롤 애니메이션 적용해줘.

### 기대 동작
- WebP 프레임 시퀀스 변환 조건을 자동으로 보강하거나 누락으로 판정한다.
- "WebP 변환" 문구 없이 진행하면 스크롤 애니메이션이 깨질 수 있음을 경고한다.
- 권장 지시문 형태로 보완 후 진행한다.

### 기대 출력 포인트
- `WebP 프레임 시퀀스 변환 필요`
- `frames/ 폴더에 저장`
- `부드러운 애니메이션 조건 포함`

### 실패 판정
- WebP 변환 없이 원본 영상 파일 그대로 사용
- frames 폴더 미생성
- 스크롤 애니메이션 깨짐 경고 없음

## CASE 02. frames 폴더 누락

### 입력 예시
배포 전인데 index.html이랑 assets 폴더만 있어.

### 기대 동작
- `frames/` 폴더 누락을 즉시 실패 판정한다.
- 배포 전 검증 단계에서 FAIL 처리한다.
- frames 폴더 없이는 완료로 보지 않는다.

### 기대 출력 포인트
- `FAIL: frames/ 폴더 누락`
- `배포 불가 판정`
- `WebP 프레임 변환 재실행 필요`

### 실패 판정
- frames 폴더 없이 배포 가능하다고 안내
- 산출물 구조 기준 확인 없이 진행
- 완료로 판정

## CASE 03. ZIP으로 Netlify 업로드 시도

### 입력 예시
ZIP 파일로 압축했는데 Netlify Drop에 바로 올리면 되지?

### 기대 동작
- Netlify Drop 업로드는 ZIP이 아니라 **output 폴더 기준**임을 안내한다.
- ZIP은 내부 보관/전달용으로 분리해서 설명한다.
- 올바른 업로드 방법을 제시한다.

### 기대 출력 포인트
- `ZIP = 보관용`
- `Netlify Drop = output 폴더 기준`
- `배포 산출물은 폴더 구조`

### 실패 판정
- ZIP과 배포 폴더를 구분하지 않음
- ZIP 그대로 업로드하라고 안내
- 산출물 구조 설명 없음

## CASE 04. 초기 진입 실패 - Gemini 비디오 생성 모드 미진입

### 입력 예시
Gemini에 그냥 영상 만들어달라고 쳤는데 안 되네?

### 기대 동작
- 일반 채팅 요청이 아니라 **비디오 생성 모드 진입**이 먼저 필요하다고 안내한다.
- 현재 상태를 "영상 생성 전 단계"로 판정한다.
- 다음 실행 순서를 짧고 명확하게 제시한다.
- Pro 모델 확인이 필요하면 함께 언급한다.

### 기대 출력 포인트
- `일반 채팅 아님`
- `비디오 생성 모드 진입 필요`
- `영상 생성 전 단계`
- `Pro 모델 확인`

### 실패 판정
- 그냥 다시 요청해보라고만 함
- 비디오 생성 모드 진입 필요성을 설명하지 않음
- 현재 막힌 원인을 잘못 진단함

## CASE 05. 경로 실패 - 상대경로/절대경로 깨짐

### 입력 예시
로컬에서는 되는데 배포 후 프레임이 안 보여.

### 기대 동작
- `frames/` 존재 여부만 보지 않고 **경로 참조 방식**을 확인한다.
- `index.html` 기준 상대경로인지 점검한다.
- 절대경로, 로컬 전용 경로, 잘못된 assets 경로를 실패 원인 후보로 제시한다.
- 파일 존재와 경로 참조를 분리해서 판정한다.

### 기대 출력 포인트
- `상대경로 점검`
- `절대경로/로컬경로 사용 여부 확인`
- `index.html 기준 경로 확인`
- `파일 존재 != 경로 정상`

### 실패 판정
- frames 폴더만 있으면 정상이라고 판단
- 경로 문제를 점검하지 않음
- 배포 후 원인 미진단

## 한 줄 판정 원칙
좋아 보이는 결과물이 아니라, **중간 자산이 빠지지 않은 재배포 가능한 폴더 구조**가 남아야 통과다.
