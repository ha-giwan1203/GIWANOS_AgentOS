# SP3 생산계획 운영절차 v3.0

## 1. 목적
이 문서는 **실행 순서**만 정리한다.

---

## 2. 일일 운영 순서

### 2.1 ERP 계획 반영
1. Sheet2 `OUTER 생산계획`에 ERP 계획 붙여넣기
2. 품번 / 수량 / 시간 정보 누락 여부 확인

### 2.2 현재고 입력
1. Sheet3 `재고조사` 이동
2. D+1 현재고 입력
3. D+2 현재고 입력
4. 공란은 0으로 간주하되, 실제 누락인지 확인

### 2.3 자동 생산계획 생성
1. 매크로 `APPLY_TO_PLAN` 실행
2. 내부 처리 순서
   - Sheet5 계산
   - 기준정보 매핑
   - D+1 정렬
   - 야간 3,900EA 컷
   - 초과분 주간 이월
   - D+2 정렬
   - PLAN_RESULT_FIXED 출력
   - Sheet4 반영
   - V/W 수동서열 자동 반영
   - 빈행 숨김

### 2.4 자동계획 1차 검증
- Sheet4 야간 / 주간 데이터 정상 반영 확인
- K열 지시수량 확인
- T열 이월 반영 확인
- 숨김행/합계행 상태 확인

---

## 3. 수동서열 운영

### 3.1 V/W 입력 방식
1. Sheet4에서 이동할 행 확인
2. V열에 `야간` 또는 `주간` 입력
3. W열에 목표순번 입력

예시
- 야간 3번째로 이동: `V=야간`, `W=3`
- 주간 1번째로 이동: `V=주간`, `W=1`

### 3.2 반영 방법
- 방법 A: `APPLY_TO_PLAN` 재실행
- 방법 B: `APPLY_MANUAL_SEQ` 실행

### 3.3 반영 후 확인
- B/E/G/T만 위치 이동됐는지
- 수식열은 유지되는지
- V/W가 자동 초기화됐는지

---

## 4. 버튼 이동 운영

### 4.1 버튼 준비
- 최초 1회 `SetupMoveButtons` 실행
- 일반모듈 `modBtnAction` 설치 확인

### 4.2 같은 구간 이동
- 이동 블록 선택
- ▲ 또는 ▼ 클릭
- 한 칸 이동 확인

### 4.3 주야간 교차이동
- 야간 마지막 블록에서 ▼ → 주간 상단 이동
- 주간 첫 블록에서 ▲ → 야간 하단 이동

### 4.4 다중선택 이동
- 연속된 블록만 선택
- 이동 후 선택 블록 유지 확인
- 빈 행 포함이면 이동 금지

---

## 5. 복구 절차
1. 생산계획이 꼬였다고 판단되면 `RESTORE_PLAN` 실행
2. 백업 복구 후 다시 `APPLY_TO_PLAN` 실행
3. 필요 시 수동서열 재적용

## 7. 문제 발생 시 확인 순서
1. Sheet4 E열 데이터 실제 존재 여부
2. V/W 오타 여부
3. 목표순번 범위 초과 여부
4. 연속 블록 선택 여부
5. K가 0이면 T/M 같이 확인
6. 버튼 문제면 `SetupMoveButtons` 및 `modBtnAction` 확인
7. 심하면 `RESTORE_PLAN` 후 재실행
