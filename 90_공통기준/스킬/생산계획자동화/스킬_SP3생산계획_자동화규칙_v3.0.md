# SP3 생산계획 자동화 스킬 v3.0

## 1. 목적
이 문서는 **변하지 않는 운영 규칙**만 정의한다.

대상 파일은 SP3 생산지시서 최종본이며, 아래 업무를 일관되게 수행하기 위한 기준 문서다.
- ERP 계획 반영
- 현재고 기준 D+1 / D+2 생산계획 산출
- 야간 3,900EA 기준 이월 처리
- 자동 정렬
- 수동서열 반영
- 생산계획 시트 내 버튼 이동 보정
- 백업 / 복원
- 수식 보호 및 정합성 유지

---

## 2. 파일 구성

| 순서 | 시트명 | 역할 |
|---|---|---|
| 1 | SP3 LINE 기준 정보 | SUB 기준정보 마스터 |
| 2 | OUTER 생산계획 | ERP 계획 원본 붙여넣기 |
| 3 | 재고조사 | 현재고 입력 + D+1/D+2 필요 계산 |
| 4 | 생산계획 | 최종 생산지시 출력 |
| 5 | 계획 확인 | D+1/D+2 지시량 검증 |
| 6 | 생산지원 관리 | 보조 시트 |
| 7 | 수출지원 | 보조 시트 |
| 8 | (2) | 숨김 보조 시트 |
| 9 | PLAN_RESULT_FIXED | 매크로 중간 결과 |
| 10 | PLAN_BACKUP_FIXED | 생산계획 백업 |

핵심 운영 시트는 Sheet1, Sheet2, Sheet3, Sheet4, Sheet5다.

---

## 3. 생산 구간 정의

| 구분 | 의미 | 실제 생산 구간 |
|---|---|---|
| D | 작업 기준일 | 오늘 작업 |
| D+1 | 내일 납품분 | 오늘 야간 생산 |
| D+2 | 모레 납품분 | 내일 주간 생산 |

- 야간 구간(행 3~100) = D+1 대상
- 주간 구간(행 104~203) = D+2 대상 + 야간 이월분

---

## 4. 생산계획 시트 구조 (Sheet4)

### 4.1 행 구조
- 행 3~100: 야간 데이터
- 행 101: 야간 합계
- 행 102: 주간 제목
- 행 103: 주간 헤더
- 행 104~203: 주간 데이터
- 행 204~206: 주간 합계

### 4.2 열 구조

| 열 | 항목 | 구분 |
|---|---|---|
| B | 순서 | 매크로 입력 |
| E | SUB 품번 | 매크로 입력 |
| G | 시간 | 매크로 입력 |
| T | 이월수량 | 매크로 입력 |
| K | 지시수량 | 수식 유지 |
| M | 과부족 | 수식 유지 |
| V | 구간 | 수동 입력 |
| W | 목표순번 | 수동 입력 |

### 4.3 수식 보호 원칙
- 매크로가 직접 쓰는 열: **B / E / G / T**
- 수식열(C,D,F,H~S,K,M)은 절대 직접 덮어쓰지 않는다.
- 수동 이동 / 버튼 이동도 동일하게 **B / E / G / T만 이동**한다.

---

## 5. 계산 원칙

### 5.1 K열 지시수량
- T열 이월수량이 있으면 T열 우선
- 없으면 M열 과부족 기준
- 박스 적입수량 기준 올림 처리

### 5.2 야간 3,900EA 컷
운영 기준은 다음과 같다.
- 야간 누적 진행
- **3,900EA를 넘긴 첫 품번까지는 야간 포함**
- 그 다음 품번부터 주간 이월

### 5.3 이월 처리
- 이월값은 T열에 저장
- 주간 K열은 T열을 우선 반영

---

## 6. 자동 정렬 기준
정렬 우선순위는 아래와 같다.

1. SUB 823 전체 마지막
2. 8PI → 10PI
3. 토션순
4. 같은 토션 내 P/TRAY 위치
5. 같은 토션 + 같은 P/TRAY 내 각도
6. 실제 모델
7. 차종
8. 하우징
9. 홀더
10. 센스
11. 특이사항
12. SUB 품번
13. 동률 시 원본 순서 유지

### 6.1 토션 기준
#### 8PI
4.5 → 4.0 → 3.5 → 3.0 → 5.0 → 5.5 → 7.0 → 기타

#### 10PI
3.5 → 3.0 → 4.0 → 4.5 → 기타

### 6.2 P/TRAY 위치 기준
- 8PI 4.5 = 뒤
- 8PI 4.0 = 앞
- 10PI 4.5 = 앞
- 10PI 4.0 = 뒤
- 기타 = 앞

### 6.3 각도 기준
0 → 180 → 270 → 90 → 공란

## 8. 수동서열 운영 규칙
- V열: `야간 / 주간`
- W열: 목표순번(숫자)
- 적용 대상: **B / E / G / T만 재배치**
- 반영 후 V/W는 **자동 초기화**
- 즉, V/W는 보존 데이터가 아니라 **1회성 이동 명령값**이다.

### 8.1 허용 범위
- 야간 입력 범위: V3:V100 / W3:W100
- 주간 입력 범위: V104:V203 / W104:W203

---

## 9. 버튼 이동 운영 규칙
### 9.1 기본 원칙
- 데이터 블록 선택 시 버튼 표시
- 같은 구간에서는 한 칸 위/아래 이동
- 구간 끝에서는 주야간 교차이동
- 이동 후 선택 블록 유지

### 9.2 다중선택 규칙
- **연속된 데이터 블록만 허용**
- 빈 행 포함 선택 금지
- Ctrl 다중영역 선택 비권장
- 다중선택 교차이동 허용

---

## 10. 백업 / 복원 규칙
- 자동반영 전 `PLAN_BACKUP_FIXED`에 백업
- 복구는 `RESTORE_PLAN` 사용
- 백업 범위는 생산계획 시트 A1:W206 기준

---

## 11. 금지사항
- Sheet4 직접 잘라내기/붙여넣기 금지
- 수식열 직접 수정 금지
- 서열 조정은 반드시 **V/W 또는 버튼 이동**으로만 수행
- ERP 원본은 Sheet2 외 시트에 직접 덮어쓰기 금지

---

## 12. 검증 기준
- 자동반영 후 B/E/G/T만 입력됐는지
- K/M 등 수식열 유지되는지
- 야간/주간 순번(B열) 재부여 정상인지
- V/W 반영 후 자동 초기화되는지
- 버튼 이동 후 선택 블록 유지되는지
- 823 마지막 정렬이 유지되는지
