# 도메인 특화 스킬 작성 가이드: 제조업/ERP

이 문서는 자동차 부품 제조업 현장에서 사용하는 스킬을 작성할 때 참고하는 가이드다.
skill-creator가 새 스킬을 작성하거나 기존 스킬을 개선할 때, 해당 도메인이 제조/ERP/데이터 정산이면 이 문서를 읽는다.

---

## 도메인 분류

| 범주 | 예시 | 핵심 특성 |
|------|------|-----------|
| **ERP 연동** | 라인배치, 생산실적 조회 | 특정 URL + DOM 구조 의존, 동기화 타이밍 |
| **데이터 정산/집계** | 조립비 정산, 월간 실적 | 다중 소스 파일, 수식 보존, 교차 검증 |
| **설비/품질 관리** | ZDM 점검, 불량 현황 | 일일 반복, 체크리스트형, 예외 처리 |

## ERP 연동 스킬

### 셀렉터 전략

ERP 웹 화면을 자동 조작하는 스킬은 가장 깨지기 쉽다.
요소를 찾는 우선순위:

1. `data-*` 속성 (가장 안정적)
2. `id` 속성
3. CSS 클래스 + 구조적 위치 (예: `.grid-row:nth-child(3)`)
4. 텍스트 내용 기반 (최후 수단)

셀렉터가 실패하면 스크린샷을 찍어 사용자에게 확인을 요청한다.

### 타이밍과 대기 조건

나쁜 예: "페이지 로딩을 기다린다"
좋은 예: "`.grid-body tr`이 1개 이상 렌더링될 때까지 최대 5초 대기, 초과 시 재시도 1회"

구체적인 완료 조건 + 타임아웃 + 실패 시 행동을 함께 적는다.

### 시스템 동기화 제약

ERP는 배치 동기화 스케줄이 있는 경우가 많다. 예시:

```markdown
## 동기화 금지 구간
| 구간 | 사유 |
|------|------|
| 매시 x0:10~x0:13 | ERP 배치 동기화 |

검색/저장 직전 시간 확인 → 금지 구간이면 대기 함수 호출
```

### 중단/재개 메커니즘

ERP 자동화는 수백 건을 순차 처리하므로 중간에 끊길 수 있다.
"마지막 처리 위치"를 추적하고 이어서 실행할 수 있어야 한다:

```markdown
### 시작 위치 확인
작업 시작 전 사용자에게 반드시 확인:
- 마지막 처리 완료 항목 (품번/행번호)
- 이어서 시작할 위치
```

## 데이터 정산/집계 스킬

### 파일 구조 정밀 문서화

컬럼 인덱스, 헤더 행 위치, 시트명을 정확히 기록한다. 이게 빠지면 스킬이 작동할 수 없다.

주의할 점:
- 2행 헤더 (1행 상위 카테고리, 2행 실제 컬럼명)가 흔하다
- 병합셀이 있어서 openpyxl로 읽을 때 None이 나올 수 있다
- 시트명이 공백이거나 특수문자인 경우가 있다 → 인덱스로 접근이 안전

```markdown
## 파일 구조
### GERP 실적 파일
- Row 0: 상위 헤더 (병합셀)
- Row 1: 하위 헤더 (실제 컬럼명)
- Row 2+: 데이터

| idx | 컬럼명 | 용도 |
|-----|--------|------|
| 2 | 조립라인 | 라인 분류 키 |
| 6 | 제품번호 | DB 매칭 키 |
```

### 수식 보존

Excel/openpyxl 환경에서 데이터만 수정하고 수식은 건드리지 않아야 한다:
- `data_only=False` (기본값)로 열어야 수식이 보존된다
- 수식 셀은 건드리지 않고 데이터 셀만 업데이트
- 이 규칙은 스킬 본문과 스크립트 양쪽에 명시

### 교차 검증

두 개 이상의 소스를 합치는 스킬은 결과 검증 단계가 필수다:
- 소스 A 합계 vs 소스 B 합계 비교
- 차이 발생 항목 목록 추출
- 차이율 N% 초과 항목 하이라이트

### 비즈니스 규칙에 '왜'를 포함

단순 규칙 나열이 아니라 이유를 설명하면 모델이 예외 상황에서 더 나은 판단을 한다.

나쁜 예: "Usage=2인 품번은 수량을 2배로 계산한다"
좋은 예: "Usage=2인 품번은 제품 1개 생산에 해당 부품이 2개 사용되므로, 조립 기준 실적 = 제품 기준 실적 × 2로 환산한다"

## 설비/품질 관리 스킬

### 기본값 + 커스터마이징

일일 점검처럼 대부분 동일 입력을 반복하는 업무는 기본값(전 항목 정상)을 설정하되, 예외 처리 경로도 열어둔다.

### 멱등성 보장

같은 스킬을 두 번 실행해도 문제없어야 한다.
"이미 입력된 항목은 스킵" 같은 로직이 필요하다.

## 도메인 스킬 품질 체크리스트

```
□ 프론트매터에 한국어 트리거 키워드가 5개 이상
□ 입력 파일의 시트/컬럼/행 구조가 정밀하게 문서화
□ 매핑 테이블(품명→라인, 코드→코드)이 빠짐없이 포함
□ 수식 보존/멱등성 등 데이터 안전 규칙이 명시
□ 오류 대응 시나리오가 3개 이상
□ 중단/재개 가능한 구조 (순차 처리 스킬의 경우)
□ 시스템 제약(동기화 타이밍, 접속 조건 등)이 문서화
□ 비즈니스 규칙에 '왜'가 포함
□ 스크립트가 있다면 실행 명령어가 복붙 가능한 형태
□ SKILL.md 본문이 500줄 이내 (초과 시 references/ 분리)
```
