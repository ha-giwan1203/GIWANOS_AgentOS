# 스킬 패턴 카탈로그

새 스킬을 작성할 때, 아래 패턴 중 1~3개를 조합한다.
각 패턴에서 필수 블록과 선택 블록을 확인하고, 해당 블록을 SKILL.md에 포함시킨다.

---

## 패턴 목록

| ID | 패턴명 | 설명 | 검증된 사례 |
|----|--------|------|-------------|
| P1 | 데이터 파이프라인 | 파일 입력 → 변환/계산 → 파일 출력 | assembly-cost-settlement |
| P2 | 웹 자동화 | 브라우저/API로 외부 시스템 조작 | line-batch-management, zdm-daily-inspection |
| P3 | 리포트 생성 | 데이터 → 표/차트/인사이트 문서 | 정산 리포트 (P1과 조합) |
| P4 | 마스터 데이터 관리 | 기준정보 비교/검증/갱신 | (신규 — 검증 예정) |
| P5 | 워크플로우 오케스트레이션 | 여러 단계를 순서대로 실행 + 중간 확인 | (신규 — 검증 예정) |

### 조합 예시

```
조립비 정산 = P1 + P3 + P5
라인배치 관리 = P2 + P4
ZDM 일상점검 = P2
품번-라인배정 검증 = P1 + P4
```

## P1: 데이터 파이프라인

파일을 읽어서 변환/계산하고 결과 파일을 출력하는 패턴.

### 필수 블록

**입력 파일 구조 명세**
```markdown
## 입력 파일
### [파일명]
- 시트: [시트명 또는 인덱스]
- 헤더 행: [행 번호]
- 데이터 시작 행: [행 번호]

| 컬럼 idx | 컬럼명 | 용도 | 타입 |
|----------|--------|------|------|
```

**변환/계산 규칙**
비즈니스 규칙을 '왜'와 함께 명시한다.

**출력 파일 구조 명세**
입력과 동일한 형식으로 출력 파일 구조를 정의한다.

**교차 검증**
입력 합계와 출력 합계의 정합성 체크.

### 선택 블록
- 매핑 테이블 (50행 이상이면 references/로 분리)
- 예외 처리 규칙
- 수식 보존 규칙 (기존 xlsx를 수정하는 경우)

---

## P2: 웹 자동화

브라우저를 통해 외부 시스템을 조작하는 패턴.

### 필수 블록

**대상 URL/시스템**
```markdown
## 대상 시스템
- URL: [URL]
- 인증: [방식]
```

**셀렉터 전략**
요소를 찾는 우선순위와 fallback 전략.

**대기/타이밍 규칙**
구체적 완료 조건 + 타임아웃 + 실패 시 행동.

**중단/재개 메커니즘**
순차 처리인 경우 마지막 처리 위치 추적.

### 선택 블록
- 시스템 동기화 제약 (금지 구간 등)
- 스크린샷 기반 검증
- 멱등성 보장 로직

---

## P3: 리포트 생성

데이터를 분석해서 보고용 문서를 만드는 패턴.

### 필수 블록

**출력 구조**
```markdown
## 출력 형식
1. 요약 (핵심 수치 3~5개)
2. 상세표 (문제 항목 위주)
3. 개선안 (조치 필요 항목)
4. 3줄 인사이트
```

**차트/시각화 규칙** (해당 시)
어떤 차트를, 어떤 데이터로, 어디에 넣을지.

### 선택 블록
- 파일 형식 선택 (xlsx/docx/md)
- 자동 메일 발송 연동

---

## P4: 마스터 데이터 관리

기준정보를 비교/검증/갱신하는 패턴.

### 필수 블록

**기준 데이터 정의**
어떤 파일/시트가 마스터인지 명확히 한다.

**비교 원칙**
- 기준 파일이 마스터다
- 정규화 규칙 (공백, 대소문자, 접두어 등)
- 부분 일치 규칙 (컬러코드 확장 등)

**판정 코드 체계**
불일치 항목을 분류하는 코드와 그 의미, 실무 조치.

### 선택 블록
- 유사도 탐색 (오타/오기입 감지)
- 이력 관리 (변경 전후 추적)

---

## P5: 워크플로우 오케스트레이션

여러 단계를 순서대로 실행하면서 중간에 사용자 확인을 받는 패턴.

### 필수 블록

**단계 정의**
```markdown
## 워크플로우
| 순서 | 단계 | 자동/수동 | 완료 조건 |
|------|------|-----------|-----------|
| 1 | 파일 수집 | 수동 (사용자 업로드) | 2개 파일 확인 |
| 2 | 데이터 처리 | 자동 | 처리 건수 > 0 |
| 3 | 검증 | 자동 | 차이 0건 |
| 4 | 사용자 확인 | 수동 | 승인 |
| 5 | 최종 출력 | 자동 | 파일 생성 |
```

**중간 체크포인트**
어느 단계에서 사용자 확인이 필요한지, 무엇을 보여주고 무엇을 물어볼지.

### 선택 블록
- 롤백 메커니즘
- 부분 실행 (특정 단계만 재실행)

---

## 공통 구조 블록

모든 패턴에 적용되는 필수 요소:

1. **프론트매터**: name, description (트리거 키워드 포함)
2. **개요**: 이 스킬이 해결하는 문제 + 자동화 전후 비교
3. **입력/출력 테이블**
4. **실행 절차**: Step 단위, 각 스텝에 완료 조건
5. **오류 대응**: 최소 3개 시나리오
6. **금지 사항**
7. **검증 기준**: 완료로 판정하는 조건
