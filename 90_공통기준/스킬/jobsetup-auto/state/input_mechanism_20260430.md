> 어제 v0.3 SKILL.md가 가정한 `pyautogui.type`/`triple_click` 기반 입력은 **0% 동작**.
> 본 문서는 실측 검증된 입력 시퀀스. run_jobsetup.py는 이 시퀀스 그대로 구현해야 한다.
> 검증 환경: 1456×819 해상도, 제품 `1.RSP3PC0129_A` [40] 베어링 부시 조립, A1 패턴 `0 ± 0.05`

## 결정적 결함 (어제 v0.3 가정 vs 실측)

| 어제 가정 (v0.3 SKILL.md) | 실측 (2026-04-30) |
|----------|------|
| `triple_click` X1 → 전체 선택 | ❌ 미작동 |
| `ctrl+a` + `Delete` | ❌ 부분 반영 (그리드 깨짐) |

## 실측 입력 시퀀스 (성공)

### 양수 입력 (예: X3 = 0.04)
```
1. left_click(X3 입력칸 좌표)            # 활성화
2. left_click(C 버튼 좌표)                # 클리어
3. left_click(numpad 0)                   # 0
4. left_click(numpad .)                   # .
5. left_click(numpad 0)                   # 0
6. left_click(numpad 4)                   # 4
```
결과: 입력칸 = `0.04`, 그리드 X3 컬럼 = `0.04` (실시간 부분 반영, 마지막 키 직후)

### 음수 입력 (예: X3 = -0.04)
```
1. left_click(X3 입력칸 좌표)
2. left_click(C 버튼 좌표)
3. key("minus")                            # 키보드 - 키 (numpad에 - 없음)
4. left_click(numpad 0)
5. left_click(numpad .)
6. left_click(numpad 0)
7. left_click(numpad 4)
```
결과: 입력칸 = `-0.04`, 그리드 X3 = `-0.04`. **음수 부호는 키보드 `-` 키만 작동** (numpad UI는 양수 전용).

## OK/NG 동작 관찰

- 첫 진입 시 X1/X2/X3 모두 OK ✓ 자동 표시 (이전 저장 데이터 또는 fresh open 시 default)
- 입력 시퀀스 완료 후 OK 자동 체크 유지 (재클릭 불필요)
- 결과 종합 박스(1340, 415) = "OK" 자동 표시
- **OK→NG 토글 미작동**: 단순 NG 라디오 클릭으로 OK 해제 안 되었음 — lock 상태 의심
- 변경된 입력값을 다시 클리어하려면 입력칸 활성 → C 버튼 → 새 시퀀스 흐름이 정상 (검증됨)
- **즉 OK 체크 직접 클릭은 무인 자동화에서 불필요** — 입력 시퀀스만 수행하면 OK 자동 유지

## 품번 일반성 (오늘 새 발견)

- 매일 1번 품번이 바뀐다 (어제 `1.RSP3SC0383_A` → 오늘 `1.RSP3PC0129_A`)
- SP3 RETRACTOR 카테고리는 **공정 11개 구조 동일** (확정)
- 공정 [40] 검사항목 1개 동일 (`스플 베어링 부시 "0"점 MASTER GAGE`, A1 `0 ± 0.05`)
- **즉 공정/검사항목 구조는 SP3 RETRACTOR 카테고리 표준** — 단일 품번 학습이 카테고리 내 공유 가능
- 단 다른 카테고리(SD9 등) 진입 시 재학습 필요 — 본 스킬 범위 외

## DB 영향 분석

- 입력칸/그리드 변경은 **SmartMES 메모리 임시 상태**
- 저장 클릭 전에는 DB 미반영
- 검사항목/공정 재선택 시 일부 잔존 (fresh load 트리거 미확인)
- SmartMES 클라이언트 재시작 시 메모리 클리어 = 화면 잔존값 사라짐
- **무인 실행 안전성**: 저장 시점 명확히 정의된 후에만 commit 모드 활성

## run_jobsetup.py 도입 절차 (이 문서가 baseline)

1. 좌표 캘리브레이션 = 본 문서 좌표값 (1456×819 한정)
2. 입력 함수: `enter_value(field_xy, value)`:
   - left_click(field_xy)
   - left_click(C_xy)
   - 만약 value < 0: keyboard `-`
   - for 각 char in str(abs(value)):
     - left_click(numpad[char])
3. 측정값 정규분포 생성 = 어제 SKILL.md 정책 그대로 (`random.gauss`, σ=오차/3)
4. OK 체크 = 자동 (별도 클릭 불필요)
5. 저장 시점 = 첫 commit 1건 학습 후 SKILL.md 박음
6. 좌표 의존 = 1456×819 외 해상도는 fail-fast
