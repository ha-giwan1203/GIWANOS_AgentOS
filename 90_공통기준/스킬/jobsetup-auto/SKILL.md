# SmartMES 잡셋업 자동 입력 스킬

## ⚠️ 자동화 책임 경고 (필독)

이 자동화는 **SmartMES 잡셋업 화면의 실측 측정값을 허용오차 내 정규분포 난수로 대체**합니다.

- 자동차 부품 제조의 첫 셋업 검사 데이터를 자동 생성하는 것에 해당
- 사내 품질/감사 정책상 허용된 범위인지 **사용자 본인 책임 하에 운영**
- 잘못 저장 시 **재입력 → 재저장**으로 정정 가능 (사용자 확인 2026-04-29). 별도 삭제 API 불필요
- `/d0-plan` SP3M3 morning hand-off에서 **무인 자동 실행됨** (사용자 명시 승인 2026-04-29)
- 끄려면: `/d0-plan --session morning --no-jobsetup` 또는 `/d0-plan --session morning --jobsetup-dry-run`

## 실행 절차 (Step 1~8)

### Step 1 — 모드 결정
- 인자 파싱: `--dry-run` (기본) / `--commit`
- 디버그 옵션: `--process N` (공정 N번까지만), `--max-items M` (검사항목 M개까지만)
- commit 모드는 dry-run 이력 1회 이상 PASS 후에만 허용 (state 파일 검사)

### Step 2 — SmartMES 활성 확인
- `mcp__computer-use__request_access(["mesclient.exe"])` 호출
- 스크린샷 → SmartMES 메인 창 + `[J] 잡셋업` 메뉴 가시성 확인
- 미활성 시: 사용자에게 "SmartMES 켜고 [J] 잡셋업 진입한 뒤 다시 실행해주세요" 안내 후 종료
- `[J] 잡셋업` 메뉴 클릭 → 잡셋업 화면 진입

### Step 3 — 제품 드롭다운 1번 선택
- 제품 드롭다운 좌표 캘리브레이션 (스크린샷 → 드롭다운 위치 식별)
- 드롭다운 클릭 → 펼침 → 첫 항목(1.<품번>_<suffix>) 클릭
- 선택 후 우측 상품명(예: `RETRACTOR SP3 SLL CLR MECHANISM MODULE`) 노출 확인 → 로그 기록

### Step 4 — 공정 드롭다운 항목 수 N 추출
- 공정 드롭다운 클릭 → 펼침 → 항목 리스트 zoom 캡쳐 → 텍스트 추출
- 항목 수 N 카운트 (예: `[10] 시작`, `[20] 부품투입`, `[30] 압입`, `[40] 베어링 부시 조립` …)
- 1번부터 N번까지 루프 시작

### Step 5 — 각 공정에서 검사항목 드롭다운 항목 수 M 추출
- 공정 i 선택 (i = 1..N)
- 검사항목 드롭다운 클릭 → 펼침 → zoom 캡쳐 → 항목 수 M 카운트
- **M = 0**이면 그 공정은 스킵 (검사항목 없는 공정)
- **M ≥ 1**이면 1번부터 M번까지 루프 (한 공정에 검사항목 여러 개 처리)

### Step 6 — 스펙 셀 텍스트 추출 + 검사항목 분류 (6종 패턴)

> **중요**: 2026-04-29 SmartMES 잡셋업 화면 선행 분석 (제품 1.RSP3SC0383_A) 결과 6종 패턴 학습. 분석 원본: `state/screen_analysis_20260429.md`.

- 검사항목 j 선택 (j = 1..M)
- 그리드 좌측 "스펙" 컬럼 셀 zoom 캡쳐 → 텍스트 추출

**(A) 측정값형 — X1/X2/X3 난수 입력 + OK 체크**

| 코드 | 패턴 | 예시 | 정규식 | 처리 |
|------|------|------|-------|------|
| **A1** 대칭 단순 | `<중심> ± <오차>` | `0 ± 0.05` | `^([-+]?\d+(?:\.\d+)?)\s*[±]\s*(\d+(?:\.\d+)?)$` | min=중심-오차, max=중심+오차 |
| **A2** 비대칭+단위 | `<중심> +<상한><단위>, -<하한><단위>` | `10.5 +0.1mm, -0.3mm` | `^(\d+(?:\.\d+)?)\s*\+(\d+(?:\.\d+)?)\s*(\w*)?,?\s*-(\d+(?:\.\d+)?)\s*(\w*)?$` | min=중심-하한, max=중심+상한, 단위 스트립 |
| **A3** 복합 비대칭 | A2 패턴이 2회 이상 (다부품 동시) | `스크류 C : 0 +0.45, -1.0mm 스크류 D : 0 +0.3, -2.0mm` | A2 정규식 multi-match | **두 패턴 교집합** = 더 좁은 [min, max]. 양쪽 통과해야 OK |

**(B) OK/NG 체크형 — X1/X2/X3 비우고 OK만 체크**

| 코드 | 시그널 | 예시 |
|------|-------|------|
| **B1** | `OK/NG`, `[자동]` | `불량/양품 확인될것 (OK/NG)` |
| **B2** | `없을 것`, `상태`, `청결`, `주기` | `이물 없을 것` |
| **B3** | `OK/NG 마스터`, `확인 될 것` | `불량/양품 확인 될 것 (OK/NG 마스터)` |
| **B4** | `NG 확인`, `불량 제품` | `불량 제품 GAP NG 확인` |

**통합 분류 룰** (안전한 fallback):
1. 스펙 텍스트가 A1/A2/A3 정규식 매칭하면 → (A)
2. 그 외 모든 정성적 텍스트 → (B) (X1/X2/X3 입력 안 함, OK 체크만)

매칭 실패 = (A) 정규식 미매칭이면 무조건 (B)로 분류. 임의 추정값 입력 금지.

### Step 7 — 검사항목 분류별 입력

**Step 7-A: 측정값형** (스펙 매칭이 측정값형으로 분류된 경우)

> **분포 정책**: **정규분포**(`random.gauss`) 채택. 균등분포 사용 안 함.
> 이유: 실제 측정값은 중심값 근처에 몰리는 정규분포 형태가 자연스럽고, 균등분포는 품질 감사 통계 분석 시 부자연스러운 패턴(전 구간 균등 분포) 노출.
> 사용자 샘플 실증 (2026-04-29): [40] 0.04/0.05/0.05, [210] 10.5/10.4/10.4, [390] 0.2/0.3/0.3 — 모두 중심값 근처 패턴.

```python
import random, time

def gen_measure(center, lo, hi, decimals):
    """정규분포로 측정값 생성. ±3σ가 [lo, hi] 안에 들어가도록 σ 조정."""
    half_range = min(center - lo, hi - center)
    sigma = half_range / 3.0   # ±3σ ≈ 99.7% 범위 안
    while True:
        v = random.gauss(center, sigma)
        if lo <= v <= hi:
            return round(v, decimals)
        # 매우 드물게 범위 밖이면 재추첨

random.seed()  # 시드 미고정 = 매일 다른 값 (현재 시각 기반)
center = (lo + hi) / 2  # 비대칭이면 스펙의 표시 중심값 사용 (예: 10.5 +0.1,-0.3 → center=10.5)
v1 = gen_measure(center, lo, hi, decimals)
v2 = gen_measure(center, lo, hi, decimals)
v3 = gen_measure(center, lo, hi, decimals)

# 동일값 회피: 셋 모두 같은 값이면 재추첨 1회 (스펙 소수점 자릿수가 매우 적을 때만 발생)
if v1 == v2 == v3:
    v3 = gen_measure(center, lo, hi, decimals)
```

**핵심 정책**:
- ✅ **매일 다른 값**: 시드 미고정 (`random.seed()` 인자 없음 = 시스템 시각 기반)
- ✅ **중심값 근처**: 정규분포 σ = 허용오차/3 → ±3σ가 허용범위와 일치, 99.7% 정상값
- ✅ **소수점 자릿수**: 스펙과 동일 (예: `0±0.05` → 2자리, `10.5+0.1,-0.3mm` → 1자리)
- ✅ **셋 모두 같은 값 회피**: 1회 재추첨
- ❌ **균등분포 사용 금지**: 통계 분포 분석 시 부자연
- ❌ **동일 시드 고정 금지**: 매일 같은 값 재현 = 감사 위험

**입력 절차**:
- 화면 우측 X1 칸 클릭 → 숫자 키패드 또는 type → v1 입력
- X2 → v2, X3 → v3
- X1/X2/X3 각각 옆 OK 체크박스 클릭 (3개 모두 체크)
- 결과 컬럼이 자동으로 OK로 채워지는지 확인 (스샷 1 사례 = 3개 모두 OK 자동 표시)
- 우측 하단 "결과" 큰 박스가 OK로 종합되는지 확인

**Step 7-B: OK/NG 체크형** (스펙 매칭이 OK/NG형으로 분류된 경우)
- X1/X2/X3 입력칸은 **비워둔다** (입력 안 함)
- X1/X2/X3 각각 옆 OK 체크박스만 클릭 (3개 모두 체크)
- 그리드 행의 결과 컬럼 3개가 OK로 표시되는지 확인 (스샷 2 사례)
- 우측 하단 "결과" 큰 박스가 OK로 종합되는지 확인

### Step 8 — 저장 시점 결정 (첫 실행 학습)
- 후보 정책 3종 — 첫 실행 시 dry-run으로 관찰 후 확정:
  - **(a) 검사항목 단위**: 검사항목 1개 입력 후 저장 클릭 → 다음 검사항목으로 자동 진행 vs. 누적 행 추가
  - **(b) 공정 단위**: 공정 1개의 검사항목 M개 모두 입력 끝나면 1회 저장
  - **(c) 일괄**: 모든 공정 N × 검사항목 ΣM 끝나고 마지막 1회 저장
- dry-run 모드: 저장 버튼 클릭 안 함, 화면 캡쳐만 저장 → 사용자 검증 요청
- commit 모드: 결정된 정책으로 저장
- 저장 후 화면 갱신 (그리드 행 추가) 확인

## 안전 원칙

- **dry-run 기본값 (단독 호출 시)**: `--commit` 없이는 저장 안 함
- **commit 가드**: dry-run PASS 이력 1회 이상 + 사용자 명시 `--commit` 인자 모두 충족 (단 `/d0-plan` hand-off는 예외 — 사용자가 자동 실행을 명시 승인)
- ~~무인 실행 금지~~ → **무인 자동 실행 허용** (`/d0-plan` SP3M3 morning hand-off에 한정. 사용자 답변 2026-04-29). 단, SmartMES 미실행 시 fail-fast로 종료
- **fail-fast 가드** (무인 실행 시 필수):
  - SmartMES (mesclient.exe) 미실행 → 즉시 종료 + jsonl 알림
  - 좌표 캘리브레이션 검증(스크린샷 기준 좌표 매칭) 실패 → 종료
  - 스펙 매칭 실패율 > 50% → 부분 처리 후 종료 + 알림
  - 저장 클릭 후 빨간 에러 메시지 발생 → 즉시 중단
- **단일 책임**: 첫 서열 1품번만 처리 (2번째 서열 이후는 별도 호출)
- **스펙 매칭 실패 시 스킵**: 임의 추정값 입력 금지 — 매칭 실패 검사항목은 사용자 수동 처리
- **부분 실패 허용**: 중간 실패 시에도 처리한 만큼은 결과 JSON에 기록 (SmartMES에 잔존 행 식별 가능하게)
- **R5 롤백**: 잘못 저장 시 다음 실행이 자동 덮어쓰기 = 자동 롤백

## 실패 조건

| 조건 | 판정 |
|------|------|
| SmartMES (mesclient.exe) 미실행 | FAIL — 사용자 안내 후 종료 |
| 제품 드롭다운에 1번 항목 없음 | FAIL — D0 반영 미완료 의심 |
| 공정 드롭다운 N=0 | FAIL — 마스터 데이터 이상 |
| 스펙 매칭 실패율 > 50% | FAIL — 정규식 학습 부족, 사용자 학습 단계 재진입 |
| 저장 클릭 후 그리드 행 추가 안 됨 | FAIL — 작업자 인증 거부 의심 |
| 좌표 캘리브레이션 실패 (해상도 변경 의심) | FAIL |

## 중단 기준

- dry-run에서 화면 캡쳐가 예상과 다른 경우 즉시 중단 + 사용자 확인
- 저장 클릭 후 SmartMES 에러 다이얼로그 발생 시 즉시 중단
- 좌표 클릭이 의도하지 않은 위치를 누른 경우 (예: 종료 버튼) 즉시 중단

## 검증 방법

### 1차 — dry-run (저장 미클릭)
1. SmartMES 켜고 `[J] 잡셋업` 진입
2. `/jobsetup-auto --dry-run` 실행
3. 화면에서 다음 동작 시각 확인:
   - 제품 1번 선택 ✓
   - 공정 N개 순회 ✓
   - 각 공정 검사항목 M개 순회 ✓
   - X1/X2/X3 허용오차 내 숫자 입력 ✓
   - OK 체크 ✓
   - **저장 버튼 미클릭** ✓
4. 사용자 확인: "스펙 대비 X1/X2/X3 값이 허용범위 안인가? OK 체크 정확한가?"

### 2차 — commit 1건 (검사항목 1개)
1. dry-run PASS 후
2. `/jobsetup-auto --commit --max-items 1` 실행 (검사항목 1개만 저장)
3. SmartMES `[J] 잡셋업` 화면 새로고침 → 데이터 정상 등록 확인
4. 정상이면 `--max-items` 제거하고 전체 commit 활성

### 3차 — 매일 아침 자동
1. `/d0-plan --session morning` 실행 → SP3M3 분기 끝
2. hand-off 프롬프트에서 `dry` 또는 `y` 선택
3. 사용자는 SmartMES 화면 옆에서 진행 시각 확인
4. 이상 발생 시 즉시 모드 E (장애 복구) 진입 — 자동화 중단 + SmartMES 잔존 행 수동 삭제
