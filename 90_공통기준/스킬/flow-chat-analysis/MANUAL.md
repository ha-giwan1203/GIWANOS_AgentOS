# flow-chat-analysis — MANUAL

> 5 Step 절차 / 분류 체계 / 보고서 형식. SKILL.md는 호출 트리거 + 80줄 요약.

## 개요
Flow.team SP3S03 프로젝트 채팅방 원문을 입력 → 품질문제 + 설비문제 자동 분류 → 월별 보고서.

## 트리거 / 비트리거

발동:
- 채팅 분석 / 채팅 품질문제 / 채팅 설비이슈
- SP3S03 월별 보고 / 채팅방 품질·설비 보고
- Flow 채팅 분석 / 채팅 월간 보고

발동 X:
- 단순 채팅 내용 질문 (분류·보고 아닌 경우)
- ERP/MES 데이터 기반 불량 보고 → quality-defect-report
- 일반 품번 조회 → partno-management

## 입력

### 필수
1. 대상 월: YYYY-MM (예: 2026-03)
2. 채팅 원문: 형식 A 또는 B

### 형식 A — 텍스트 붙여넣기
```
[2026-03-31 06:42] 김민건_대원테크
스프링 하우징 재고 부족하여 06:20분까지 생산하고 라인 마감하였습니다.

[2026-03-31 07:45] 하지완_대원테크
상기 서브품 3종 MES BOM 미 구성으로 등록 불가...
```
- 메시지 사이 빈 줄로 구분
- `[날짜 시간] 작성자명` 헤더 + 본문

### 형식 B — 텍스트 파일
위 형식과 동일한 .txt 파일 경로 제공

### 선택 입력
- 채팅방명 (기본값: SP3S03 운영)
- 분류 필터 (품질만 / 설비만 / 전체)

## 실행 절차

### Step 1 — 입력 파싱
- 메시지 단위 파싱: 날짜/시각, 작성자명, 본문, 첨부 여부 ([이미지], [파일])
- 대상 월 외 메시지 제외

### Step 2 — 키워드 1차 필터링

품질 키워드: 불량, NG, 스크래치, 찍힘, 오염, 치수, 외관, 누락, 오삽, 미삽, 역삽, 이종, 파손, 크랙, 변형, 흠집, 도장, 용접, 검사, 클레임, 반품, 재검

설비 키워드: 설비, 고장, 수리, 정지, 에러, 알람, 센서, 실린더, 모터, 컨베이어, 지그, 유압, 공압, 전기, 누유, 파손, 마모, 교체, 보전, 점검, 비전, 카메라, 금형

키워드 0건 매칭 → "기타" 분류 또는 제외.

### Step 3 — 문맥 판정 (LLM 분류)

**대분류: 품질**
| 중분류 | 소분류 예시 |
|--------|-----------|
| 외관 | 스크래치, 찍힘, 오염, 도장불량 |
| 조립 | 오삽, 미삽, 역삽, 누락 |
| 검사 | NG, 재검, 판정불일치, 치수불량 |
| 클레임 | 고객클레임, 사내반품 |
| 원인미확정 | 키워드 매칭됐으나 분류 불가 |

**대분류: 설비**
| 중분류 | 소분류 예시 |
|--------|-----------|
| 고장 | 센서에러, 비전불량, 모터고장, 알람 |
| 수리 | 교체, 파손, 마모 |
| 정지 | 라인정지, 설비정지, 긴급정지 |
| 에러 | PLC에러, 통신에러, 프로그램에러 |
| 보전 | 점검, 예방보전, 윤활 |
| 기구 | 실린더, 지그, 치공구, 금형, 체결 |

**대분류: 기타**
| 중분류 | 소분류 예시 |
|--------|-----------|
| 자재/재고 | 재고부족, 결품, 투입지연 |
| 운영공지 | 생산계획, 생산취소, 일정변경 |
| 단순전달 | 인사, 공지, 확인요청 |

#### 분류 규칙
1. 키워드 1차 매칭으로 후보 선별
2. 본문 문맥 → 대/중/소 확정
3. **다중 태깅 허용** (품질+설비 동시 가능)
4. 분류 불가 → "기타" + 원문 보존
5. 단순 전달/공지/인사 제외

### Step 4 — 집계
- 월별 총 건수 (품질/설비/기타)
- 중분류별 건수
- 상위 반복 이슈 5건 (중분류+소분류 빈도순)
- 중복 메시지 제거 (동일 작성자, 유사 내용, 같은 날짜)

### Step 5 — 보고서 출력
```
## SP3S03 채팅방 이슈 분석 — {YYYY}년 {M}월

### 요약
- 분석 기간: {YYYY}-{MM}-01 ~ {YYYY}-{MM}-{말일}
- 총 분석 메시지: N건
- 이슈 메시지: N건 (품질 N건, 설비 N건, 기타 N건)
- 상위 품질 이슈: ...
- 상위 설비 이슈: ...

### 품질문제 (N건)
| # | 날짜 | 작성자 | 중분류 | 소분류 | 내용 요약 |

### 설비문제 (N건)
| # | 날짜 | 작성자 | 중분류 | 소분류 | 내용 요약 |

### 기타 (N건)

### 3줄 인사이트
1. ...
2. ...
3. ...
```

## 중단 기준
1. 원문 1,000건 초과 → 처리 시간 확인 후 분할 안내
2. 분류 불확실 비율 50% 초과 → 키워드 사전 검토
3. 대상 월 외 메시지 다수 혼입 → 필터 재확인

## 검증 항목
- 대상 월 필터 정확 (월 외 0건)
- 분류 합계 = 총 이슈 건수
- 중복 제거 적용
- 보고서 형식 준수
