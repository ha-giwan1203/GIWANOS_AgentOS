# flow-chat-analysis 스킬 — 구현 계획

## 목표

Flow 채팅 원문에서 품질·설비 이슈를 월별 분류/집계/보고하는 스킬 v1 작성.

## 비목표

- Flow 자동 로그인 / 채팅방 자동 접속
- 팝업 채팅 직접 조작 (브라우저 자동화)
- 공식 미확인 API 강행 연동
- 이미지/첨부파일 분석 (텍스트 기반만)

## 완료 조건

1. SKILL.md 작성 완료 (트리거, 입력, 출력, 실행 절차 포함)
2. 예시 입력으로 수동 검증 1회 이상 통과
3. 분류 체계 적용 시 품질/설비/기타 정확 분류율 80% 이상
4. 월별 보고서 출력 형식이 정의와 일치

  ```

  ```
- 빈 줄로 메시지 구분

## 분류 체계

### 대분류: 품질

- 외관: 스크래치, 찍힘, 오염, 도장불량
- 조립: 오삽, 미삽, 역삽, 누락
- 검사: NG, 재검, 판정불일치, 치수불량
- 클레임: 고객클레임, 사내반품
- 원인미확정: 키워드 매칭됐으나 분류 불가

### 대분류: 설비

- 고장: 센서에러, 비전불량, 모터고장, 알람
- 수리: 교체, 파손, 마모
- 정지: 라인정지, 설비정지, 긴급정지
- 에러: PLC에러, 통신에러, 프로그램에러
- 보전: 점검, 예방보전, 윤활
- 기구: 실린더, 지그, 치공구, 금형, 체결

### 대분류: 기타

- 생산계획, 자재/재고, 단순 전달/공지

## 분류 규칙

1. 키워드 1차 매칭으로 후보 선별
2. 문맥 판정으로 대/중/소 분류 확정
3. 다중 태깅 허용: 한 메시지에 품질+설비 동시 분류 가능
4. 분류 불가 시 "기타" + 원문 보존

## 접근 방식

### 선택한 방법
- v1: 사용자가 채팅 원문을 텍스트로 제공 → Claude가 키워드+문맥으로 분류 → 월별 보고서 생성
- LLM 기반 분류 (키워드 1차 필터 + 문맥 판정)

### 선택 이유
- 공식 API에 메시지 조회 엔드포인트가 없어 자동 수집 불가
- 수동 입력이면 수집 경로 변경에 영향받지 않음
- 키워드만으로는 오분류 위험 → LLM 문맥 판정 병행

### 선택하지 않은 방법 및 이유
- 브라우저 자동화: Flow.team 채팅이 팝업 전용이라 MCP 탭 그룹 접근 불가
- 공식 API 자동 수집: 메시지 조회 엔드포인트 미존재
- 내부 API 캡처: 비공식이라 유지보수 리스크 높음 → v2 옵션으로만 유지

### 참조 구현
- 파일: 90_공통기준/스킬/quality-defect-report/SKILL.md
- 참조 포인트: 불량 유형 분류 체계, 보고서 구조
