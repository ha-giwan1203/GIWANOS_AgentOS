# flow-chat-analysis 스킬 — 구현 계획

## 목표

Flow 채팅 원문에서 품질·설비 이슈를 월별 분류/집계/보고하는 스킬 v1 작성.

## 비목표

- Flow 자동 로그인 / 채팅방 자동 접속
- 팝업 채팅 직접 조작 (브라우저 자동화)
- 공식 미확인 API 강행 연동
- 이미지/첨부파일 분석 (텍스트 기반만)

## 완료 조건

1. SKILL.md 작성 완료 (트리거, 입력, 출력, 실행 절차 포함)
2. 예시 입력으로 수동 검증 1회 이상 통과
3. 분류 체계 적용 시 품질/설비/기타 정확 분류율 80% 이상
4. 월별 보고서 출력 형식이 정의와 일치

## 입력 정의

- 대상 월: YYYY-MM 형식
- 채팅 원문: 텍스트 파일 또는 직접 붙여넣기
- 원문 형식 (1메시지 단위):
  ```
  [YYYY-MM-DD HH:MM] 작성자명
  메시지 본문 (여러 줄 가능)
  ```
- 빈 줄로 메시지 구분

## 분류 체계

### 대분류: 품질

- 외관: 스크래치, 찍힘, 오염, 도장불량
- 조립: 오삽, 미삽, 역삽, 누락
- 검사: NG, 재검, 판정불일치, 치수불량
- 클레임: 고객클레임, 사내반품
- 원인미확정: 키워드 매칭됐으나 분류 불가

### 대분류: 설비

- 고장: 센서에러, 비전불량, 모터고장, 알람
- 수리: 교체, 파손, 마모
- 정지: 라인정지, 설비정지, 긴급정지
- 에러: PLC에러, 통신에러, 프로그램에러
- 보전: 점검, 예방보전, 윤활
- 기구: 실린더, 지그, 치공구, 금형, 체결

### 대분류: 기타

- 생산계획, 자재/재고, 단순 전달/공지

## 분류 규칙

1. 키워드 1차 매칭으로 후보 선별
2. 문맥 판정으로 대/중/소 분류 확정
3. 다중 태깅 허용: 한 메시지에 품질+설비 동시 분류 가능
4. 분류 불가 시 "기타" + 원문 보존

## 구현 단계

### Step 1: 입력 파서 설계

- 수정 파일: SKILL.md (입력 포맷 정의 섹션)
- 채팅 원문 → 메시지 단위 파싱 (날짜, 작성자, 본문)

### Step 2: 분류 엔진 설계

- 수정 파일: SKILL.md (분류 규칙 섹션)
- 키워드 매핑 테이블 + 문맥 판정 프롬프트

### Step 3: 집계 규칙 정의

- 수정 파일: SKILL.md (집계 섹션)
- 월별 필터링, 대/중/소 카운트, 상위 이슈 추출

### Step 4: 보고서 템플릿 작성

- 수정 파일: SKILL.md (출력 템플릿 섹션)
- 위 출력 정의 형식 반영

### Step 5: SKILL.md 최종 작성

- 신규 파일: 90_공통기준/스킬/flow-chat-analysis/SKILL.md
- frontmatter + 트리거 + 입력 + 분류 + 출력 + 실행 절차

### Step 6: 수동 검증

- 예시 채팅 원문으로 스킬 실행
