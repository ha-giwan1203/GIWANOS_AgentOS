# Flow.team 채팅방 분석 스킬 — 조사 문서

## 1. 목적

Flow.team SP3S03 프로젝트 채팅방에서 설비문제·품질문제 관련 메시지를 수집·분류하여 월별 보고서를 생성하는 스킬 설계를 위한 사전 조사.

## 2. Flow.team 공식 API 조사 결과

### 2.1 Open API 센터

- URL: https://api.flow.team/docs
- 이용 조건: PRO/울트라 플랜 (14일 무료체험 가능)
- 현재 플랜: 무료 → API 사용 불가

### 2.2 Chats API 엔드포인트 (공개 문서 기준)

| 메서드 | 경로 | 기능 |
|--------|------|------|
| GET | /v1/chats/participants/{participantId} | 참여중인 채팅방 조회 |
| GET | /v1/chats/{roomId} | 채팅방 상세정보 조회 |
| POST | /v1/chats/{roomId}/messages | 메시지 전송 |
| POST | /v1/chats/{roomId}/participants | 참여자 초대 |

### 2.3 공식 API 가능 범위

- 채팅방 목록 조회
- 채팅방 상세정보 조회
- 채팅방에 메시지 전송
- 참여자 초대

### 2.4 공식 API 불가 범위

- **메시지 목록 조회 (이력 읽기) — 엔드포인트 없음**
- **메시지 검색 — 엔드포인트 없음**
- 메시지 삭제/수정
- 첨부파일 조회

### 2.5 UI 기능 vs OpenAPI 노출 차이

Flow.team 도움말 센터에는 채팅방 내 키워드·기간 검색 UI 기능이 존재한다고 명시되어 있음.
그러나 이 기능은 공개 OpenAPI에는 노출되지 않은 상태.
즉 **"UI에 있는 기능 = API로 가능"이 아님**.

### 2.6 PRO 전환 시 해결 여부

- PRO 전환은 API 접근의 **필요조건**이지 **충분조건이 아님**
- 공개 문서 기준으로 PRO에서도 메시지 조회 API는 보이지 않음
- PRO 전환 후 추가 엔드포인트가 열리는지는 실제 전환 후 확인 필요 (검증필요)

## 4. 수집 경로 후보 비교

### 경로 1: 공식 API

- 현황: 메시지 조회 엔드포인트 미존재
- 리스크: 낮음 (공식 지원)
- 유지보수성: 높음
- 판정: **현재 불가. PRO 전환 후 재확인 필요**

### 경로 2: 수동 내보내기 / 원문 붙여넣기

- 현황: Flow.team 채팅 내보내기 기능 미확인 (유료 기능일 가능성)
- 대안: 사용자가 채팅방 스크롤 → 텍스트 복사 → 파일로 저장
- 리스크: 낮음 (사용자 직접 수행)
- 유지보수성: 높음 (입력 형식만 고정하면 됨)
- 판정: **v1 권장 경로. 가장 빠르게 실물 스킬 완성 가능**

### 경로 3: 내부 API 캡처 (비공식)

- 현황: 채팅방 스크롤 시 발생하는 XHR/Fetch 요청 캡처로 내부 API 발견 가능
- 방법: F12 Network 탭 → 스크롤 → 요청 URL/헤더/파라미터 기록
- 리스크: 높음 (비공식, 언제든 변경 가능, 인증 토큰 관리 필요)
- 유지보수성: 낮음
- 판정: **v2 옵션으로만 취급. 스킬 본체에 박지 않음**

### 권장안

v1: 경로 2 (수동 입력) → v2: 경로 3 (내부 API) 또는 경로 1 (PRO 전환 후)

## 8. 스킬 구조 설계 (3단계 분리)

### 단계 1: 수집기 (Collector)

- v1: 사용자가 채팅 원문을 텍스트 파일로 제공
- v2: 내부 API 캡처 스크립트 또는 공식 API (PRO)

### 단계 2: 정규화기 (Normalizer)

- 입력: 원문 텍스트
- 출력: 정규화된 JSON/CSV
- 필드: 날짜, 작성자, 본문, 채팅방명

### 단계 3: 분류·보고 스킬 (Classifier + Reporter)

- 입력: 정규화된 데이터
- 처리: 키워드 매칭 → 대/중/소 분류 → 월별 집계
- 출력: 월별 보고서 (위 출력 정의 형식)
