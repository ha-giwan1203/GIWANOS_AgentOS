# Flow 채팅방 분석

> 5 Step 절차 / 분류 체계 / 보고서 형식은 [MANUAL.md](MANUAL.md). 용어는 ../GLOSSARY.json.
> 채팅방 ID: roomSrno 2938379 (SP3S03 운영)

## 입력 (필수)
1. 대상 월: YYYY-MM
2. 채팅 원문: 텍스트 붙여넣기 또는 .txt 파일

## 절차 (요약)
1. 메시지 파싱 (`[날짜 시간] 작성자` 헤더 + 본문)
2. 키워드 1차 필터링 (품질/설비 약 23개 each)
3. 문맥 판정 → 대분류(품질/설비/기타) + 중분류 + 소분류
4. 집계 (월별 총 + 중분류별 + 상위 5건 + 중복 제거)
5. 월별 보고서 출력

## 분류 체계
- **품질**: 외관 / 조립 / 검사 / 클레임 / 원인미확정
- **설비**: 고장 / 수리 / 정지 / 에러 / 보전 / 기구
- **기타**: 자재·재고 / 운영공지 / 단순전달

## verify
- 대상 월 외 메시지 0건
- 분류 합계 = 총 이슈 건수
- 중복 제거 적용
- 보고서 형식 (요약 → 품질 → 설비 → 기타 → 인사이트) 준수

## 실패 시
- 키워드 매칭 0건 → 원문 형식 불일치 의심
- 분류 불확실 50% 초과 → 사전 검토 요청
- 상세 → MANUAL.md "실패 조건" + "중단 기준"
