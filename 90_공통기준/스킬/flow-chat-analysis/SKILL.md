# Flow 채팅방 분석 (flow-chat-analysis)

## 개요

Flow.team SP3S03 프로젝트 채팅방의 원문 메시지를 입력받아, 품질문제와 설비문제를 자동 분류하고 월별 보고서를 생성한다.

## 트리거 조건

다음 중 하나라도 해당하면 이 스킬을 발동한다:
- "채팅 분석", "채팅 품질문제", "채팅 설비이슈"
- "SP3S03 월별 보고", "채팅방 품질/설비 보고"
- "Flow 채팅 분석", "채팅 월간 보고"

발동하지 않는 경우:
- 단순 채팅 내용 질문 (분류/보고 아닌 경우)
- ERP/MES 데이터 기반 불량 보고 → quality-defect-report 스킬 사용
- 일반 품번 조회 → partno-management 스킬 사용

## 입력

### 필수 입력

1. **대상 월**: YYYY-MM 형식 (예: 2026-03)
2. **채팅 원문**: 아래 형식 중 하나로 제공

#### 형식 A — 텍스트 붙여넣기

```
[2026-03-31 06:42] 김민건_대원테크
스프링 하우징 재고 부족하여 06:20분까지 생산하고 라인 마감하였습니다.

[2026-03-31 07:45] 하지완_대원테크
상기 서브품 3종 MES BOM 미 구성으로 등록 불가하여 금일 야간 구형 일반 바코드 생성 후 생산 진행하겠습니다.
```

- 메시지 사이 빈 줄로 구분
- `[날짜 시간] 작성자명` 헤더 + 본문

#### 형식 B — 텍스트 파일

- 위 형식과 동일한 .txt 파일 경로 제공

### 선택 입력

- 채팅방명 (기본값: SP3S03 운영)
- 분류 필터 (품질만 / 설비만 / 전체)

## 실행 절차

### Step 1 — 입력 파싱

채팅 원문을 메시지 단위로 파싱한다.

각 메시지에서 추출:
- 날짜/시각
- 작성자명
- 본문
- 첨부 여부 ([이미지], [파일] 태그 보존)

대상 월에 해당하지 않는 메시지는 제외한다.

### Step 2 — 키워드 1차 필터링

각 메시지 본문에서 아래 키워드를 검색하여 후보를 선별한다.

**품질 키워드:**
불량, NG, 스크래치, 찍힘, 오염, 치수, 외관, 누락, 오삽, 미삽, 역삽, 이종, 파손, 크랙, 변형, 흠집, 도장, 용접, 검사, 클레임, 반품, 재검

**설비 키워드:**
설비, 고장, 수리, 정지, 에러, 알람, 센서, 실린더, 모터, 컨베이어, 지그, 유압, 공압, 전기, 누유, 파손, 마모, 교체, 보전, 점검, 비전, 카메라, 금형

키워드가 하나도 매칭되지 않는 메시지는 "기타"로 분류하거나 제외한다.

### Step 3 — 문맥 판정 (LLM 분류)

키워드 매칭된 후보 메시지를 읽고 아래 분류 체계에 따라 대/중/소를 판정한다.

#### 분류 체계

**대분류: 품질**

| 중분류 | 소분류 예시 |
|--------|-----------|
| 외관 | 스크래치, 찍힘, 오염, 도장불량 |
| 조립 | 오삽, 미삽, 역삽, 누락 |
| 검사 | NG, 재검, 판정불일치, 치수불량 |
| 클레임 | 고객클레임, 사내반품 |
| 원인미확정 | 키워드 매칭됐으나 분류 불가 |

**대분류: 설비**

| 중분류 | 소분류 예시 |
|--------|-----------|
| 고장 | 센서에러, 비전불량, 모터고장, 알람 |
| 수리 | 교체, 파손, 마모 |
| 정지 | 라인정지, 설비정지, 긴급정지 |
| 에러 | PLC에러, 통신에러, 프로그램에러 |
| 보전 | 점검, 예방보전, 윤활 |
| 기구 | 실린더, 지그, 치공구, 금형, 체결 |

**대분류: 기타**

| 중분류 | 소분류 예시 |
|--------|-----------|
| 자재/재고 | 재고부족, 결품, 투입지연 |
| 운영공지 | 생산계획, 생산취소, 일정변경 |
| 단순전달 | 인사, 공지, 확인요청 |

#### 분류 규칙

1. 키워드 1차 매칭으로 후보 선별
2. 본문 문맥을 읽고 대/중/소 확정
3. **다중 태깅 허용**: 한 메시지가 품질+설비 동시 해당 가능
4. 분류 불가 시 "기타" + 원문 보존
5. 단순 전달/공지/인사 등은 제외

### Step 4 — 집계

분류된 메시지를 아래 기준으로 집계한다:
- 월별 총 건수 (품질 / 설비 / 기타)
- 중분류별 건수
- 상위 반복 이슈 5건 (중분류 + 소분류 기준 빈도순)
- 중복 메시지 제거 (동일 작성자, 유사 내용, 같은 날짜)

### Step 5 — 보고서 출력

아래 형식으로 월별 보고서를 생성한다.

```
## SP3S03 채팅방 이슈 분석 — {YYYY}년 {M}월

### 요약
- 분석 기간: {YYYY}-{MM}-01 ~ {YYYY}-{MM}-{말일}
- 총 분석 메시지: N건
- 이슈 메시지: N건 (품질 N건, 설비 N건, 기타 N건)
- 상위 품질 이슈: {중분류}({N건}), {중분류}({N건}), ...
- 상위 설비 이슈: {중분류}({N건}), {중분류}({N건}), ...

### 품질문제 (N건)

| # | 날짜 | 작성자 | 중분류 | 소분류 | 내용 요약 |
|---|------|--------|--------|--------|---------|
| 1 | 03-05 | 김OO | 조립 | 오삽 | A품번 오삽 3건 발생 |

### 설비문제 (N건)

| # | 날짜 | 작성자 | 중분류 | 소분류 | 내용 요약 |
|---|------|--------|--------|--------|---------|
| 1 | 03-12 | 이OO | 고장 | 센서에러 | #90공정 센서 오작동 |

### 기타 (N건)
- 해당 시 간략 목록

### 3줄 인사이트
1. 이번 달 가장 빈번한 품질 이슈는 {X}이며 {원인/패턴} 확인 필요
2. 설비 {X} 관련 반복 발생으로 예방보전 점검 권장
3. {기타 패턴/추세 인사이트}
```

## 주의사항

- 이미지/첨부 메시지는 "[이미지]" 태그만 보존하고 분석 대상에서 제외 (v1 한계)
- 동일 이슈 반복 메시지는 중복 집계하지 않고 대표 1건만 표시
- 분류 판정이 애매한 경우 "원인미확정"으로 처리하고 원문을 보존
- 키워드만으로 확정하지 않고 반드시 문맥을 읽어 판정
- 이 스킬은 채팅 원문 수집 기능을 포함하지 않음 (v1은 수동 입력)
