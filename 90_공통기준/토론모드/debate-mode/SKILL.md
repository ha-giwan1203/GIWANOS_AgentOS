# 토론모드 (Debate Mode)

## 개요
Claude가 ChatGPT 브라우저 화면을 직접 읽고, 반박/질문을 생성하여 입력하는 반자동 토론 시스템.
사용자가 주제와 방향을 결정하고, Claude가 실행한다.

## 실행 모드

| mode | 설명 |
|------|------|
| start | 새 토론 시작. 주제 + 초기 프롬프트 필요 |
| continue | 기존 토론 이어가기. ChatGPT 탭이 열려있어야 함 |
| review | 토론 로그를 읽고 요약/분석 |

## 입력

| 항목 | 필수 | 설명 |
|------|------|------|
| 주제 | ✅ | 토론 주제 |
| 초기 프롬프트 | start 시 | ChatGPT에 보낼 첫 질문 |
| 턴 수 | 선택 | 기본 3턴 |
| 관점 | 선택 | Claude의 입장/역할 (예: 비판자, 현실검증자) |

## 실행 절차

### Step 1. 브라우저 준비
- ChatGPT 탭 확인 (없으면 열기)
- 로그인 상태 확인
- 새 대화 또는 기존 대화 선택

### Step 2. 프롬프트 입력
- 텍스트 입력 영역 찾기
- 프롬프트 입력 → Enter 전송

### Step 3. 응답 대기 + 읽기
- 응답 완료 대기 (polling)
- get_page_text로 전체 텍스트 추출
- 마지막 assistant 응답만 파싱

### Step 4. 반박 생성
- GPT 응답 핵심 주장 요약 (3줄)
- 반박 포인트 2~3개 도출
- "근거 + 대안" 구조로 작성
- 사용자에게 보여주고 승인 대기

### Step 5. 반박 입력
- 승인된 반박을 ChatGPT에 입력
- 응답 대기 → 읽기

### Step 6. 반복 또는 종료
- 설정된 턴 수까지 Step 3~5 반복
- 종료 시 전체 로그 저장

## 로그 저장
- 경로: 90_공통기준/토론모드/logs/
- 파일명: debate_YYYYMMDD_HHMMSS.md + .json
- 턴별 타임스탬프, 입력/출력, 오류 여부 기록

## 오류 대응
| 상황 | 대응 |
|------|------|
| 응답 미완료 | 10초 대기 후 재확인, 3회 실패 시 중단 |
| 텍스트 추출 실패 | read_page fallback |
| 로그인 만료 | 사용자에게 재로그인 요청 후 대기 |

## 금지사항
- ChatGPT API 사용
- 자동 로그인
- 사용자 승인 없이 반박 입력
- 이전 로그 덮어쓰기
