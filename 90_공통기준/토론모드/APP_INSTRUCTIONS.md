# 코워크 앱 자체지침

작성일: 2026-04-09
상태: ACTIVE
적용 대상: ChatGPT 코워크 앱 / 토론모드 / debate-mode

## 1. 목적

이 문서는 코워크 앱에서 GPT와 공동작업할 때 따를 앱 자체지침이다.
저장소 공통 규칙과 별개로, 코워크 화면 진입, 메시지 전송, 응답 대기,
분석, 최종 공유 방식에 대한 최소 기준을 정한다.

## 2. 기본 원칙

1. 코워크 앱은 보조 검토자로 사용한다.
2. 최종 기준 원본은 로컬 파일과 Git 반영 결과다.
3. 코워크 앱 응답만으로 완료 판정하지 않는다.
4. ERP/MES/Flow처럼 원천이 여러 개면 `ERP 우선, MES 보조, Flow 참고`를 유지한다.
5. 앱 대화보다 파일 산출물을 우선 남긴다.

## 3. 시작 전 확인

코워크 시작 전 아래를 먼저 본다.

1. `90_공통기준/업무관리/HANDOFF.md`
2. `90_공통기준/업무관리/STATUS.md`
3. `90_공통기준/업무관리/TASKS.md`
4. `90_공통기준/토론모드/ENTRY.md`
5. `90_공통기준/토론모드/CLAUDE.md`

## 4. 앱 사용 규칙

- 기존 대화 URL이 있으면 그 대화를 우선 재사용한다.
- 새 대화는 기존 대화 URL이 없을 때만 연다.
- 메시지는 브라우저 화면을 통해 전송한다.
- API 직접 호출로 우회하지 않는다.
- 사용자 확인 없이 중간에 멈추지 말고, 응답 수집 후 분석까지 이어간다.

## 5. 메시지 구성 규칙

코워크 앱에 보낼 메시지는 아래 4요소를 기본으로 한다.

1. 현재 작업 목적
2. 기준 파일/근거
3. 요청 항목
4. 출력 형식

## 6. 응답 해석 규칙

- `채택`: 바로 반영 가능한 의견
- `보류`: 근거는 있으나 추가 검토 필요
- `버림`: 현재 파일/근거와 맞지 않는 의견

응답은 반드시 위 3분류로 다시 정리해서 반영한다.

## 7. 금지 사항

- 코워크 앱 응답만으로 원본 데이터 수정
- 근거 파일 미확인 상태에서 구조 변경
- 수동 판정을 코드에 직접 하드코딩
- commit/push 전 Git 반영 내용 없이 GPT에 먼저 완료 보고

## 8. 종료 기준

다음 조건이 충족될 때 코워크 작업을 종료한다.

- 반영 파일이 존재한다.
- 필요한 경우 commit/push가 끝났다.
- `TASKS.md`, `HANDOFF.md` 등 상태 문서가 갱신됐다.
- 사용자에게 결과와 남은 리스크를 보고했다.
