# Round 1 — Gemini 본론 (Step 6-3, 2026-04-27)

## 핵심 결론
**해석 C 채택** + **Option B 채택 + 정규식 앵커링 강제**. P2-C·P3-E 동결 보류 유지. 재논의 트리거 명문화.

## 원문 전체

### 1. 의제0 (메타 모순)

**결론**: 해석 C 채택.

**근거**: 세션108의 '시스템 개선 영구 중단' 원칙은 '선제적인 구조 확장'을 금지한 것이지, '실무 산출을 물리적으로 가로막는 시스템의 오탐(False Positive) 사후 교정'까지 막는 족쇄가 아닙니다.

**권고 범위**: P2-B 1건에 대한 수정을 '예외적 허용'이 아니라 '실무 산출물 모드 가동을 위한 필수 유지보수'로 규정하여 논리적 일관성을 확보합니다.

**약점/리스크**: 필수 유지보수와 시스템 개선의 경계가 모호해질 수 있으므로, 이번 수정이 마지막 개입임을 TASKS.md에 재명시해야 합니다.

### 2. 의제1 (P2-B)

**결론**: Option B 채택 (정상 완료 신호 기반의 좁은 패턴 확장).

**근거**: 보안 탐지 수준을 낮추지 않으면서(Option C 배제), 비즈니스 스크립트 내부에 시스템 검증 로직을 하드코딩하는 유지보수 지옥(Option A 배제)을 피할 수 있는 유일한 타협점입니다.

**권고 범위 (정상 완료 신호 정의)**: 단순 스크립트명 매칭은 절대 불가합니다. 신호는 1) stdout 상의 명시적 성공 텍스트(예: "OAuth 200 OK", "Login Success") 또는 2) 해당 스크립트 실행 결과로 생성되어야만 하는 '특정 산출물 파일(토큰, xlsx 등)의 존재 여부' 등 물리적으로 확인 가능한 2가지 조건 내에서만 엄격하게 한정해야 합니다.

**약점/리스크**: evidence_mark_read.sh의 텍스트 파싱 로직이 복잡해져 새로운 쉘 스크립트 버그를 유발할 위험이 있습니다.

### 3. 의제2 (P2-C)

**결론**: 동결 보류 유지 (철거 및 삭제 금지).

**근거 (세션77 합의와의 트레이드오프)**: 세션77 당시에는 잦은 변경으로 인한 불안정성이 높아 nightly_capability_check 같은 최우선 안전망이 필수적이었습니다. 그러나 현재는 오히려 '안전망 과다'가 병목을 일으키고 있습니다. **상황이 변했으므로 지금의 최우선 안전망은 '검증 루프를 추가하는 것'이 아니라 '시스템 개입을 멈추는 것'입니다.**

**권고 범위**: 어떠한 운영 경로(schtasks 등)도 부여하지 않고 현재의 비활성 상태를 그대로 방치합니다.

**약점/리스크**: Silent Failure(조용한 실패)를 사전 방지하지 못하는 리스크를 안고 가야 하나, 이는 실무 효율을 위해 감수해야 할 계산된 비용입니다.

### 4. 의제3 (P3-E)

**결론**: 동결 보류 유지.

**근거**: Dead Surface(사문화된 코드)를 정리하는 작업 자체도 의존성 충돌 등 새로운 리스크를 수반하는 '시스템 구조 변경' 행위이므로 동결 원칙에 위배됩니다.

**권고 범위**: AGENTS_GUIDE.md에 명문화하지 않으며, 자동 호출 흐름에도 편입시키지 않습니다.

**약점/리스크 (재논의 트리거)**: 쓰레기 코드가 누적되는 기술 부채를 안게 됩니다. 향후 **실무 산출물에 치명적 결함(예: 잘못된 정산 엑셀, 코드 포맷 파괴)이 발생하여, 해당 에이전트(artifact-validator 등)의 부재가 직접적 원인으로 증명된 경우에 한해서만 재논의를 허용해야 합니다.**

### 추가 위험 1건 (정규식 앵커 누락으로 인한 우회 취약점)

GPT는 Option B에서 성공 신호와 묶어 패턴을 추가하라고 지적했으나 구현 리스크를 간과했습니다. evidence_mark_read.sh에 패턴을 추가할 때 **시작(^)과 끝($)을 명확히 묶는 정규식 앵커링 처리를 누락하면**, 악의적인 스크립트가 파일명 일부만 교묘하게 위장하여 시스템의 ok 마크를 획득하는 **심각한 보안 우회(Bypass) 취약점**이 발생할 수 있습니다. 좁은 패턴 추가 시 반드시 완전 일치(Exact Match) 로직을 강제해야 합니다.
