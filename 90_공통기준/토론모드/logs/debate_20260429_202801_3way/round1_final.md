## 최종 합의 (보강 반영)

### 결론 1줄
근본 원인은 시스템 설계 부하(45%) + 메타·운영 위생(30%) + 실물검증 루프 결함(15%) + 모델 한계(10%, 시스템에 흡수) + 기대치 정합성(5~10%)의 누적이며, **"목표 함수 오염"** (문제 해결 최적화 → 규칙 준수 최적화)으로 요약된다. 다음 행동은 세션 성격별 분기.

### 5축 분류 + 가중치 (확정)

| 축 | 가중치 | 핵심 |
|---|---|---|
| 1. 시스템 설계 부하 | 40~45% | prefix 1,070줄 + 표면 133개 + 응답 전 결정 분기 7+ |
| 2. 메타·운영 위생 | 25~30% | TASKS 598, incident 122, kernel 146h stale, 메타 커밋 65% |
| 3. **실물검증 루프 결함 (신규 축)** | 15~20% | 문서·로그·실행결과 확인 전 "성공" 선언 — dedupe 미동작·포트 착각·share_gate 정반대 |
| 4. 모델 한계 | 10~15% | long-context attention drift 실재하나 위 3축이 증폭. 시스템 설계로 일부 통제 가능 |
| 5. 기대치 정합성 | 5~10% | "알아서 진행" vs "확인 후 진행" 마찰 |

### 세션 성격별 우선순위 분기

- **긴급 실무 세션** (D0/MES/라인배치/정산 마감 있음): **D → A → C → B**
- **시스템 정비 세션** (사용자 명시 "정비/감량/리팩터"): **A → D → C → B**

### 정량 신호 3개 + 보너스 1개 (GPT 권고 반영 — 보강)

1. **첫 답변의 실물 지향성** — 첫 응답 내용 중 실제 Git 커밋·코드 수정·API 호출 등 '실행 계획' 비중 70% 이상. 미달 → 기대치+설계 부하.
2. **단일 세션 내 PASS 도달률** — **1개 이상의 커밋 또는 확인 가능한 실물 산출물 1개** + PASS 판정 도출. 미달 → 위생+검증 루프. (GPT 보강안 반영: 모든 세션이 커밋을 목표로 하지 않음 — 산출물 일반화)
3. **자기 참조 늪(Meta-Loop) 탈출** — 메타작업 커밋(TASKS·hook·스킬·CLAUDE.md) 0~1회. 초과 → 설계+목표 함수 오염.
4. **(보너스) 모델 한계 의심 조건** — 위 3개 모두 정상인데 답변이 얕음 → 그때만 모델 한계.
