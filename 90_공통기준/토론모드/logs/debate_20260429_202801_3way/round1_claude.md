- **함의**: 사용자가 단순 요청해도 모델이 메타 결정 토큰을 먼저 소비 → 본 작업 품질 저하

- **함의**: prefix 자체를 못 바꾸면 응답 품질 천장이 정해진다

## 반대 안 예상 약점

1. **GPT 반박 가능**: "측정 단위가 줄 수밖에 없어서 토큰 부하 실증 못 함. 실제 attention 저하 증거 없음." → 반론: 사용자 자가보고 "전부"가 정성 증거. 정량은 다음 세션 측정 protocol로 가능.
2. **Gemini 반박 가능**: "모드 프로토콜은 사고 사례에서 학습된 결과. 빼면 또 사고 재발한다." → 반론: 빼는 안 1·2 이후 incident 신규 0건/24h — 빼도 즉시 재발 안 함. 단, 관찰 기간 짧음.
3. **공통 반박**: "축 4(기대치)를 너무 가볍게 봤다 — 실제로 사용자 피로도가 가장 큰 변수일 수 있다." → 부분 수용. 옵션 D(매 세션 1줄 합의) 우선 권장에 이미 반영.
4. **자기 모순 가능**: 이 토론(3-way debate) 자체가 또 하나의 메타 작업 → 진단 행위가 진단 대상을 강화. 단, 사용자 명시 요청이므로 진행 정당.
