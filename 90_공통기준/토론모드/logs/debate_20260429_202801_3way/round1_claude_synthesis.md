## 진단 키워드: "목표 함수 오염"
시스템이 Claude의 사고 품질을 높이기보다 실수를 막는 울타리 위주로 진화 → Claude의 최적화 목표가 **문제 해결 → 규칙 준수**로 변질. 빼는 안 1+2 신설은 이를 인지한 부분 시정이지만 충분하지 않음.

## 다음 행동 — 세션 성격별 분기

### 긴급 실무 세션 (D0/MES/라인배치/정산 등 산출물 마감 있음)
순서: **D → A → C → B**
- D: 첫 줄에 "오늘 산출물 1줄 합의"를 박는다 ("오늘은 D0 morning 07:05/07:15 결과만 판정한다")
- A: 산출 전후 토막 시간에 위생 5분 (incident 1건 close, TASKS 정리 1건)
- C: 첫 답변 목표 일치율·실물검증 선행률·메타작업 비율 측정
- B: 세션 종료 후 별건 등록

### 시스템 정비 세션 (사용자가 "정비/감량/리팩터" 명시)
순서: **A → D → C → B**
- A: 운영 위생 1회 정리 (TASKS 598→200줄, incident 122→0, kernel 갱신, hook count 점검)
- D: 정비 1줄 목표 합의 ("오늘은 hook 51→40개 감축만 한다")
- C: 동일 측정
- B: 1주 관찰 후에도 무게가 남으면 구조 다이어트 (rules 5→3, 자동화 표면 133→100 등)
