1. **맥락 단절 리스크**: 1줄 검증(동의/이의/검증 필요 + 근거 1문장)은 본론과 달리 채팅방 누적 맥락 필요한가? 본론 원문을 API 호출 payload에 매번 동봉하면 맥락 재구성 가능한가?
2. **이점 실측 추정**: 웹 UI 단발 검증 왕복 8~15초(탭 navigate+입력+polling) vs API 2~5초. 라운드당 약 16~30초 감축. 실제 체감 여부.
3. **리스크**:

   - (b) 키 관리·비용·실패 fallback
   - (c) 토론방 로그 보존성 (검증 결과 공개 채널에서 사라짐)
4. **대안 비교**:
