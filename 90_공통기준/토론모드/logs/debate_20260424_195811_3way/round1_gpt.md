- Q3 incident 분석에서 auto_commit_state 반복 구조 문제 실증되면 그때 신규 의제 재상정. 지금은 종결

**착수 조건**: Q3 /auto-fix 결과에서 auto_commit_state 직접 원인 반복 확인 여부
**완료 기준**: TASKS/HANDOFF에 "조건부 격상 폐기, advisory 유지" 명시, 폐기 사유 기록
**검증 방법**:
- auto_commit_state.sh 동작 변경 없음 확인
- README/HANDOFF/TASKS 폐기 결정 기록 확인

## Q2. A안 채택 — 실증됨 라벨 엄격화
**근거**:
- "실증됨"이 설계 논리만으로 붙으면 하네스가 장식품 됨. 라벨 자체의 의미 상실
- 세션103 critic WARN은 세션당 1회 한정 봐준 것. 다음부터 강제 필요
- C안(3단계 분리)은 복잡도만 늘림. "증거 있음/없음"이면 충분

**착수 조건**: Q3 incident 분류 후, 하네스 규칙 수정이 단순 문서 개정인지 확인. 실행 흐름 변경 아닌 라벨 판정 기준 강화로 한정
**완료 기준**: critic-reviewer 또는 debate/share-result 판정 규칙에 문구 반영
- "실증됨 = 커밋 SHA, 파일 diff, incident_ledger, 실행 로그 중 최소 1개 실물 증거 필수"
- 증거 없으면 자동 "일반론" 강등
**검증 방법**: 샘플 2건 테스트
- SHA/incident 있는 주장 → 실증됨 유지
- 설계 논리만 있는 주장 → 일반론 강등

## Q3. A안 채택 — 즉시 /auto-fix 1회 실행
**근거**:

**착수 조건**: 세션105 토론 결론 기록 후 즉시 /auto-fix 1회만 실행. 자동 수정 금지, 분류 리포트만
