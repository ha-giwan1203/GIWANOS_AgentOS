- **의제 3**: D 자동 차단 + **HANDOFF에 에스컬레이션 로그 강제**(추적성 확보)
- **의제 4**: R1~R5는 영향력 체크리스트로 전환. R5는 **'물리적 백업'이 아닌 '논리적 롤백 스크립트' 존재 여부**가 기준
- **의제 5**: **모드 전환 시점(First Response) + 외부 데이터 쓰기(A/ERP) 직전**에만 상태 선언 헤더 노출
- **의제 6**: 시간 임계보다 **'프로세스 고착(Locking)' 우선** + **'데이터 정합성(Integrity) 훼손' 추가**

### 사각지대 지적
1. 의제 3 [실증됨]: GPT의 C 강제 승격은 **'Silent Update' 위험**을 간과. 사용자가 모르는 사이 시스템 변경. **승격 전 'C모드 전환 필요' 경고 선행 필수**.
2. 의제 4 [구현경로미정]: R1~R5가 단순 텍스트 나열이 될 위험. plan.md 체크박스가 아닌 **실제 코드 검증 단계(Pre-commit hook 등)와 연동** 방법론 부족.
3. 의제 5 [일반론]: '외부반영 A'에만 헤더를 붙이면 **사용자가 AI가 B/C 로직으로 판단 중인지 놓침**. 모드 전환 시점도 헤더 필요.
4. 의제 6 [환경미스매치]: GPT의 30초/60초는 **네트워크 지연이 잦은 제조 현장 ERP에서 너무 민감**. **트랜잭션 생존 여부**가 더 정확.

2. **설비 인터치 타임아웃**: 자동화 에이전트가 MES 설비 인터페이스(PLC 등) 건드리는 작업 시 **물리적 안전 사고 방지** 기준 추가
