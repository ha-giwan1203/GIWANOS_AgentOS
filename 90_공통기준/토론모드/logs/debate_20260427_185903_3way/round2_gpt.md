## 의제 3 — 자동 승격 차단

**권고**: B 분류 감지 ON, D 자동 진입 OFF (비대칭 설계 채택).

1. **D 자동 진입은 끄되 B 감지는 켠다** [실증됨]: Claude 출력에 "[B분류 감지] hook/settings/SKILL 절차 변경 가능성 있음, D 자동 진입 안 함, C 모드 plan-first로 처리" 형식 경고 라인.
2. **자동 D 끄면 위장 위험 있다 → 보완은 C 강제 승격** [일반론]: 시스템 파일 경로(.claude/hooks, settings, commands, agents, SKILL.md Step 변경, CLAUDE.md, gate 판정 분기) 중 하나라도 수정 대상이면 **D가 아니라 C**.
3. **본 저장소 기준 D보다 E/C 분리가 더 중요** [실증됨]: 세션110 D0 중복 7건 ERP 잔존 + DELETE API 발견 사례, 세션115 d0-plan 첨부 가드 사례. 토론보다 **외부 반영 가드**가 우선.

## 의제 4 — R1~R5 형식주의

**권고**: R1~R5는 C 전용 + **반증형 영향반경**으로 재정의 + R5에 ERP/MES 잔존 데이터·dry-run 필수 포함.

1. **그대로 두면 서류 채우기 게임** [일반론]: 나쁜 예 — "R1.파일A / R2.있음 / R3.있음 / R4.없음 / R5.중". 의미 없음.
2. **반증형 질문 도입** [구현경로미정]:
   - R1: 이 파일이 진짜 원인인가? 수정 없이 해결 가능한가?
   - R2: 어떤 hook/skill/command가 즉시 동작이 바뀌는가? 외부 ERP/MES 영향?
   - R3: grep 역참조로 어디서 호출되는가? 같은 포트/프로필/경로 공유 자동화?
   - R4: incident_ledger·최근 커밋에 같은 패턴? 세션107/110/115 D0 계열과 같은가?
   - R5: 잔존 데이터? 되돌리기 가능? 삭제/업로드/등록 같은 비가역?
3. **R5는 ERP/MES 환경에서 가장 중요** [실증됨]: D0 중복 18건 → 7건 정리 사례. R5 필수 질문: "실패 시 ERP/MES/SmartMES/파일서버/Z드라이브에 잔존 데이터? dry-run/parse-only/건수확인/삭제·복구 경로 있는가?"
4. **R1~R5는 C 전용** [일반론]: 모든 모드에 강제하면 사고 죽음. A=간단검증, B=개선제안 단계만, C=필수, D=하네스, E=시스템수정 전환 시만.

## 의제 5 — 응답 헤더 [모드] 표기

**권고**: 모든 응답 표기 금지 + **B/C/D/E + ERP/MES 외부 반영 포함 A**에만 1줄 헤더.

1. **모든 응답 표기는 노이즈** [일반론]: "사용자 스타일상 못 견딘다, 업무 속도 죽임".
2. **고위험 작업에는 헤더 필요** [실증됨]: settings/hook 변경 후 세션 재시작·완료 판정 강한 규정 → 시스템 수정은 일반과 다르게 다뤄야.
3. **헤더 1줄, 길게 쓰지 말 것** [일반론]: 예 — `[모드: E 장애 복구 / 기준: D0 업로드 차단 / 실행: 최소복구만]`, `[모드: C 시스템 수정 / 대상: SKILL.md Step 변경 / 실행 전 plan 필요]`.
4. **세션115는 헤더 필요 케이스** [실증됨]: 처음부터 `[모드: E 장애 복구 / 위험: 첨부 파일 vs Z드라이브 자동탐색 충돌 / 실행 전 입력 소스 확인]`이 박혔어야 자동 실행 안 탔을 가능성 큼.

## 의제 6 — 모드 E 정량 기준

**권고**: "이미 시작 AND (시간 임계 OR 외부 응답 부재 OR 잔존 데이터 위험 OR 입력 소스 충돌)". ERP/MES 건수 불일치는 시간 무관 즉시 E.

1. **E 정량화 가능** [일반론]: 핵심 — "이미 시작된 실무 실행이 멈췄고, 시간 또는 외부 시스템 응답 문제로 정상 산출/등록/업로드 불가능".
2. **OR 조건 (Claude 안 + GPT 추가)** [실증됨]:
   - 시간 임계 초과
   - 외부 응답 부재
   - **잔존 데이터 위험** (GPT 추가 — 응답 200인데 중복 등록, selectList 성공인데 잘못된 파일, 삭제 성공인데 SmartMES 캐시)
   - **입력 소스 충돌** (GPT 추가 — 첨부 vs Z드라이브, OAuth 통과인데 다른 프로필)
3. **시간 임계는 작업별 차등** [환경미스매치]:
   - UI 요소 대기: 30초 → 경고
   - ERP ajax/selectList: 60초 → E 후보 (세션115 d0-plan 20→60s 상향 실증)
   - OAuth/login redirect: 60초 또는 재진입 1회 이상 → E
   - 업로드 후 저장 응답: 60초 → E
   - 스케줄러 정시: 예정 시각 + 10분 무로그 → E
   - 외부 시스템 등록 후 건수 불일치: 시간 무관 즉시 E
4. **"이미 시작" 조건이 B와 E를 가른다** [일반론]: "분석해봐"=B, "고쳐라"=C, "지금 멈췄다"=E, "스케줄러 로그 없다"=E, "자동 실행이 다른 파일 잡을 수 있다"=E 후보.
5. **ERP/MES 잔존 데이터는 시간 무관 즉시 E** [실증됨]: D0 중복 사례.
