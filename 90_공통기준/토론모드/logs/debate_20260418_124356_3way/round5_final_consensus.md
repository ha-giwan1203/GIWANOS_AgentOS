## 최종 3자 합의안

### Claude Code = 워크플로우 설계 + 실행 + 무결성 검증
- 도구 호출 시점·입력·출력·검증 절차 모두 Claude가 설계
- GPT/Gemini 입력은 무결성 검증 후 채택 (Gemini 조건)
- MCP/hooks/skill로 자동 호출
- 실행·검증·Git 반영 단일 주체

### GPT = 추론·창의·토론 입력 (자가신고 5개)
1. 아이디어 다변화
2. 반대논리 토론
3. 초안→설득문 리라이트
4. 복잡한 개념 설명
5. 빠른 비교·의사결정

### Gemini = 거시적 분석·통합·멀티모달 입력 (자가신고 5개)
1. 법률·논문 수천 매 전수 분석
2. 장시간 영상 콘텐츠 기획
3. Workspace(Gmail/Drive/Sheets) 자동 관리
4. 수십만 줄 레거시 코드 구조 매핑
5. 교재/강의 영상 통합 학습

### 통합 방식 (외부 실증 인용)
- MCP: Claude Code에 Gemini MCP 등록 (Composio) → 자동 호출
- 기존 토론모드: GPT/Gemini 채팅방 직접 호출 (검증·토론용)

### 외부 실증
- Triple Stack 5x 생산성 (Netalith 2026)
- 한국 사용자: 탐색=Gemini, 정리=Claude, 사고=GPT (gpters)
