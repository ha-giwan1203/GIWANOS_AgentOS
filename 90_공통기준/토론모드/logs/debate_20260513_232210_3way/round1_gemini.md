# Round 1 — Gemini 본론

## 결론
콘솔 fetch PoC 폐기에 동의. 기존 UI 자동화 로직을 이벤트 기반 감시와 텍스트 추출 단발화로 전환하여 라운드당 약 30~50초 가속 달성이 가장 실무적.

## 가속화 분석 4항목

### 1) 폴링 단축 및 추출 최적화 (라운드당 약 30~50초 절감)
- 중지 버튼 감시 주기 3/5/8초 → **0.5/1/2초** 단축
- 생성 중 실시간 본문 추출 폐기 → **중지 버튼 사라진 직후 1회 집중 추출** (One-shot extraction)
- 불필요한 DOM 탐색 시간 라운드당 20초 이상 절감
- 응답 생성 속도에 맞춰 즉시 다음 단계 전이

### 2) 백그라운드 Throttling 우회 — bringToFront 최적화
- `bringToFront=true`를 무분별 호출 X
- **듀얼 트리거 방식**:
  - 프롬프트 전송 직후 1회
  - 평균 응답 길이 고려한 임계 시간(예: 30초) 경과 후 1회
- 탭 활성 상태 강제 유지로 브라우저 실행 지연 최소화

### 3) SKILL.md REFERENCE vs hook gate — 역할 분담
- **SKILL.md**: 가속화의 '이유' + 방법론 명시
- **hook gate**: 폴링 주기·타임아웃 값 실질 제어 '실행 장치'
- 분담으로 지침 위반·로직 회귀 효과적 차단

### 4) 가속화 시 안정성 누락 위험 (회귀 방지)
- **텍스트 유실 race condition**: 중지 버튼 사라졌으나 DOM 최종 텍스트 미렌더링 짧은 찰나에 추출 시 끝부분 잘림
- **Cloudflare 매크로 탐지**: 고빈도 폴링이 보안망에 매크로 신호로 분류 가능
- 인증 챌린지 화면 발생 시 가속 즉시 중단 + UI fallback 전환 **안전 차단(Safe-cutoff) 로직 병행 필수**

제안 4항목으로 plan.md 작성. 특히 1회 집중 추출 + 인증 챌린지 감지 시 즉시 fallback 구현 초안 우선 검토.
