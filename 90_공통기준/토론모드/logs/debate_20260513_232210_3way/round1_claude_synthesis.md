# Round 1 — Claude 종합 + 최종 합의안

## 합의 요약 (Round 1 만장일치)

**원래 PLAN(콘솔 fetch 가속) → 폐기**. 대체 합의 = **폴링 단축 축소안 (UI 자동화 가속)**.

근거 3축:
1. **ToS·계정 리스크**: OpenAI/Google 공식 약관 모두 automated/programmatic access 명시 금지. 같은 origin이라도 비공식 백엔드 직접 호출 = 정지 리스크. 제조 현장 프로젝트 연속성 위협
2. **가속폭 한계**: 라운드 2~3분 중 모델 생성 50~70% 차지. 콘솔 fetch 가속폭 = 라운드당 20~60초 절감 수준 (분 단위 절감 불가)
3. **구현 부담**: Gemini 백엔드는 protobuf·XSRF·구조 빈번 변경으로 PoC 부담 매우 높음

## 채택 합의안 — 4항목

### A. 폴링 단축 + 1회 집중 추출 (라운드당 20~50초 절감)
- 중지 버튼 감시 주기: 현행 3/5/8초 → **1/2/3초** (Gemini 0.5/1/2 + GPT 1초 이상 보수 = 1/2/3초로 절충)
- 생성 중 실시간 본문 추출 폐기 → 중지 버튼 사라진 직후 **1회 집중 추출(One-shot)**
- 텍스트 유실 race condition 방지 위해 One-shot 직후 **2회 연속 동일 검사** 유지 (현행 안정성 로직 보존)

### B. bringToFront 듀얼 트리거 최적화
- 무분별 호출 폐기
- 트리거 시점:
  1. 프롬프트 전송 직후 1회
  2. 평균 응답 시간 30초 경과 후 1회
- 응답 진행 중 추가 호출 금지 (탭 활성 강제 유지)

### C. SKILL.md ↔ hook gate 역할 분담
- **SKILL.md**: 가속화 방법론·이유·트리거 시점 명시 (문서)
- **hook gate**: 폴링 주기·타임아웃·Safe-cutoff 값 실질 제어 (실행)
- 회귀 차단: 지침과 실행 분리

### D. Safe-cutoff 회귀 보험 (필수 — GPT 추가 보강 반영)
- **인증 챌린지 화면 감지 즉시 가속 중단** + UI fallback 전환
- 대화 기록 미저장 감지 시 즉시 중단
- 사용량 경고·임시 제한·보안 알림 발생 시 즉시 중단
- 1회라도 위 신호 감지 시 세션 전체 가속 모드 비활성화
