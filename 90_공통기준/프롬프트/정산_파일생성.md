# 정산 파일 생성 프롬프트 템플릿

## 프롬프트 본문

```
{월}월 조립비 정산 파일을 새로 만들어줘.

입력 파일:
- GERP 실적: {GERP_파일경로}
- 구ERP 실적: {구ERP_파일경로}
- 기준정보 (단가): {기준정보_파일경로}

출력 파일명: 정산결과_{월}월.xlsx
저장 위치: 05_생산실적\조립비정산/

라인 목록: SD9A01, SP3M3, ANAAS04, DRAAS11, HASMS02, HCAMS02, WAMAS01, WABAS01, WASAS01, ISAMS03
대상 업체코드: 0109 (대원테크)
```

---

## 사용 시점

- 매월 정산 시작 시점 (월초~중순)
- 이전 달 정산 파일이 없거나 구조를 새로 잡아야 할 때
- 스크립트(`생성_정산결과_XX월.py`)를 새로 생성해야 할 때

## 필수 입력

| 변수 | 설명 | 예시 |
|------|------|------|
| `{월}` | 정산 대상 월 (2자리) | `02` |
| `{GERP_파일경로}` | GERP 실적현황 엑셀 절대경로 | `05_생산실적\조립비정산/04_실적데이터/실적_GERP_실적현황.xlsx` |
| `{구ERP_파일경로}` | 구ERP 실적 엑셀 절대경로 | `05_생산실적\조립비정산/04_실적데이터/실적_구ERP_XX월.xlsx` |
| `{기준정보_파일경로}` | 기준정보(단가) 엑셀 절대경로 | `05_생산실적\조립비정산/01_기준정보/기준정보_단가.xlsx` |

## 기대 출력

- `정산결과_{월}월.xlsx` — 시트 구성:
  - `00_정산집계`: 라인별 GERP금액 / 구ERP금액 / 차이 요약
  - 라인별 시트 10개 (SD9A01, SP3M3, …): 품번별 주야간 수량·금액 상세
  - `차이분석`: GERP vs 구ERP 차이 원인 분석
  - `미매핑`: 기준정보에 없는 품번 목록
- 사용 스크립트: `assembly-cost-settlement` 스킬 또는 `생성_정산결과_{월}월.py`

## 관련 규칙

- GERP: 주야구분 컬럼 값 기준 (정상=주간, 추가=야간)
- 구ERP: LOT B=야간, 나머지=주간
- SP3M3 야간 고정단가: 170원
- SD9A01 야간가산 조건: 단가 ≤ 500원
- Usage=2 품번: 수량 2배 환산

## 에이전트 전환 가이드

### 전환 기준
이 프롬프트가 **3회 이상 반복** 사용되면 서브에이전트(`assembly-cost-settlement` 스킬 또는 전용 파이프라인)로 전환 권장.

### 변수화 대상
| 변수 | 현재값 예시 | 에이전트 파라미터명 |
|------|------------|----------------|
| `{월}` | `02` | `month` |
| `{GERP_파일경로}` | `04_실적데이터/GERP_실적현황_20260311.xlsx` | `gerp_file` |
| `{구ERP_파일경로}` | `04_실적데이터/구ERP_실적현황_20260311.xlsx` | `olderp_file` |
| `{기준정보_파일경로}` | `01_기준정보/기준정보_라인별정리_최종_V1_20260316.xlsx` | `master_file` |

### 에이전트 MD 변환 예시
```yaml
---
name: settlement-generator
description: 월별 조립비 정산 파일 생성. 입력파일 3개 → 정산결과 xlsx 자동 생성.
tools: Bash, Read, Write
model: sonnet
---
파라미터: month, gerp_file, olderp_file, master_file
실행: python step1_파일검증.py && python step2_gerp처리.py && ... && python step7_보고서.py
검증: step6_검증.py 자동 실행 후 PASS 확인
```
