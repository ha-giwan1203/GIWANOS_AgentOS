# 라인배치 검증 프롬프트 템플릿

## 프롬프트 본문

```
품번-라인배정 정합성 검증해줘.

기준 품번 파일: {기준품번_파일경로}
라인배정 파일: {라인배정_파일경로}

검증 항목:
- 기준 품번 대비 누락 품번
- 오기입 (라인코드 불일치)
- 컬러코드 차이
- 라인 미배정 품번

결과는 판정 기준과 함께 보고해줘.
```

---

## 사용 시점

- 라인배치 입력 작업(`line-batch-management` 스킬) 완료 후 결과 검증 시
- 기준 품번 파일이 갱신된 후 ERP/GERP 배정과 비교할 때
- 월 마감 전 라인배정 정합성 점검 시

## 필수 입력

| 변수 | 설명 | 예시 |
|------|------|------|
| `{기준품번_파일경로}` | 기준 품번 목록 엑셀 절대경로 | `10_라인배치/기준품번_SP3M3.xlsx` |
| `{라인배정_파일경로}` | ERP/GERP에서 추출한 라인배정 엑셀 절대경로 | `05_생산실적\조립비정산/실적_GERP_품번별_라인코드추출.xlsx` |

## 기대 출력

| 판정 항목 | 내용 |
|---------|------|
| 누락 | 기준 품번 중 라인배정 없는 품번 목록 |
| 오기입 | 라인코드 불일치 품번 및 현재값/기대값 |
| 컬러코드 차이 | 색상 분류 불일치 목록 |
| 미배정 | 라인코드 공란인 품번 목록 |
| 종합 판정 | PASS / FAIL (누락·오기입 0건이면 PASS) |

## 관련 스킬

- `line-mapping-validator`: 자동화된 정합성 검증 실행
- `line-batch-management`: 라인배치 입력 자동화 (검증 전 단계)

## 에이전트 전환 가이드

### 전환 기준
이 프롬프트가 **3회 이상 반복** 사용되거나 월 마감 루틴에 포함되면 서브에이전트로 전환 권장.
`line-mapping-validator` 스킬이 이미 이 역할 수행 → 스킬 우선 사용.

### 변수화 대상
| 변수 | 현재값 예시 | 에이전트 파라미터명 |
|------|------------|----------------|
| `{기준품번_파일경로}` | `10_라인배치/기준품번_SP3M3.xlsx` | `master_file` |
| `{라인배정_파일경로}` | `05_생산실적\조립비정산/실적_GERP_품번별_라인코드추출.xlsx` | `assignment_file` |

### 에이전트 MD 변환 예시
```yaml
---
name: line-mapping-checker
description: 기준 품번 vs 라인배정 정합성 자동 검증. 누락·오기입·미배정 판정.
tools: Read, Grep, Glob, Bash
model: haiku
---
파라미터: master_file, assignment_file
출력: 누락/오기입/컬러코드차이/미배정 + 종합 PASS/FAIL
```
