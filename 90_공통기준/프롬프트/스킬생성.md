# 스킬 생성 프롬프트 템플릿

## 프롬프트 본문

```
새 스킬을 만들어줘.

스킬명: {스킬명}
목적: {스킬_목적}
트리거 키워드: {트리거_키워드_목록}
등급: {A|B|C}

자동화할 절차:
{절차_단계별_설명}

입력: {필수_입력값}
출력: {기대_출력물}

실패 조건: {실패로 간주할 상황}
중단 기준: {즉시 중단해야 하는 조건}
검증 항목: {완료 후 확인할 값}
되돌리기: {원복 절차 — 읽기 전용이면 "없음"}

참고할 기존 스킬: {참고_스킬명} (없으면 생략)
저장 위치: 90_공통기준/스킬/{스킬명}/SKILL.md
```

---

## 사용 시점

- 반복되는 수동 작업을 Claude가 자동으로 수행하도록 스킬로 분리할 때
- 기존 스킬의 기능을 확장하거나 수정해야 할 때
- 새로운 업무 자동화 요구가 발생했을 때

## 필수 입력

| 변수 | 설명 | 예시 |
|------|------|------|
| `{스킬명}` | 스킬 식별자 (영문, 하이픈 구분) | `mes-production-upload` |
| `{스킬_목적}` | 스킬이 해결하는 문제 한 줄 요약 | `MES 생산실적 자동 업로드` |
| `{트리거_키워드_목록}` | 사용자가 말하면 이 스킬을 써야 하는 단어/문장 (5개 이상 권장) | `실적 업로드, MES 업로드, 생산실적 입력` |
| `{등급}` | A(실행형: 외부 시스템 변경), B(파일수정형), C(분석형: 읽기 전용) | `A` |
| `{절차_단계별_설명}` | 자동화할 작업 흐름 (순서 포함) | `1. BI 엑셀 열기 → 2. 날짜 필터 → 3. API POST` |
| `{필수_입력값}` | 스킬 실행에 필요한 파라미터 | `대상 날짜, BI 파일 경로` |
| `{기대_출력물}` | 스킬 완료 후 결과물 | `업로드 성공/실패 보고` |
| `{참고_스킬명}` | 구조를 참고할 기존 스킬 (선택) | `assembly-cost-settlement` |

## 기대 출력

- **폴더형 구조**: `90_공통기준/스킬/{스킬명}/SKILL.md`
- **SKILL.md 필수 구조**:
  - frontmatter: `name`, `description`, `grade`, `version`, `author`, `last_updated`
  - 본문 섹션: 목적 / 트리거 / 입력 / 출력 / 실행 절차 / 실패 조건 / 중단 기준 / 검증 항목 / 되돌리기 방법
- **패키지** (선택): `skill-creator-merged.skill` 내 `scripts/package_skill.py`로 `.skill` ZIP 생성 가능
- `.skill` 파일은 저장만으로 자동 활성화되지 않음 — settings.json 등록 또는 수동 적용 필요

## SKILL.md frontmatter 형식

```yaml
---
name: {스킬명}
description: {한 줄 설명 — 발동 조건 포함}
grade: {A|B|C}
version: 1.0
author: {작성자}
last_updated: {YYYY-MM-DD}
---
```

> grade 분류: A(실행형) — 외부 시스템 상태 변경 / B(파일수정형) — 로컬 파일 변경 / C(분석형) — 읽기 전용

## 실패계약 4섹션 (필수)

SKILL.md 본문에 반드시 포함:

| 섹션 | 내용 |
|------|------|
| 실패 조건 | 실패로 간주할 상황 (외부 시스템/API/UI 불일치 등) |
| 중단 기준 | 즉시 중단해야 하는 조건, 부분 반영 상태에서 멈춰야 하는 조건 |
| 검증 항목 | 완료 후 반드시 확인할 값, 원본/결과 비교 기준 |
| 되돌리기 방법 | 원복 절차 (읽기 전용 작업이면: "읽기 전용 작업이므로 되돌리기 없음") |

## 완료 판정

스킬 생성 후 PASS 선언 전 반드시 확인:
1. Git 실물 존재 확인 (`90_공통기준/스킬/{스킬명}/SKILL.md` 파일 존재)
2. TASKS.md 비충돌 (기존 작업과 상태 충돌 없음)
3. frontmatter 필수 필드 6개 모두 기재
4. 실패계약 4섹션 모두 비어있지 않음

> 위 4개 미충족 시 PASS 금지. "문서 생성 완료"를 조기 선언하지 않는다.

## 관련 스킬 · 참조 문서

- `skill-creator` (파일: `skill-creator-merged.skill`) — 스킬 설계·수정·평가·패키징 전담
- `SKILL_TEMPLATE.md` (`90_공통기준/스킬/SKILL_TEMPLATE.md`) — 현행 SKILL.md 빈 양식
- `skill-standards.md` (`skill-creator-merged.skill` 내 `references/skill-standards.md`) — description 3파트 구조, 출력 표준, 버전 관리 규칙

## 주의

- 반복 절차는 프로젝트 문서가 아니라 스킬로 분리 (CLAUDE.md 원칙)
- 스킬명은 영문 소문자·하이픈 형식 유지
- 전역 파일에 프로젝트 전용 규칙 누적 금지

## 에이전트 전환 가이드

### 전환 기준
스킬 생성 요청이 **3회 이상** 반복되거나 표준 패턴이 확립되면 에이전트로 전환 권장.
`skill-creator` 스킬이 이미 이 역할 수행 → 스킬 우선 사용.

### 변수화 대상
| 변수 | 현재값 예시 | 에이전트 파라미터명 |
|------|------------|----------------|
| `{스킬명}` | `mes-production-upload` | `skill_name` |
| `{스킬_목적}` | `MES 생산실적 자동 업로드` | `purpose` |
| `{트리거_키워드_목록}` | `실적 업로드, MES 업로드` | `trigger_keywords` |
| `{절차_단계별_설명}` | `1. BI 엑셀 열기 → 2. 날짜 필터 → 3. API POST` | `procedure_steps` |

### 에이전트 MD 변환 예시
```yaml
---
name: skill-scaffolder
description: 새 스킬 SKILL.md 초안 자동 생성. 트리거·파라미터·절차 구조화 후 90_공통기준/스킬/{skill_name}/ 저장.
tools: Read, Write, Glob
model: sonnet
---
파라미터: skill_name, purpose, trigger_keywords, procedure_steps, reference_skill
출력: 90_공통기준/스킬/{skill_name}/SKILL.md
```
