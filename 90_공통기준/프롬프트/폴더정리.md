# 폴더 정리 프롬프트 템플릿

## 프롬프트 본문

```
{폴더경로} 폴더의 파일을 분석하고 정리 방안을 제안해줘.

분류 기준:
- 현재 운영 중인 파일 (최신본, 기준파일)
- 구버전/중간본 → _구버전/ 또는 _삭제예정/ 이동 후보
- 중복 파일 (내용 동일, 파일명만 다른 것)
- 임시 파일 (probe, temp, test 등)

출력:
1. 현황 요약 (총 파일 수, 유형별 분포)
2. 정리 제안 목록 (파일명, 현재위치 → 제안위치/처리방안)
3. 즉시 삭제 가능한 파일 목록 (확인 후 처리)

실제 이동/삭제는 확인 후에 진행해줘.
```

---

## 사용 시점

- 폴더 내 파일이 누적돼 구버전/최신본 구분이 어려울 때
- 작업 완료 후 임시 파일·중간본 정리 시
- 정기 파일 정리 (월 마감 후 등)

## 필수 입력

| 변수 | 설명 | 예시 |
|------|------|------|
| `{폴더경로}` | 정리할 폴더 절대경로 | `C:\Users\User\Desktop\업무리스트\05_생산실적\조립비정산` |

## 기대 출력

- 현황 테이블: 파일명 / 수정일 / 크기 / 유형 판정
- 정리 제안: 운영유지 / _구버전 이동 / _삭제예정 이동 / 즉시삭제
- 이동/삭제 실행 전 사용자 확인 요청

## 주의

- 원본 파일 직접 수정·삭제는 명시 요청 시에만 실행
- 기준파일·마스터파일은 임의 이동하지 않음
- 파일명, 경로, 식별값은 사용자 확인 없이 변경하지 않음

---

## 에이전트 전환 가이드

### 전환 기준
폴더 정리가 **월 마감 루틴의 일부**로 정기화되면 서브에이전트로 전환 권장.
임시파일 패턴(`_probe`, `_temp`, `v2~v9`)이 반복 발생하면 전환 효과 큼.

### 변수화 대상
| 변수 | 현재값 예시 | 에이전트 파라미터명 |
|------|------------|----------------|
| `{폴더경로}` | `C:\Users\User\Desktop\업무리스트\05_생산실적\조립비정산` | `target_dir` |

### 에이전트 MD 변환 예시
```yaml
---
name: folder-cleaner
description: 폴더 파일 분석 후 운영/구버전/임시 분류 제안. 실제 이동은 사용자 확인 후 수행.
tools: Glob, Bash, Read
disallowedTools: Write, Edit
model: haiku
---
파라미터: target_dir
출력: 1.현황요약 2.정리제안(운영유지/구버전이동/삭제예정) 3.즉시삭제후보
규칙: 기준파일·마스터파일 임의 이동 금지, 사용자 확인 후에만 실행
```
