## 적용 영역

ERP/MES 자동화 중 **외부 시스템 등록·삭제·전송 후 잔존 데이터** 발생 가능 영역. 본 프로토콜은 직접 정정이 어렵거나 ERP↔MES 비대칭 잔존이 알려진 영역에만 적용.

## 절차 (3단계)

### 1단계 — 잔존 식별
1. **사고 인지 시점 1줄 기록**: ERP 등록 N건 + MES 응답 코드 / 사용자 첨부 파일 vs 자동 탐색 차이 / 응답 200인데 중복 의심
2. **ERP 측 잔존**: 등록 화면 재조회 또는 등록 API 응답 SHA·REG_NO 수집
3. **MES 측 잔존**: 본 영역의 조회 API 호출 (D0 계열은 `POST http://lmes-dev.samsong.com:19220/v2/prdt/schdl/list.api`) — `sewmacLabelScanQty` 필드가 ERP `REG_NO` 매핑임을 명심 (필드명 misleading, 세션110 식별)

### 2단계 — 청소 가능 여부 판정
| 조건 | 청소 시도 | 잔존 결정 |
|------|----------|----------|
| ERP 삭제 API 존재 + MES 동기화 가능 | 실행 | — |
| ERP 삭제 가능 / MES 삭제 불가 | ERP만 청소 + MES 잔존 보고 | MES 데이터는 그대로 두기 (사용자 결정) |
| 외부 시스템 응답 부재 (timeout) | 청소 시도 안 함 | 사용자 보고 + 마스터 정보 검증 후 결정 |

### 3단계 — 실행 (D0 계열 표준)
- **dry-run 우선**: 본 영역 표준 도구 `.claude/tmp/erp_d0_dedupe.py --dry-run`
- 결과 검토 후 `--execute` 전환
- 삭제 후 reverse verification: ERP `selectList` 재조회 + MES `list.api` 재조회

## PLC 인터치 안전 기준 (참고용)

**현재 저장소에 직접 PLC 호출하는 자동화는 없다.** 모든 MES 통신은 SmartMES API(HTTP) 경유. 본 항목은 **장래 자동화 도입 시 안전 기준**으로 보관.

장래 도입 시 적용 원칙:
1. **물리 동작 제어 자동화 금지**: PLC를 통한 모터·실린더·로봇 직접 제어는 본 저장소 자동화 범위 외. 사용자 수동 조작이 표준.
2. **조회 전용 인터페이스만 허용**: PLC tag 조회·상태 모니터링은 가능. write 명령은 사용자 명시 승인 필수.
3. **Interlock 회로 우회 금지**: 안전 회로(Interlock)는 절대 우회하지 않는다. 자동화 도구는 안전 회로 통과 후 진입.
4. **응답 timeout**: PLC 응답 timeout은 통신 단계 100ms 표준. 초과 시 즉시 모드 E 진입 + 사용자 통지.
