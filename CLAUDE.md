# 업무리스트 프로젝트

## 프로젝트 목적
자동차 부품 제조업(삼송 G-ERP) 업무 자동화 저장소.
생산·라인배치·정산·보고 관련 문서와 데이터를 구조화하고 반복 업무를 자동화한다.

## 프로젝트 구조
> 상세: `README.md` 참조. 핵심 폴더만 아래 기재.

- 05_생산실적/조립비정산: 정산 파이프라인 (도메인 CLAUDE.md 있음)
- 10_라인배치: 라인배정 자동화 (도메인 CLAUDE.md 있음)
- 90_공통기준: 스킬, 프롬프트, 업무관리, 토론모드 (도메인 CLAUDE.md 있음)
- 98_아카이브: 완료/구버전 보관
- 99_임시수집: 미분류 임시 파일

## 상태 원본 단일화 규칙
작업의 완료/미완료/진행중/차단 상태는 **TASKS.md에만 기록한다.**
- STATUS.md: 운영 요약, 재개 위치만. 독립적 상태 판정 금지
- HANDOFF.md: 세션 변경사항과 다음 AI 액션만. 상태 선언 금지
- 판정 우선순위: TASKS.md → STATUS.md → HANDOFF.md → CLAUDE.md → Notion

## 업무 원본 계층

| 계층 | 기준 원본 | 용도 |
|------|----------|------|
| 1차 (현업) | `90_공통기준/업무관리/업무_마스터리스트.xlsx` | 업무 일정, 마감일, 우선순위 |
| 2차 (AI) | `TASKS.md` / `STATUS.md` / `HANDOFF.md` | AI 작업 상태, 인수인계 |

충돌 시: 현업 일정 → xlsx 우선, AI 작업 상태 → TASKS.md 우선

## AI 인수인계 순서
1. `TASKS.md` — 상태 원본 (1순위)
2. `STATUS.md` — 재개 위치·주의사항
3. `HANDOFF.md` — 직전 세션 메모
4. 루트 `CLAUDE.md` → 도메인 `CLAUDE.md`

## 도메인 진입 프로토콜
사용자 요청에 도메인 키워드 포함 시, **도메인 문서를 먼저 읽는다. 임의 실행 금지.**

| 키워드 | 도메인 문서 경로 |
|--------|-----------------|
| 토론 / 토론모드 / GPT토론 / 공동작업 | `90_공통기준/토론모드/CLAUDE.md` |
| 정산 / 조립비 / settlement | `05_생산실적/조립비정산/CLAUDE.md` |
| 라인배치 / OUTER / ERP 라인배정 | `10_라인배치/CLAUDE.md` |
| MES 업로드 / 실적 업로드 / 실적 올려 | `90_공통기준/스킬/production-result-upload/SKILL.md` |
| 일상점검 / ZDM / 점검 입력 | `90_공통기준/스킬/zdm-daily-inspection/SKILL.md` |

## 작업 종료 시 상태 갱신
1. `TASKS.md` — 완료 이동, 신규 항목 추가
2. 도메인 `STATUS.md` — 재개 위치 기록
3. `HANDOFF.md` — 세션 변경사항·다음 AI 할 일
4. Notion — 필요 시 TASKS 기준 단방향 반영

## 공통 작업 원칙
- 기준 파일과 마스터 파일을 우선 존중한다
- 접두어, 코드, 식별값은 사용자 확인 없이 임의 제거하지 않는다
- 애매한 경우 확정하지 않고 판정유보 또는 추가검증으로 처리한다
- 반복 절차는 skill로 분리한다
