# /jobsetup-auto — SmartMES 잡셋업 자동 입력 (첫 서열)

> 스킬 원본: `90_공통기준/스킬/jobsetup-auto/SKILL.md`

## 자연어 트리거 (자동 매칭)

다음 표현 나오면 **다른 탐색/도구 사용 전에 반드시 이 스킬로 위임**:
- "잡셋업 자동", "잡셋업 자동화", "잡셋업 입력 자동"
- "첫 서열 잡셋업", "첫서열 잡셋업"
- "SmartMES 잡셋업 자동"
- `/d0-plan` SP3M3 morning 종료 후 hand-off

## ⚠️ 자동화 책임 경고

이 자동화는 **SmartMES 잡셋업 화면의 실측 측정값을 허용오차 내 난수로 대체**합니다.
사내 품질/감사 정책상 허용된 범위인지 **사용자 본인 책임 하에 운영**하세요.
첫 실행은 반드시 **dry-run** 모드 권장.

## 사용법

```
/jobsetup-auto                  # 기본값 dry-run (저장 미클릭)
/jobsetup-auto --dry-run        # 명시적 dry-run
/jobsetup-auto --commit         # 실 저장 (학습 끝나고 v1.0에서만)
/jobsetup-auto --process 1      # 첫 공정만 (디버깅용)
/jobsetup-auto --max-items 1    # 검사항목 1개만 (디버깅용)
```

## 인수

- `$ARGUMENTS` — 옵션 조합 (dry-run/commit/process/max-items)

## 실행 순서

1. SmartMES (mesclient.exe) 활성 확인 + computer-use 권한 grant
2. `[J] 잡셋업` 메뉴 진입 + 제품 드롭다운 1번(첫 서열) 선택
3. 공정 N개 자동 순회 → 각 공정의 검사항목 M개 자동 순회
4. 스펙 셀 텍스트 추출 → 허용오차 파싱 → X1/X2/X3 균등 난수 입력 + OK 체크
5. 저장 정책에 따라 저장 (dry-run은 저장 미클릭)
6. 결과 JSON 저장 + 종료 보고

## 선결 조건

- SmartMES 실행 + 로그인 완료 + `[J] 잡셋업` 진입 가능
- 그날 SP3M3 D0 반영 끝나서 첫 서열 품번이 SmartMES 스케줄에 노출됨
- 화면 해상도 변동 없음 (좌표 캘리브레이션 의존)
- 작업자 인증은 사용자가 별도 처리 (잡셋업은 단독 처리 가능)

## 안전 원칙

- **dry-run 기본값** — `--commit` 없이는 저장 안 함
- **commit 가드** — dry-run PASS 이력 1회 이상 후에만 `--commit` 허용
- **무인 실행 금지** — 사용자가 화면에서 진행 시각 확인 가능할 때만
- **단일 책임** — 첫 서열 1품번만 처리
- **스펙 매칭 실패 시 스킵** — 임의 추정값 입력 금지

## 첫 실행 학습 항목

dry-run 첫 실행에서 학습 후 SKILL.md에 박는 항목:
- 좌표 캘리브레이션 (제품/공정/검사항목 드롭다운, X1/X2/X3 입력칸, OK 체크박스, 저장 버튼)
- 스펙 텍스트 정규식 패턴 (`5.0±0.1` / `4.9~5.1` / min·max 별도 컬럼)
- 저장 단위 정책 (검사항목 단위 / 공정 단위 / 일괄)
- SmartMES 잡셋업 행 수동 삭제 절차 (R5 롤백)

자세한 절차는 `90_공통기준/스킬/jobsetup-auto/SKILL.md` 참조.
