### 1.3 해결된 함정
- **jQuery.ajax 경로 필수**: fetch 직접 호출 시 500 에러 (공통 설정 미상속). jQuery 사용 시 정상.
- **EXT_PLAN_REG_NO 매핑**: 동일 품번 상단에 여러 건 있을 때 (기존 주간용 + 신규 야간용) **REG_NO 최대값** 사용해야 신규 건 선택됨.
- **주간/야간 중복 품번**: 서버는 주간 P/F + 야간 A를 **다른 EXT_PLAN_REG_NO로 각각 별도 행** 허용. force 추가 시 반드시 신규 REG_NO 사용 (그렇지 않으면 덮어쓰기로 A 행 미생성).
- **s_grid 초기화 타이밍**: 상단 행 클릭 후 `sGridList.searchListData` 호출 전 s_grid 초기화하면 payload에 기존 행 유실. 폴링으로 기존 P/R 행 로드 확인 후 addRow.

## 2. 생산계획 운영 규칙 (사용자 설명 기반 확정)

### 2.1 파일 구조 — 단일 파일에 두 라인
- 경로: `Z:\15. SP3 메인 CAPA점검\SP3M3\생산지시서\{YYYY}년 생산지시\{MM}월\SP3M3_생산지시서_({YY.MM.DD})[- 수정본].xlsm`
- 폴더 월단위 동적 생성 (스킬에서 경로 계산)

| 라인 | ERP 코드 | 원본 시트 | 추출 필터 | 품번 컬럼 | 수량 컬럼 |
|------|---------|---------|---------|---------|---------|
| MAIN | SP3M3 (LINE_DIV_CD=02) | `출력용` | R1 헤더 `◀ D+1 야간계획`~R34 / R35 `◀ D+2 주간계획`~ | I열 (신MES 품번, 정제형) | K열 (지시) |
| OUTER | SD9A01 (LINE_DIV_CD=01) | `OUTER 생산계획` | B열 `=='SD9M01'` (D+1 블록) | D열 (품번 그대로, 접미사 포함) | E열 (수량) |

- OUTER 시트에는 7개 라인 섞여 있음 (SD9M01/SR2M01/SP3A05/SP2M02/SM3M01/SD9M02/SD9M03/D6M1). 각 라인 섹션 사이 `◀ 라인명 D+1` 헤더 행.
- **OUTER 품번은 접미사 포함 그대로 사용** (G열 품번정제 쓰지 않음) — ERP 운영 규칙

### 2.2 파일명 날짜 규칙
- 저장 시점 **전일 오후** → 파일명은 **내일 날짜(D+1)**
- 예: 4/23 오후 확정 → `SP3M3_생산지시서_(26.04.24).xlsm`
- 시트 내 헤더 `◀ D+1` = 파일명 날짜, `◀ D+2` = 파일명 날짜+1

### 2.3 업로드 세션 (하루 2회, 총 3개 배치)

파일명 = OUTER 생산일 = D+1. SP3M3 야간은 **야간 시작일(오늘) 기준** ERP 등록.

| 세션 | 실행 시점 | 업로드 내용 | 사용 파일(파일명 날짜) | **ERP 생산일** | 수량 규칙 |
|------|---------|-----------|---------|-----------|---------|
| 저녁 | 17~19시 확정계획 확인 후 | ① SP3M3 야간 (출력용 야간 섹션) | 내일 날짜 | **오늘** (파일명 날짜 - 1, 야간 시작일) | 전량 |
|  |  | ② SD9A01 OUTER D+1 (OUTER 시트 SD9M01 블록) | 같은 파일 | **내일** (파일명 날짜) | 전량 |
| 아침 | 07:10경 계획 확정 후 | ③ SP3M3 주간 (출력용 주간 섹션) | 오늘 날짜 (전날 저녁 생성) | **오늘** (파일명 날짜) | **누적 ≥ 3600 도달 행까지 포함** |
